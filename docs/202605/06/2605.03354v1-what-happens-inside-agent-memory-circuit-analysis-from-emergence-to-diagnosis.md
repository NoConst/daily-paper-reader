## 摘要
智能体记忆故障是隐性的：基于大语言模型（LLM）的智能体即使在无法跨会话提取、保留或检索所需信息时，仍能生成流畅的响应。“写入-管理-读取”循环描述了这些系统的外部流水线，但并未揭示哪些内部计算实现了每个阶段。通过追踪 Qwen-3 系列（0.6B至14B）和两种记忆框架（mem0 和 A-MEM）的内部特征电路，我们报告了三项发现。首先，控制信号早于内容信号被检测到：路由电路在 0.6B 时就具有因果活性，而内容电路在我们的追踪设置下直到 4B 才产生可检测的信号，这导致在部署阶段，小模型表现出看似胜任的路由能力，但在提取和基元化（grounding）方面却会隐性失败。其次，在内容组中，“写入”和“读取”共享一个位于后层的枢纽，该枢纽作为基座模型中已存在的上下文基元化基质发挥作用；只有记忆框架在该基质上招募了功能性基元化方向，且该枢纽在两种框架间具有迁移性。第三，涌现并不意味着可控性：尽管内容电路在 4B 时变得可检测，但直到 8B 时才变得可靠可控，这表明检测和干预具有不同的规模阈值。在实际应用方面，两组电路之间的特征空间分离使得在无监督的情况下，能够以 76.2% 的准确率实现逐操作故障定位，为原本隐性的智能体记忆故障提供了阶段级的诊断手段。

## Abstract
Agent memory failures are silent: an LLM-based agent can produce a fluent response even when it fails to extract, retain, or retrieve the information needed across sessions. The write-manage-read loop describes the external pipeline of these systems but leaves open which internal computations implement each stage. Tracing internal feature circuits across the Qwen-3 family (0.6B--14B) and two memory frameworks (mem0 and A-MEM), we report three findings. First, control is detectable before content: routing circuitry is causally active at 0.6B, while content circuitry produces no detectable signal until 4B under our tracing setup, creating a deployment regime where small models route with apparent competence but silently fail at extraction and grounding. Second, within the content group, Write and Read share a late-layer hub that operates as a context-grounding substrate already present in the base model; only memory framing recruits a functional grounding direction on this substrate, and the hub transfers across both frameworks. Third, emergence does not imply steerability: although the content circuit becomes detectable at 4B, it becomes reliably steerable only at 8B, indicating that detection and intervention have distinct scale thresholds. As a practical implication, the feature-space separation between the two circuit groups enables per-operation failure localization at 76.2% accuracy without supervision, providing a stage-level diagnostic for otherwise silent agent-memory failures.

### 1. 核心问题与整体含义
*   **研究动机**：大语言模型（LLM）智能体在处理记忆时常发生“隐性失败”（Silent Failure）。即使记忆提取、管理或检索出错，模型仍能生成流畅但不正确的响应。现有的评估方法（如端到端准确率）无法区分故障究竟发生在“写入（Write）”、“管理（Manage）”还是“读取（Read）”阶段。
*   **核心目标**：通过机械可解释性（Mechanistic Interpretability）手段，深入模型内部追踪记忆操作的特征电路，揭示记忆能力如何随模型规模涌现，并利用这些内部特征实现故障的精准诊断。

### 2. 方法论
*   **核心思想**：利用**预训练线性转码器（Pre-trained Linear Transcoders, PLTs）**将模型的 MLP 层分解为具有单语义特征的稀疏激活，从而追踪从输入 token 到输出 logit 的因果特征电路。
*   **关键技术细节**：
    *   **特征电路追踪（Five-step Pipeline）**：包括归因（计算特征对输出的影响）、聚合（跨样本统计重复出现的特征）、路径追踪（寻找关键计算路径）、因果验证（通过消融和放大实验验证特征作用）以及 Jaccard 相似度分析（量化不同操作间的电路重叠）。
    *   **因果干预**：使用“零消融”（Zero Ablation）切断特征路径，或使用“放大干预”（Amplification）增强特定特征，观察模型输出概率的变化。
    *   **故障诊断算法**：基于不同记忆阶段在特征空间上的分离性，通过“最大损伤消融”原则，在无需人工标注的情况下定位导致失败的特定阶段。

### 3. 实验设计
*   **模型对象**：Qwen-3 家族全系列（0.6B, 4B, 8B, 14B）。
*   **记忆框架**：对比了 **mem0**（基于扁平键值对）和 **A-MEM**（基于 Zettelkasten 网络结构）两种主流框架。
*   **数据集与 Benchmark**：
    *   主实验：**LongMemEval**（包含 500 个问题，涵盖 5 种记忆类型）。
    *   泛化性验证：**LoCoMo** 和 **MemoryAgentBench**。
*   **对比方法**：在故障定位任务中，对比了多数类预测、输出熵（Confidence）、行为规则过滤以及有监督的逻辑回归分类器。

*   **实验规模**：每个模型规模、每个操作阶段均生成了 200 张归因图，总计分析了数千个特征电路。
*   **充分性评价**：实验设计非常充分且严谨。它不仅跨越了模型参数规模（从 0.6B 到 14B），还跨越了不同的记忆系统实现（mem0 vs A-MEM），并包含了因果干预、交叉系统迁移验证和多基准测试。这种多维度的对比确保了结论的客观性和普适性。

### 6. 主要结论与发现
*   **控制先于内容（Control-before-content）**：路由电路（Manage）在 0.6B 规模时就已具备因果活性，而负责提取和接地的“内容电路”（Write/Read）直到 4B 规模才产生可检测信号。这意味着小模型可能表现出“看似聪明”的路由决策，但实际上无法正确处理记忆内容。
*   **共享接地枢纽（Shared Grounding Hub）**：Write 和 Read 操作在模型后层共享一个“枢纽集群”。该枢纽是模型固有的上下文处理基质，记忆框架通过特定的提示词在该基质上“招募”了功能性的接地方向。
*   **涌现与可控性的阈值差异**：内容电路在 4B 时涌现（可检测），但直到 8B 时才变得可靠可控（可干预）。
*   **高效诊断**：利用电路特征的分离性，实现了 **76.2%** 的无监督故障定位准确率，显著优于有监督的黑盒分类器。

### 7. 优点
*   **视角独特**：首次将电路分析从单次检索扩展到复杂的“写-管-读”多调用智能体流水线。
*   **理论与实践结合**：不仅提供了关于模型内部机理的科学发现，还转化为了一个实用的、无需标注的故障诊断工具。
*   **跨框架验证**：证明了发现的特征电路在不同记忆框架间具有迁移性，增强了结论的可靠性。

### 8. 不足与局限
*   **模型家族局限**：实验主要集中在 Qwen-3 系列，虽然跨越了规模，但尚未在 Llama 或 Gemma 等其他架构上验证。
*   **组件覆盖**：主要分析了 MLP 层的特征，对注意力头（Attention Heads）的追踪相对较少。
*   **提示词长度限制**：实验使用的提示词较短（45-65 tokens），在极长上下文或极其复杂的智能体交互场景下，电路动态可能会更加复杂。
