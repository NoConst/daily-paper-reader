## 摘要
长期对话记忆需要检索散布在多个会话中的证据，然而单次检索在处理时间相关和多跳问题时往往会失败。现有的迭代方法通过生成的内容或文档级信号来优化查询，但没有一种方法能明确诊断证据鸿沟（即累积检索集中缺失的内容），导致查询优化缺乏针对性。我们提出了 EviMem，它结合了 IRIS（通过不足信号进行迭代检索）和 LaceMem（对话证据记忆的分层架构）。IRIS 是一个闭环框架，通过充分性评估检测证据鸿沟，诊断缺失内容并驱动有针对性的查询优化；LaceMem 是一种由粗到精的记忆层级结构，支持细粒度的鸿沟诊断。在 LoCoMo 数据集上，EviMem 在时间相关问题（从 73.3% 提升至 81.6%）和多跳问题（从 65.9% 提升至 85.2%）上的评判准确率（Judge Accuracy）均优于 MIRIX，且延迟降低了 4.5 倍。代码：https://github.com/AIGeeksGroup/EviMem。

## Abstract
Long-term conversational memory requires retrieving evidence scattered across multiple sessions, yet single-pass retrieval fails on temporal and multi-hop questions. Existing iterative methods refine queries via generated content or document-level signals, but none explicitly diagnoses the evidence gap, namely what is missing from the accumulated retrieval set, leaving query refinement untargeted. We present EviMem, combining IRIS (Iterative Retrieval via Insufficiency Signals), a closed-loop framework that detects evidence gaps through sufficiency evaluation, diagnoses what is missing, and drives targeted query refinement, with LaceMem (Layered Architecture for Conversational Evidence Memory), a coarse-to-fine memory hierarchy supporting fine-grained gap diagnosis. On LoCoMo, EviMem improves Judge Accuracy over MIRIX on temporal (73.3% to 81.6%) and multi-hop (65.9% to 85.2%) questions at 4.5x lower latency. Code: https://github.com/AIGeeksGroup/EviMem.

*   **研究动机**：在跨越多个会话的长对话中，用户提出的问题往往涉及**时序推理**（如“某人何时改变了计划？”）或**多跳合成**（如“某团队凭借哪支舞夺冠？”）。
*   **现有局限**：传统的单次检索（Single-pass）难以捕获散落在不同会话且缺乏关键词重叠的证据。现有的迭代检索方法（如 IRCoT, FLARE）虽然引入了反馈，但大多基于生成内容的置信度或单篇文档的质量，**缺乏对“累积证据集整体是否充分”以及“具体还缺失什么”的显式诊断**，导致查询优化盲目且低效。

### 2. 方法论：EviMem 框架
EviMem 由两个相互协作的核心组件构成：

#### **LaceMem（分层记忆架构）**
提供由粗到精的记忆表示，作为诊断的基石：
*   **Index 层（索引层）**：将对话转换为原子语义元组 `(主语, 谓语, 宾语, 时间)`，支持精确的语义搜索。
*   **Edge 层（边缘层）**：通过同源关系和语义相似度连接元组，形成稀疏图，支持多跳扩展。
*   **Raw 层（原始层）**：保留原始对话文本，确保生成答案时的忠实度。

#### **IRIS（基于不足信号的迭代检索）**
实现闭环检索逻辑，其算法流程如下：
1.  **双路径检索**：同时运行“锚点路径”（原始问题）和“优化路径”（诊断驱动的查询），平衡覆盖面与针对性。
2.  **实体追踪**：监控每个实体的证据覆盖率，防止因某个关键实体信息缺失导致的推理失败。
3.  **充分性评估（核心）**：LLM 将当前证据集分类为 **EXACT（精确）**、**INFERRABLE（可推断）** 或 **PARTIAL（部分）**，并生成缺失信息的自然语言描述（诊断）。
4.  **诊断驱动优化**：利用诊断出的“证据鸿沟”来精准重写下一轮的查询语句。
5.  **自适应生成**：根据证据层级调整生成策略，若证据严重不足则选择拒绝回答（Abstain）以减少幻觉。

### 6. 主要结论与发现

*   **诊断的价值**：显式的证据鸿沟诊断比盲目的查询重写更能提高检索召回率，尤其是在第二轮迭代中提升最明显。

### 7. 优点
*   **闭环设计**：将检索从“开环序列”转变为“基于反馈的闭环系统”，解决了语义漂移问题。
*   **细粒度诊断**：利用原子元组（Tuples）进行充分性评估，比传统的文档级评估更精准。
*   **高可靠性**：通过三级充分性分类和实体追踪，系统能有效识别何时该“闭嘴”，显著降低了对抗性场景下的幻觉。

### 8. 不足与局限
*   **静态记忆构建**：目前的 LaceMem 索引是基于对话快照离线构建的，尚未在论文中展示如何实时、流式地更新记忆层。

*   **API 成本**：虽然比多智能体系统便宜，但多次迭代仍涉及多次 LLM 调用，在极大规模应用中成本仍需进一步优化。
