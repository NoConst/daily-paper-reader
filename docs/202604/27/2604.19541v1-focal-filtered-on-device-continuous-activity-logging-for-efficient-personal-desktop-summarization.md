## 摘要
桌面交互流提供了交织用户任务的连续且隐私敏感的记录。在端侧将这些流转换为按任务组织的个人日志面临两个主要挑战：详尽的视觉语言模型（VLM）处理会耗尽本地资源，而全局流处理会导致跨任务的上下文污染。我们提出了 FOCAL（过滤式端侧连续活动日志记录），这是一个采用统一“过滤-计划-记录”架构的隐私优先多智能体系统。它级联了一个用于噪声抑制的轻量级过滤智能体（Filter Agent）、一个用于任务归因的纯文本大脑智能体（Brain Agent）、一个用于选择性视觉推理的记录智能体（Record Agent）以及一个用于上下文连贯摘要的任务隔离记忆智能体（Memory Agent）。在 DesktopBench（包含 420 个复杂会话中的 2,572 张截图）上的实验表明，与基准模型相比，FOCAL 将总 Token 消耗降低了 60.4%，VLM 调用次数减少了 72.3%，同时将关键信息召回率（KIR）从 0.38 提升至 0.61。至关重要的是，在 A→B→A 任务中断情况下，FOCAL 保持了 0.81 的任务准确率（Task Acc）和 0.80 的 KIR，而基准模型的任务准确率则崩溃至 0.03。FOCAL 开创了将无指令桌面流高效、端侧地摘要为多视角个人日志的先河。

## Abstract
Desktop interaction streams provide a continuous, privacy-sensitive record of interleaved user tasks. Transforming these streams into task-organized personal logs on-device faces two main challenges: exhaustive Vision-Language Model (VLM) processing strains local resources, and global stream processing causes cross-task context pollution. We present FOCAL (Filtered On-device Continuous Activity Logging), a privacy-first multi-agent system utilizing a unified filter-plan-log architecture. It cascades a lightweight Filter Agent for noise suppression, a text-only Brain Agent for task attribution, a Record Agent for selective visual reasoning, and a task-isolated Memory Agent for context-coherent summarization. Experiments on DesktopBench (comprising 2,572 screenshots across 420 complex sessions) show FOCAL reduces total token consumption by 60.4% and VLM call count by 72.3% versus a baseline, while boosting Key Information Recall (KIR) from 0.38 to 0.61. Crucially, under $A{\to}B{\to}A$ task interruptions, FOCAL maintains Task Acc 0.81 and KIR 0.80, whereas the baseline collapses to Task Acc 0.03. FOCAL pioneers the efficient, on-device summarization of instruction-free desktop streams into multi-perspective personal logs.

### FOCAL 论文深度总结

#### 1. 核心问题与整体含义（研究动机和背景）
论文关注的是**个人桌面活动日志（Desktop Activity Logging）**的自动化生成。桌面工作流通常具有多任务交织（如在写文档、查资料、回复邮件间频繁切换）和高度隐私敏感的特点。
*   **研究动机**：现有的视觉语言模型（VLM）虽然能理解屏幕内容，但在端侧（On-device）部署面临两大挑战：
    1.  **资源消耗过高**：对每一帧截图进行 VLM 推理会严重占用本地 CPU/GPU 资源，导致系统卡顿。
    2.  **上下文污染（Context Pollution）**：用户在不同任务间切换时，传统的全局处理方式会将不相关的任务信息混在一起，导致生成的摘要逻辑混乱。
*   **核心目标**：在保护隐私的前提下，实现一种高效、低功耗且能准确区分交织任务的端侧桌面活动总结系统。

#### 2. 方法论：核心思想与技术细节
FOCAL 提出了一个**“过滤-规划-记录”（Filter-Plan-Log）**的多智能体架构，核心思想是**“在昂贵的视觉推理之前进行任务感知的控制”**。

*   **核心架构（五个智能体）**：
    1.  **Filter Agent（过滤智能体）**：轻量级模块，负责捕获截图并剔除无意义的静止或冗余帧。
    2.  **Brain Agent（大脑智能体）**：**关键创新点**。仅利用轻量级的文本元数据（如应用名称、窗口标题）对整个会话进行一次性规划，决定哪些帧需要 VLM 处理（自适应采样），并将每一帧归类到特定的任务 ID。
    3.  **Record Agent（记录智能体）**：仅对 Brain Agent 选中的关键帧调用 VLM，结合当前任务的局部记忆生成语义描述。
    4.  **Memory Agent（记忆智能体）**：实现**任务隔离记忆**。为每个识别出的任务维护独立的存储空间，防止 A 任务的信息干扰 B 任务。
    5.  **Summary Agent（摘要智能体）**：在会话结束时，基于各任务隔离的记忆生成结构化的多视角总结。
*   **算法流程**：系统首先通过元数据识别任务边界和重要性，随后进行有选择的视觉推理，最后将碎片化的观察结果按任务路由到对应的记忆库中进行聚合。
