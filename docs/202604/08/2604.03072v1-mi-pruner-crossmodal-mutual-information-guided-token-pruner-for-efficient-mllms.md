## 摘要
对于多模态大语言模型 (MLLMs)，视觉信息相对于文本而言较为稀疏。因此，视觉剪枝研究应运而生，旨在实现高效推理。目前的方法通常基于视觉编码器或 LLM 解码器中的注意力分数来衡量 Token 的重要性，随后选择高注意力分数的视觉 Token 并剪除其余部分。在本文中，我们采用了一种不同且更精准的方法。我们不依赖于特定机制的信号，而是在视觉和文本特征交互之前，直接计算它们之间的互信息 (MI)。这使我们能够在特征层面显式地衡量跨模态依赖性。我们的 MI-Pruner 简单、高效且具有非侵入性，无需访问内部注意力图或进行架构修改。实验结果表明，我们的方法在极低延迟的情况下优于以往基于注意力的剪枝方法。

## Abstract
For multimodal large language models (MLLMs), visual information is relatively sparse compared with text. As a result, research on visual pruning emerges for efficient inference. Current approaches typically measure token importance based on the attention scores in the visual encoder or in the LLM decoder, then select visual tokens with high attention scores while pruning others. In this paper, we pursue a different and more surgical approach. Instead of relying on mechanism-specific signals, we directly compute Mutual Information (MI) between visual and textual features themselves, prior to their interaction. This allows us to explicitly measure crossmodal dependency at the feature levels. Our MI-Pruner is simple, efficient and non-intrusive, requiring no access to internal attention maps or architectural modifications. Experimental results demonstrate that our approach outperforms previous attention-based pruning methods with minimal latency.

多模态大语言模型（MLLM）在处理高分辨率图像时，视觉 Token 的数量远超文本，且由于自注意力机制的平方复杂度，导致推理成本极高。现有的视觉剪枝方法主要依赖**注意力分数**（来自视觉编码器或 LLM 解码器）来衡量 Token 重要性。然而，作者指出注意力分数存在局限性：
*   **噪声与偏差**：受位置偏差、注意力汇聚（Attention Sinks）和多头多样性影响，注意力分数并不等同于信息重要性。
*   **实现瓶颈**：提取内部注意力图与 FlashAttention 等加速算子不兼容，且增加内存开销。
*   **动机**：本文提出直接在投影空间（Projection Space）计算视觉与文本特征之间的**互信息（Mutual Information, MI）**，以一种非侵入式、理论完备且高效的方式识别并保留与查询最相关的视觉 Token。

### 2. 方法论
MI-Pruner 的核心思想是将 Token 剪枝建模为一个**子模态函数最大化问题**，通过最大化跨模态互信息来选择视觉子集。

*   **核心流程**：
    1.  **特征提取与归一化**：在投影层后提取视觉特征 $V$ 和文本特征 $T$，并映射到单位超球面上。
    2.  **分布构建**：利用 Boltzmann 分布计算视觉-文本（跨模态）和视觉-视觉（模态内）的相似度矩阵，进而推导出条件概率分布 $p(t|v)$ 和边缘分布 $p(t)$。
    3.  **点向互信息（PMI）计算**：计算每个视觉 Token 与文本 Token 之间的 PMI，衡量语义相关性；同时计算视觉 Token 间的 PMI，衡量冗余度。
    4.  **贪心搜索选择**：定义评分函数 $S_i = \lambda S_{cross} - (1-\lambda) S_{self}$。其中 $S_{cross}$ 捕捉最强的跨模态关联，$S_{self}$ 捕捉模态内冗余。
    5.  **高效推理**：该方法不依赖 LLM 内部交互，复杂度仅为 $O(N_V)$，可作为即插即用的模块集成到任何 Enc-MLP-Dec 架构的模型中。
