## 摘要
多页文档视觉问答（DocVQA）需要对长篇且视觉密集的文档中的语义、布局和视觉元素进行推理。现有的无 OCR（OCR-free）方法在处理能力与精度之间面临权衡：端到端模型在处理长文档时扩展性较差，而基于视觉检索的流水线则显得脆弱且被动。我们提出了 Doc-$V^*$，这是一个无 OCR 的智能体（agentic）框架，将多页 DocVQA 建模为序列化的证据聚合过程。Doc-$V^*$ 从缩略图概览开始，随后通过语义检索和针对性的页面获取进行主动导航，并在结构化工作记忆中聚合证据以进行有据推理。通过专家轨迹的模仿学习进行训练，并利用群体相对策略优化（GRPO）进一步优化，Doc-$V^*$ 在回答准确性与证据寻找效率之间取得了平衡。在五个基准测试中，Doc-$V^*$ 的表现优于开源基准模型，并接近闭源商业模型，在域外性能上比 RAG 基准提升了高达 47.9%。其他结果表明，有效的证据聚合源于选择性注意力，而非增加输入页数。

## Abstract
Multi-page Document Visual Question Answering requires reasoning over semantics, layouts, and visual elements in long, visually dense documents. Existing OCR-free methods face a trade-off between capacity and precision: end-to-end models scale poorly with document length, while visual retrieval-based pipelines are brittle and passive. We propose Doc-$V^*$, an \textbf{OCR-free agentic} framework that casts multi-page DocVQA as sequential evidence aggregation. Doc-$V^*$ begins with a thumbnail overview, then actively navigates via semantic retrieval and targeted page fetching, and aggregates evidence in a structured working memory for grounded reasoning. Trained by imitation learning from expert trajectories and further optimized with Group Relative Policy Optimization, Doc-$V^*$ balances answer accuracy with evidence-seeking efficiency. Across five benchmarks, Doc-$V^*$ outperforms open-source baselines and approaches proprietary models, improving out-of-domain performance by up to \textbf{47.9\%} over RAG baseline. Other results reveal effective evidence aggregation with selective attention, not increased input pages.

#### 1. 核心问题与整体含义（研究动机和背景）
多页文档视觉问答（DocVQA）要求模型能够同时处理长文档中的语义、布局和视觉元素（如表格、图表）。现有的无 OCR 方法面临**容量与精度的权衡**：
*   **端到端模型**：受限于二次方注意力成本和上下文长度，处理长文档时扩展性差，且存在“信息丢失在中间”的问题。
*   **检索增强生成（RAG）**：虽然降低了噪声，但属于被动处理，容易受到检索错误的影响，且难以进行多跳推理。
*   **研究动机**：受人类阅读行为（先看缩略图概览，再按需寻找证据）启发，论文提出 Doc-V* 框架，将 DocVQA 视为一个**主动的、序列化的证据聚合过程**。

#### 2. 方法论
Doc-V* 是一个基于智能体（Agent）的无 OCR 框架，核心流程如下：
*   **全局缩略图概览（Global Thumbnail Overview）**：将文档页面调整为低分辨率缩略图并排列成网格，作为初始观察。这提供了文档类型、布局和图表分布的宏观先验，显著降低了初始视觉 Token 消耗（压缩比约 10-12 倍）。
*   **交互式动作空间**：
    *   `retrieval_page`：类似于“Ctrl+F”，通过语义查询检索未访问页面。
    *   `fetch_page`：根据缩略图观察或上下文，直接获取特定页码的高分辨率图像。
    *   `answer`：在证据充足时终止交互并输出答案。
*   **结构化视觉推理（ReAct 风格）**：模型输出遵循“思考（分析、计划、总结）- 动作”协议。
*   **结构化工作记忆（Working Memory）**：通过拼接每一步的摘要，缓解长序列下的遗忘和重复获取问题。
*   **两阶段训练策略**：
    1.  **监督微调（SFT）**：使用 GPT-4o 合成的 9,019 条高质量交互轨迹进行模仿学习。
    2.  **群体相对策略优化（GRPO）**：利用强化学习，通过奖励信号（答案准确性、证据召回率、格式规范性）联合优化准确率与搜索效率。

*   **实验规模**：在 5 个主流基准测试上进行了全面评估，涵盖了从标准文档到超长、异构文档的多种场景。
*   **消融实验**：针对全局缩略图、页面分析模块、工作记忆以及不同动作（检索 vs. 获取）的必要性做了详细消融。

#### 6. 主要结论与发现
*   **性能卓越**：Doc-V* 在开源模型中表现最佳，甚至在某些指标上超越了 GPT-4o 和 Claude-3.7。
*   **泛化能力强**：在域外（OOD）设置下，相比静态 RAG 基准提升了高达 **47.9%**。
*   **效率与精度的平衡**：长文档理解的关键在于**选择性注意力的证据聚合**，而非单纯增加输入页数。增加页数往往会引入噪声，而 Doc-V* 能在保持较低平均输入页数的情况下获得更高精度。
*   **纠错能力**：即使初始检索器较弱，Doc-V* 也能通过主动的 `fetch_page` 动作补偿丢失的信息。

#### 7. 优点
*   **拟人化推理**：由粗到精的策略符合人类处理复杂文档的认知逻辑。
*   **Token 效率高**：通过缩略图和按需获取高分辨率页面，极大地节省了计算资源。
*   **端到端优化**：利用 GRPO 强化学习直接优化决策策略，使模型在有限步数内更具目的性。

#### 8. 不足与局限
*   **模型底座依赖**：实验主要基于 Qwen2.5-VL，尚未在其他视觉语言模型底座上验证通用性。
*   **单文档限制**：目前仅针对单文档场景，尚未扩展到需要跨多个异构文档聚合信息的复杂任务。
*   **幻觉风险**：尽管有工作记忆，但在极端复杂的推理链中，模型仍可能产生幻觉或错误的证据关联。
