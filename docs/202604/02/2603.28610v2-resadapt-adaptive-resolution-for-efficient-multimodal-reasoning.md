## 摘要
多模态大语言模型（MLLMs）通过提高输入保真度实现了更强的视觉理解能力，但由此产生的视觉 Token 增长使得同时维持高空间分辨率和长时序上下文变得代价高昂。我们认为，瓶颈不在于编码后的表示如何压缩，而在于编码器接收的像素量。为此，我们提出了 ResAdapt，这是一种输入端自适应框架，用于学习每帧在编码前应分配多少视觉预算。ResAdapt 将轻量级分配器（Allocator）与保持不变的 MLLM 主干网络相结合，使主干网络在接收经过算子转换的输入时，仍能保留其原生的视觉 Token 接口。我们将分配问题建模为上下文老虎机（contextual bandit），并使用成本感知策略优化（CAPO）来训练分配器，该方法将稀疏的展开反馈（rollout feedback）转化为稳定的准确度-成本学习信号。在预算受限的视频问答、时序定位和图像推理任务中，ResAdapt 提升了低预算运行点的表现，且通常处于或接近效率-准确度前沿，在激进压缩下的推理密集型基准测试中增益最为显著。值得注意的是，在相同的视觉预算下，ResAdapt 支持多达 16 倍的帧数，同时带来超过 15% 的性能提升。代码可在 https://github.com/Xnhyacinth/ResAdapt 获取。

## Abstract
Multimodal Large Language Models (MLLMs) achieve stronger visual understanding by scaling input fidelity, yet the resulting visual token growth makes jointly sustaining high spatial resolution and long temporal context prohibitive. We argue that the bottleneck lies not in how post-encoding representations are compressed but in the volume of pixels the encoder receives, and address it with ResAdapt, an Input-side adaptation framework that learns how much visual budget each frame should receive before encoding. ResAdapt couples a lightweight Allocator with an unchanged MLLM backbone, so the backbone retains its native visual-token interface while receiving an operator-transformed input. We formulate allocation as a contextual bandit and train the Allocator with Cost-Aware Policy Optimization (CAPO), which converts sparse rollout feedback into a stable accuracy-cost learning signal. Across budget-controlled video QA, temporal grounding, and image reasoning tasks, ResAdapt improves low-budget operating points and often lies on or near the efficiency-accuracy frontier, with the clearest gains on reasoning-intensive benchmarks under aggressive compression. Notably, ResAdapt supports up to 16x more frames at the same visual budget while delivering over 15% performance gain. Code is available at https://github.com/Xnhyacinth/ResAdapt.

### 1. 核心问题与整体含义（研究动机和背景）
多模态大语言模型（MLLM）在处理长视频或高分辨率图像时，面临**视觉标记（Visual Token）爆炸**的问题。现有的效率优化方案主要存在两个缺陷：
*   **干预过晚（Model-side）：** 如Token剪枝或合并，通常发生在视觉编码器处理之后，此时编码器已经支付了全额计算成本，且不规则的Token布局破坏了推理引擎（如FlashAttention）的优化。
*   **延迟过高（Output-side）：** 如迭代检索或放大（Zoom-in），需要多轮模型调用，增加了推理延迟。
**研究核心：** 提出 **ResAdapt** 框架，主张在**编码前（Input-side）**进行干预，通过轻量级分配器动态调整每帧的分辨率，从而在源头上节省计算资源，并将节省的空间预算转化为更长的时序覆盖。

### 2. 方法论：核心思想与关键技术
ResAdapt 的核心是将每帧的视觉预算分配建模为一个**上下文老虎机（Contextual Bandit）**问题。
*   **轻量级分配器（Allocator）：** 采用 SmolVLM 架构，输入粗糙的视觉特征和文本查询（Query），通过 Transformer 解码器生成每帧的隐藏状态，并预测 Beta 分布的参数（$\alpha, \beta$），从中采样得到每帧的缩放因子 $s_t$。
*   **成本感知策略优化（CAPO）：** 针对分配策略不可微且容易坍缩到最小预算的问题，提出 CAPO 算法。
    *   **动态成本基准（Dynamic Pivot）：** 结合全局目标和当前批次的平均成本，建立动态决策边界。
    *   **非对称奖励塑造：** 对“正确且低成本”的回答给予适度奖励，对“错误且高成本”的回答给予严厉惩罚，防止模型为了节省成本而牺牲精度。
*   **时序相似性正则化（$L_{sim}$）：** 惩罚对视觉相似的相邻帧同时分配高预算的行为，强制模型在冗余帧中进行选择性关注。
*   **操作算子（Operator）：** 实验中主要使用双线性插值（Resize）作为算子，将分配的比例转化为实际像素输入。

### 6. 主要结论与发现
*   **效率前沿：** ResAdapt 在仅保留约 **10% 视觉 Token** 的情况下，性能仍能匹配或超过全量 Token 的基准模型。
*   **时序增益：** 在相同的计算预算下，ResAdapt 可处理 **16 倍**更多的帧数，在长视频推理任务上实现了超过 **15%** 的相对性能提升。
*   **主动感知：** 学习到的策略展现出“主动感知”特性，能自动在关键帧（如出现文字、场景突变、动作关键点）处分配高分辨率，而在冗余背景帧处使用极低分辨率。
*   **算子泛化：** 尽管是在 Resize 算子上训练的，但该策略可以零样本迁移到“帧选择”任务中，且表现优于传统的均匀采样。

### 7. 优点
*   **架构兼容性：** 由于在编码前处理，完全兼容现有的 FlashAttention、vLLM 等硬件加速内核，无需定制算子。
*   **训练稳定性：** CAPO 算法有效解决了强化学习在处理成本敏感型任务时的策略坍缩问题。
*   **端到端学习：** 分配策略完全由任务奖励驱动，无需显式的显著性（Saliency）标注。

### 8. 不足与局限
*   **固定开销：** 分配器本身存在计算开销，在短视频（如 <32 帧）场景下，分配器带来的加速效果会被其自身的推理耗时部分抵消。
*   **开环决策：** 分配器在模型推理前就做出了决定，属于“开环”控制。如果分配器漏掉了关键细节，后续的推理过程无法动态请求更高分辨率的重编码。
*   **跨模态局限：** 虽然在视频上表现极佳，但在纯图像任务（尤其是 OCR 密集型）上的表现提升不如视频任务显著。
