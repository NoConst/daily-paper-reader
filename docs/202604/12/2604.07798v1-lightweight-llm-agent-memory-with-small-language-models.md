## 摘要
尽管大语言模型（LLM）智能体能够利用工具处理复杂任务，但它们仍需要记忆来维持跨轮次的一致性，并在长程交互中积累可重用的信息。然而，基于检索的外部记忆系统虽然在线开销较低，但由于查询构建和候选过滤的局限性，其准确性并不稳定。相比之下，许多系统在在线记忆操作中重复调用大模型，虽然提高了准确性，但在长程交互中会不断累积延迟。我们提出了 LightMem，这是一种由小语言模型（SLM）驱动的轻量级记忆系统，旨在提升智能体的记忆能力。LightMem 将记忆检索、写入和长期整合模块化，并将在线处理与离线整合分离，从而在有限的计算资源下实现高效的记忆调用。我们将记忆组织为用于即时对话上下文的短期记忆（STM）、用于可重用交互摘要的中期记忆（MTM）以及用于整合知识的长期记忆（LTM），并利用用户标识符支持多用户场景下的独立检索和增量维护。在线运行时，LightMem 在固定的检索预算下工作，并通过两阶段过程选择记忆：先进行基于向量的粗筛，再进行语义一致性重排序。在离线状态下，它提取可重用的交互证据并将其增量整合到长期记忆中。实验表明，该方法在不同规模的模型上均有提升，在 LoCoMo 数据集上的平均 F1 值提高了约 2.5，且表现出更高的效率和较低的中位数延迟（检索延迟 83 毫秒；端到端延迟 581 毫秒）。

## Abstract
Although LLM agents can leverage tools for complex tasks, they still need memory to maintain cross-turn consistency and accumulate reusable information in long-horizon interactions. However, retrieval-based external memory systems incur low online overhead but suffer from unstable accuracy due to limited query construction and candidate filtering. In contrast, many systems use repeated large-model calls for online memory operations, improving accuracy but accumulating latency over long interactions. We propose LightMem, a lightweight memory system for better agent memory driven by Small Language Models (SLMs). LightMem modularizes memory retrieval, writing, and long-term consolidation, and separates online processing from offline consolidation to enable efficient memory invocation under bounded compute. We organize memory into short-term memory (STM) for immediate conversational context, mid-term memory (MTM) for reusable interaction summaries, and long-term memory (LTM) for consolidated knowledge, and uses user identifiers to support independent retrieval and incremental maintenance in multi-user settings. Online, LightMem operates under a fixed retrieval budget and selects memories via a two-stage procedure: vector-based coarse retrieval followed by semantic consistency re-ranking. Offline, it abstracts reusable interaction evidence and incrementally integrates it into LTM. Experiments show gains across model scales, with an average F1 improvement of about 2.5 on LoCoMo, more effective and low median latency (83 ms retrieval; 581 ms end-to-end).

### 1. 核心问题与整体含义（研究动机和背景）
在长程交互和复杂任务中，LLM 智能体需要记忆系统来维持跨轮次的一致性。现有的记忆系统面临**效率与准确性的权衡**：
*   **检索式外部记忆（Retrieval-based）：** 虽然在线开销低，但由于查询构建简单和候选过滤不足，容易引入噪声，导致准确性不稳定。
*   **大模型驱动记忆（LLM-driven）：** 通过多次调用大模型进行记忆读写，准确性高但延迟巨大，难以满足实时交互需求。
*   **研究动机：** 利用**小语言模型（SLM）**处理高频、结构化的在线记忆操作，将沉重的记忆抽象与整合任务交给离线的大模型，从而在保证准确性的同时大幅降低延迟。

### 2. 方法论：核心思想与技术细节
LightMem 的核心思想是**模块化解耦**和**多级记忆架构**：

*   **三层记忆结构：**
    1.  **短期记忆 (STM)：** 即 SLM 的上下文窗口，存储最近的对话，不持久化。
    2.  **中期记忆 (MTM)：** 存储用户特定的情节摘要、时间戳和访问统计，支持用户隔离。
    3.  **长期记忆 (LTM)：** 存储脱敏的、跨用户的语义知识，以**轻量级图结构**组织，支持多跳推理。
*   **在线处理流水线（由三个专门的 SLM 驱动）：**
    1.  **控制器 (SLM-1)：** 负责意图识别和查询规划。将用户输入重写为多个“假设性查询 (HQs)”，并根据意图在 MTM 和 LTM 之间分配检索预算。
    2.  **选择器 (SLM-2)：** 执行**两阶段检索**。第一阶段进行基于向量的粗筛（返回 2K 个候选）；第二阶段进行语义一致性重排序和压缩（筛选出最终的 Top-K 记忆）。
    3.  **写入器 (SLM-3)：** 在每轮对话后提取关键信息，增量更新 MTM，并处理冗余或冲突信息。
*   **离线整合流水线：**
    *   由大模型（如 GPT-4o）异步执行。将 MTM 中的高价值情节抽象为 LTM 中的知识节点和边，实现记忆的长期演化，且不阻塞在线交互。

*   **性能提升：** 在 LoCoMo 基准上，LightMem 相比最强基准 A-MEM，F1 分数平均提升了约 **2.5**。在多跳和时间推理任务中优势尤为明显。
*   **语义一致性：** 在 DialSim 上，SBERT 语义相似度显著提高，证明其不仅是词汇重叠的增加，更是对记忆理解的深化。

### 7. 优点与亮点
*   **高效的 SLM 协作：** 巧妙地将 SLM 定位为“控制器”和“过滤器”，而非生成器，充分发挥了小模型在结构化任务中的速度优势。
*   **在线/离线解耦：** 这种架构设计解决了记忆系统随时间增长而导致的延迟累积问题，使系统具备良好的可扩展性。
*   **两阶段检索机制：** 通过语义重排弥补了向量检索在复杂查询下的精度不足。
*   **多用户支持：** 通过用户标识符和 MTM/LTM 的划分，兼顾了个人隐私与知识共享。

### 8. 不足与局限
*   **意图分解失败风险：** 在处理模糊查询时，SLM-1 可能会生成过多的 HQs，导致检索到的内容虽然包含正确答案，但会引入无关的通用知识（Focus Dilution），影响回答的简洁性。
