## 摘要
大语言模型（LLM）个性化的终极目标是为每个用户提供一个单独的 LLM，使其与该用户的偏好完美对齐。然而，由于计算、内存和系统复杂性的限制，为每个用户维护一个独立的 LLM 是不切实际的。我们通过开发一种有原则的方法来应对这一挑战，该方法选择一小组 LLM 组合，以捕捉异构用户中的代表性行为。我们通过多维权重向量对用户在多个特征（如安全性、幽默感、简洁性）上的偏好进行建模。给定这些维度的奖励函数，我们的算法 PALM（对齐 LLM 组合）生成一小组 LLM 组合，使得对于任何权重向量，该组合中都包含一个针对相应标量化目标的近乎最优的 LLM。据我们所知，这是首个在 LLM 个性化组合的规模和近似质量方面都提供理论保证的研究结果。它刻画了系统成本与个性化之间的权衡，以及覆盖用户偏好全景所需的 LLM 多样性。我们提供的实证结果验证了这些保证，并展示了相比于常用基准模型更出色的输出多样性。

## Abstract
The holy grail of LLM personalization is a single LLM for each user, perfectly aligned with that user's preferences. However, maintaining a separate LLM per user is impractical due to constraints on compute, memory, and system complexity. We address this challenge by developing a principled method for selecting a small portfolio of LLMs that captures representative behaviors across heterogeneous users. We model user preferences across multiple traits (e.g., safety, humor, brevity) through a multi-dimensional weight vector. Given reward functions across these dimensions, our algorithm PALM (Portfolio of Aligned LLMs) generates a small portfolio of LLMs such that, for any weight vector, the portfolio contains a near-optimal LLM for the corresponding scalarized objective. To the best of our knowledge, this is the first result that provides theoretical guarantees on both the size and approximation quality of LLM portfolios for personalization. It characterizes the trade-off between system cost and personalization, as well as the diversity of LLMs required to cover the landscape of user preferences. We provide empirical results that validate these guarantees and demonstrate greater output diversity over common baselines.

### 2. 方法论：PALM 算法
*   **核心思想**：利用多目标对齐中的线性标量化框架。虽然用户偏好权重是连续且无限的，但一个针对特定权重优化的策略在邻近权重下通常也是近优的。
*   **关键技术细节**：
    *   **偏好建模**：将用户偏好表示为概率单纯形 $\Delta^d$ 上的权重向量 $w$，目标是最大化奖励函数的加权和减去正则项（如 KL 散度）。
    *   **PALM 算法流程**：
        1.  **网格构建（Construct Grid）**：结合“乘性网格”（处理单纯形内部）和“加性网格”（处理边界），生成一组结构化的权重向量。这种混合设计能以更少的点实现更高的覆盖精度。
        2.  **策略训练**：调用预定义的 Oracle（如 PPO 或 GRPO 算法）为网格中的每个权重训练最优策略。
        3.  **剪枝（Pruning）**：利用贪心算法解决“最小集合覆盖问题”，剔除冗余策略，保留能以指定精度（$\epsilon, \delta$）覆盖全空间的最小策略子集。
*   **理论保证**：论文首次给出了关于组合规模和近似质量的显式界限，证明了所需模型数量仅与奖励维度 $d$ 相关，而与用户数量无关。

### 3. 实验设计
*   **实验场景与数据集**：
    1.  **RLVR-GSM**：数学推理任务，目标是平衡“简洁度”与“正确性/帮助性”。
    2.  **Safety Alignment**：安全对齐任务，平衡“有用性”与“安全性”。
    3.  **Helpful Assistants**：助手对话任务，涉及“有用性”、“无害性”和“幽默感”三个维度。
*   **对比基准（Benchmarks）**：

*   **评估指标**：近似误差（乘性差距 $\epsilon$ 和加性差距 $\delta$）、策略使用分布的困惑度（Perplexity，衡量组合内模型利用率）。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了 2 维和 3 维奖励空间，组合规模从 1 扩展到 7。
*   **消融实验**：进行了“Uniform PALM”实验，验证了 PALM 特有的初始权重生成方案优于简单的均匀采样后剪枝。

### 6. 主要结论与发现
*   **极小规模覆盖**：在所有测试任务中，**少于 7 个策略**的组合即可实现近乎最优（亚最优性小于 1.5%）的性能，足以覆盖广泛的用户偏好。
*   **PALM 的优越性**：在相同组合规模下，PALM 的近似误差显著低于随机和均匀基准。
*   **行为多样性**：PALM 生成的模型在行为上更具区分度，而基准方法常产生行为高度相似的冗余模型。
*   **成本可扩展性**：个性化成本不再随用户增长，而是转化为一个固定规模的多模型服务问题。

### 7. 优点
*   **理论与实践结合**：不仅提出了算法，还给出了严谨的收敛性和规模证明。
*   **系统友好**：通过剪枝显著降低了推理部署的负担，对工业界模型下线和资源调度具有指导意义。
*   **诊断价值**：PALM 的覆盖保证可以作为诊断工具，如果组合内缺乏多样性，说明底层奖励函数未能捕捉到独特的行为信号。

### 8. 不足与局限
*   **线性标量化假设**：方法基于奖励的线性组合，可能无法捕捉用户更复杂的非线性偏好（如阈值约束）。
*   **Oracle 依赖**：算法假设可以为任何权重训练出最优策略，但在实际中，RL 训练的不稳定性可能导致生成的策略并非真正最优。
*   **维度灾难风险**：虽然模型数与用户数无关，但随奖励维度 $d$ 的增加，网格点数仍呈指数增长，对于超多目标（如 $d > 10$）的个性化可能仍面临挑战。
*   **代理模型偏差**：部分实验使用了 MOD 推理时对齐作为微调的代理，这可能与实际全量微调后的模型行为存在细微偏差。
