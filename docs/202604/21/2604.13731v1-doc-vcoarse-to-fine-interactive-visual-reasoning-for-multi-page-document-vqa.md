## 摘要
多页文档视觉问答（DocVQA）需要对长篇且视觉密集的文档中的语义、布局和视觉元素进行推理。现有的无 OCR 方法在容量和精度之间面临权衡：端到端模型在处理长文档时扩展性较差，而基于视觉检索的流水线则显得脆弱且被动。我们提出了 Doc-$V^*$，这是一个无 OCR 的智能体框架，它将多页 DocVQA 建模为序列化的证据聚合过程。Doc-$V^*$ 从缩略图概览开始，随后通过语义检索和针对性的页面获取进行主动导航，并在结构化工作记忆中聚合证据以进行有据推理。通过专家轨迹的模仿学习进行训练，并利用群体相对策略优化（GRPO）进一步优化，Doc-$V^*$ 在回答准确性与证据寻找效率之间取得了平衡。在五个基准测试中，Doc-$V^*$ 的表现优于开源基准模型，并接近闭源商业模型，在域外性能上比 RAG 基准提升了高达 47.9%。其他结果表明，有效的证据聚合源于选择性注意力，而非增加输入页数。

## Abstract
Multi-page Document Visual Question Answering requires reasoning over semantics, layouts, and visual elements in long, visually dense documents. Existing OCR-free methods face a trade-off between capacity and precision: end-to-end models scale poorly with document length, while visual retrieval-based pipelines are brittle and passive. We propose Doc-$V^*$, an \textbf{OCR-free agentic} framework that casts multi-page DocVQA as sequential evidence aggregation. Doc-$V^*$ begins with a thumbnail overview, then actively navigates via semantic retrieval and targeted page fetching, and aggregates evidence in a structured working memory for grounded reasoning. Trained by imitation learning from expert trajectories and further optimized with Group Relative Policy Optimization, Doc-$V^*$ balances answer accuracy with evidence-seeking efficiency. Across five benchmarks, Doc-$V^*$ outperforms open-source baselines and approaches proprietary models, improving out-of-domain performance by up to \textbf{47.9\%} over RAG baseline. Other results reveal effective evidence aggregation with selective attention, not increased input pages.

#### 1. 核心问题与整体含义（研究动机和背景）
多页文档视觉问答（DocVQA）要求模型能够理解长文档中的语义、布局和视觉元素（如表格、图表）。现有的无 OCR 方法面临**容量与精度**的权衡：
*   **端到端模型（E2E）：** 难以扩展到超长文档，受限于计算成本、上下文长度限制以及“迷失在中间（lost-in-the-middle）”效应。
*   **检索增强生成（RAG）：** 过程是被动的，容易受到检索错误的影响，且难以处理需要多跳推理的复杂问题。
*   **研究动机：** 模仿人类阅读行为——先通过缩略图快速浏览获取全局结构，再根据需求主动寻找、验证并整合证据。

#### 2. 方法论：核心思想与技术细节
Doc-V* 是一个**无 OCR 的智能体（Agentic）框架**，将 DocVQA 建模为一个序列化的证据聚合过程。
*   **全局缩略图概览（Global Thumbnail Overview）：** 将文档页面调整为低分辨率（256x256）并排列成网格图，作为初始观察，提供布局、标题和图表分布等导航先验。
*   **交互式推理协议（ReAct 风格）：** 智能体在每一步输出 `<think>`（包含分析、计划、总结）和 `<action>`。
*   **动作空间（Action Space）：**
    1.  `retrieval_page`：基于语义查询进行页面检索（类似 Ctrl+F）。
    2.  `fetch_page`：根据页码直接获取高分辨率页面（用于查看邻近页或缩略图锁定的页面）。
    3.  `answer`：在证据充足时输出最终答案。
*   **结构化工作记忆（Working Memory）：** 串联每一步的推理摘要，防止智能体在多轮交互中遗忘或陷入循环。
*   **两阶段训练策略：**
    1.  **SFT（监督微调）：** 使用 GPT-4o 蒸馏出的高质量交互轨迹进行模仿学习。
    2.  **GRPO（群体相对策略优化）：** 强化学习阶段，通过奖励函数（涵盖答案准确性、证据召回率和格式规范性）优化推理效率和准确性。

#### 6. 主要结论与发现
*   **性能卓越：** Doc-V* 在开源模型中达到领先水平，在 DUDE 等测试中甚至超越了部分闭源模型。
*   **泛化能力强：** 在域外（OOD）设置下，相比静态 RAG 基线提升了高达 47.9%。

#### 7. 优点（亮点）
*   **主动感知：** 改变了传统模型被动接收信息的模式，通过交互式导航降低了噪声干扰。
*   **多粒度理解：** 结合了低分辨率全局概览和高分辨率局部细节，平衡了计算成本与信息精度。
*   **强化学习优化：** 引入 GRPO 算法，使模型在有限的步数内学会更高效的证据搜索策略。

#### 8. 不足与局限
*   **模型依赖：** 实验主要基于 Qwen-2.5-VL 这一单一骨干网络，尚未验证该框架在其他视觉语言模型上的通用性。
*   **场景限制：** 目前仅评估了单文档问答场景，未涉及需要跨多个异构文档聚合信息的复杂任务。
*   **推理延迟：** 相比于单次推理的 RAG，多轮交互智能体在推理延迟和计算成本上相对较高（处于 RAG 和全页输入之间）。
