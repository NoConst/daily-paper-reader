## 摘要
长期对话大语言模型（LLM）智能体需要能够从历史交互中恢复相关证据的记忆系统，同时又不会让无关上下文淹没回答阶段。然而，现有的记忆系统（包括分层系统）通常仍仅依赖向量相似度进行检索。这往往会导致证据集臃肿：添加许多表面相似的对话轮次几乎不会增加召回率，反而会降低检索精度，增加回答阶段的上下文成本，并使检索到的记忆难以检查和管理。为了解决这一问题，我们提出了 HiGMem（分层且由大语言模型引导的记忆系统），这是一种两级“事件-轮次”记忆系统，允许 LLM 将事件摘要作为语义锚点，以预测哪些相关的轮次值得阅读。这使得模型能够先检查高层事件摘要，然后专注于较小的一组潜在有用轮次，通过推理提供简洁可靠的证据集，同时避免了与向量检索相比过高的检索开销。在 LoCoMo10 基准测试中，HiGMem 在五个问题类别中的四个上取得了最佳 F1 分数，并将对抗性 F1 分数从 A-Mem 的 0.54 提高到 0.78，同时检索的轮次减少了一个数量级。代码已在 https://github.com/ZeroLoss-Lab/HiGMem 公开。

## Abstract
Long-term conversational large language model (LLM) agents require memory systems that can recover relevant evidence from historical interactions without overwhelming the answer stage with irrelevant context. However, existing memory systems, including hierarchical ones, still often rely solely on vector similarity for retrieval. It tends to produce bloated evidence sets: adding many superficially similar dialogue turns yields little additional recall, but lowers retrieval precision, increases answer-stage context cost, and makes retrieved memories harder to inspect and manage. To address this, we propose HiGMem (Hierarchical and LLM-Guided Memory System), a two-level event-turn memory system that allows LLMs to use event summaries as semantic anchors to predict which related turns are worth reading. This allows the model to inspect high-level event summaries first and then focus on a smaller set of potentially useful turns, providing a concise and reliable evidence set through reasoning, while avoiding the retrieval overhead that would be excessively high compared to vector retrieval. On the LoCoMo10 benchmark, HiGMem achieves the best F1 on four of five question categories and improves adversarial F1 from 0.54 to 0.78 over A-Mem, while retrieving an order of magnitude fewer turns. Code is publicly available at https://github.com/ZeroLoss-Lab/HiGMem.

### 2. 方法论
HiGMem 采用了**两级事件-轮次（Event-Turn）架构**和 **LLM 引导的检索策略**：
*   **分层记忆架构**：
    *   **轮次层（Turn Layer）**：存储原始对话及元数据（关键词、标签、时间戳、上下文）。
    *   **事件层（Event Layer）**：将相关的对话轮次聚合成“事件”，包含简短摘要和结构化事实表（Fact Sheet）。
*   **记忆构建机制**：
    *   **实时更新**：新对话进入时，LLM 提取元数据生成“轮次节点”。
    *   **事件关联**：通过向量检索找到最相关的事件候选，再由 LLM 决定是加入现有事件（并更新摘要）还是创建新事件。
*   **LLM 引导的检索流程**：
    *   **语义锚点定位**：使用查询关键词同时检索相关的“事件节点”和“轮次节点”。
    *   **推理预测**：LLM 检查检索到的事件摘要，预测该事件下哪些具体的对话轮次对回答问题有用。
    *   **两级过滤**：整合预测的轮次和语义检索的轮次，再次通过 LLM 过滤，形成最终精简的证据集。

### 6. 主要结论与发现
*   **性能卓越**：HiGMem 在 5 类问题中的 4 类取得了最高 F1 分数。尤其在**对抗性问题**上，F1 从 0.54 提升至 0.78。
*   **极度精简**：相比 A-Mem，HiGMem 检索到的对话轮次减少了**一个数量级**（平均 8.09 轮 vs 99.84 轮），但召回率基本持平。
*   **成本效益**：在混合部署模式下（小模型检索，大模型回答），由于输入 Token 数大幅减少，总成本降低了约 **2.7 倍**。
*   **分层有效性**：消融实验证明，事件摘要作为“语义锚点”能显著缩小搜索空间并提高证据精度。

### 7. 优点
*   **高精度检索**：通过 LLM 推理代替纯向量匹配，解决了 RAG 系统常见的“语义漂移”问题。
*   **成本优化**：显著降低了下游昂贵模型处理长上下文的负担。
*   **可解释性**：分层的事件结构使得记忆库更易于人类阅读和管理。
*   **即插即用**：作为一种架构设计，可以适配不同的 LLM 骨干模型。

### 8. 不足与局限
*   **中间开销**：虽然降低了回答阶段的成本，但在记忆构建和检索阶段增加了额外的 LLM 调用次数，导致**首字延迟（TTFT）增加**。
*   **多人对话挑战**：在 DialSim 数据集上的表现略逊于 A-Mem，说明目前的“事件-轮次”结构在处理复杂的多人交互、嘈杂环境时仍有优化空间。
*   **推理依赖**：系统的有效性高度依赖于 LLM 判断事件相关性的能力，对于极小规模或推理能力较弱的模型，效果可能会打折扣。
