## 摘要
大语言模型（LLM）智能体已得到快速普及，并在广泛的应用中展现出卓越的能力。为了提高推理和任务执行能力，现代 LLM 智能体通常会集成记忆模块或检索增强生成（RAG）机制，使其能够进一步利用先前的交互或外部知识。然而，这种设计也引入了一系列严重的隐私漏洞：存储在记忆中的敏感信息可能会通过基于查询的攻击被泄露。尽管此类攻击是可行的，但现有攻击的性能往往有限，攻击成功率（ASR）较低。在本文中，我们提出了 ADAM，这是一种新型隐私攻击，其特点是对受害者智能体记忆进行数据分布估计，并采用熵引导的查询策略来最大化隐私泄露。大量实验表明，我们的攻击显著优于现有最先进的方法，攻击成功率最高可达 100%。因此，这些结果强调了为当前 LLM 智能体开发鲁棒的隐私保护方法的紧迫需求。

## Abstract
Large Language Model (LLM) agents have achieved rapid adoption and demonstrated remarkable capabilities across a wide range of applications. To improve reasoning and task execution, modern LLM agents would incorporate memory modules or retrieval-augmented generation (RAG) mechanisms, enabling them to further leverage prior interactions or external knowledge. However, such a design also introduces a group of critical privacy vulnerabilities: sensitive information stored in memory can be leaked through query-based attacks. Although feasible, existing attacks often achieve only limited performance, with low attack success rates (ASR). In this paper, we propose ADAM, a novel privacy attack that features data distribution estimation of a victim agent's memory and employs an entropy-guided query strategy for maximizing privacy leakage. Extensive experiments demonstrate that our attack substantially outperforms state-of-the-art ones, achieving up to 100% ASRs. These results thus underscore the urgent need for robust privacy-preserving methods for current LLM agents.

#### 1. 核心问题与整体含义（研究动机和背景）
随着大语言模型（LLM）智能体在医疗、金融、法律等领域的广泛应用，为了实现长期的上下文理解和任务连续性，智能体普遍集成了**长期记忆模块**和**检索增强生成（RAG）机制**。
*   **研究动机**：智能体记忆中存储了大量敏感的用户交互记录（如病历、个人偏好）。现有的隐私攻击（如 MEXTRA）多依赖静态、手工编写的提示词，效率低下且容易被 LLM 的对齐机制过滤。
*   **核心问题**：如何系统性地、高效地从黑盒智能体的记忆模块中提取完整的私有数据记录？

#### 2. 方法论（ADAM 框架）
ADAM（**A**daptive querying and **D**istribution estimation to extract private data from **A**gent **M**emory）的核心思想是**通过估计受害者记忆的数据分布，利用主动学习策略生成最具信息量的查询**。

*   **核心流程**：
    1.  **初始化**：从少量种子话题（Anchors）开始，生成初始恶意查询。
    2.  **锚点提取（Anchor Extraction）**：从智能体的响应中提取关键词和主题，通过命名实体识别（NER）和去重构建动态锚点池。
    3.  **分布估计（Distribution Estimation）**：利用聚类算法（如 DBSCAN）估计记忆中的主题分布。通过更新选择概率，优先选择新提取的、未被充分探索的主题，同时惩罚已多次查询的主题。
    4.  **自适应查询生成**：
        *   **加权 K-Center 策略**：在嵌入空间中选择一组既具有代表性又彼此差异化的锚点。
        *   **熵引导选择（Entropy-guided Selection）**：利用辅助 LLM 生成候选查询，并计算其熵值。高熵值代表该查询更有可能触及未知的记忆区域，从而最大化信息增益。
    5.  **迭代优化**：根据观测到的响应不断修正分布估计，循环往复直至达到提取目标或收敛。

#### 6. 主要结论与发现
*   **攻击效能**：ADAM 在所有指标上均显著优于现有方法，在多个场景下实现了 **100% 的 ASR**。
*   **分布估计的重要性**：实验证明，准确估计记忆中的数据分布是提升提取效率的关键。
*   **防御脆弱性**：现有的防御手段（如查询重写、关键词过滤、RA-LLM、Erase-and-check）对 ADAM 的抑制效果有限，因为 ADAM 是在语义层面而非简单的关键词层面进行攻击。
*   **规模效应**：受害者模型越大（推理能力越强），越容易受到 ADAM 的诱导而泄露记忆。

#### 7. 优点（亮点）
*   **自适应性**：不同于静态模板，ADAM 能够根据智能体的反馈动态调整策略。
*   **理论支撑**：将攻击过程建模为 EM（期望最大化）问题，并提供了收敛性分析。
*   **黑盒适用性**：攻击不需要访问模型参数或梯度，仅通过 API 交互即可完成。
*   **高效率**：通过熵引导策略，极大地减少了冗余查询，提高了单位查询下的隐私泄露量。

#### 8. 不足与局限
*   **辅助模型依赖**：攻击过程需要一个性能较强的辅助 LLM 来生成高质量的查询和进行结果精炼。
*   **迭代次数需求**：虽然效率高，但仍需多轮交互，在有严格频率限制（Rate Limiting）的系统中可能会被触发预警（尽管附录中测试表明标准频率限制难以完全阻止攻击）。
*   **语义解析风险**：攻击的成功部分取决于辅助模型对受害者响应的解析能力，如果受害者输出格式极度混乱，可能会影响锚点的提取。
