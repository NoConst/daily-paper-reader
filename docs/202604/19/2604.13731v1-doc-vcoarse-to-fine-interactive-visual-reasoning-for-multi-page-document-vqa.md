## 摘要
多页文档视觉问答（Multi-page Document Visual Question Answering）需要在长且视觉密集的文档中对语义、布局和视觉元素进行推理。现有的无 OCR（OCR-free）方法在容量和精度之间面临权衡：端到端模型在文档长度增加时扩展性较差，而基于视觉检索的流水线则显得脆弱且被动。我们提出了 Doc-$V^*$，这是一个无 OCR 的智能体（agentic）框架，它将多页 DocVQA 视为序列化的证据聚合过程。Doc-$V^*$ 从缩略图概览开始，随后通过语义检索和有针对性的页面获取进行主动导航，并在结构化工作记忆中聚合证据以进行有根据的推理。通过专家轨迹的模仿学习进行训练，并利用群体相对策略优化（GRPO）进一步优化，Doc-$V^*$ 在回答准确性与证据寻求效率之间取得了平衡。在五个基准测试中，Doc-$V^*$ 的表现优于开源基准模型，并接近闭源商业模型，在域外性能上比 RAG 基准提升了高达 47.9%。其他结果表明，有效的证据聚合源于选择性注意力，而非增加输入页数。

## Abstract
Multi-page Document Visual Question Answering requires reasoning over semantics, layouts, and visual elements in long, visually dense documents. Existing OCR-free methods face a trade-off between capacity and precision: end-to-end models scale poorly with document length, while visual retrieval-based pipelines are brittle and passive. We propose Doc-$V^*$, an \textbf{OCR-free agentic} framework that casts multi-page DocVQA as sequential evidence aggregation. Doc-$V^*$ begins with a thumbnail overview, then actively navigates via semantic retrieval and targeted page fetching, and aggregates evidence in a structured working memory for grounded reasoning. Trained by imitation learning from expert trajectories and further optimized with Group Relative Policy Optimization, Doc-$V^*$ balances answer accuracy with evidence-seeking efficiency. Across five benchmarks, Doc-$V^*$ outperforms open-source baselines and approaches proprietary models, improving out-of-domain performance by up to \textbf{47.9\%} over RAG baseline. Other results reveal effective evidence aggregation with selective attention, not increased input pages.

#### 1. 核心问题与整体含义（研究动机和背景）
多页文档视觉问答（DocVQA）要求模型在包含复杂语义、布局和视觉元素（如表格、图表）的长文档中进行推理。现有的无 OCR 方法主要面临两难境地：
*   **端到端模型（End-to-End）：** 难以扩展到超长文档，受限于计算成本、上下文长度限制以及“信息在中部丢失”效应。
*   **检索增强生成（RAG）：** 属于被动处理，容易受到检索错误的影响，且难以处理需要多跳推理的复杂问题。
**研究动机：** 模仿人类阅读行为（主动视觉理论），即先通过缩略图获取全局结构，再根据需求主动寻找、验证并整合证据。

#### 2. 方法论
Doc-V* 是一个将多页 DocVQA 建模为**序列化证据聚合过程**的无 OCR 智能体框架。
*   **核心思想：** 采用“观察-思考-行动”的闭环交互模式，通过主动感知减少不确定性。
*   **关键技术细节：**
    *   **全局缩略图概览（Global Thumbnail Overview）：** 将文档页面调整为低分辨率缩略图并排列成网格，提供全局布局先验，大幅降低初始视觉 Token 消耗（压缩比约 10-12 倍）。
    *   **动作空间（Action Space）：** 包含 `retrieval_page`（语义搜索）、`fetch_page`（按页码精准获取高分辨率图像）和 `answer`（最终回答）。
    *   **结构化视觉推理：** 采用 ReAct 风格，强制模型输出 `<think>`（包含分析、计划、总结）和 `<action>`。
    *   **工作记忆（Working Memory）：** 维护一个持续更新的结构化记忆，存储历史步骤的摘要，防止模型在多轮交互中遗忘或陷入循环。
*   **算法流程：** 初始观察（缩略图）→ 智能体规划 → 迭代执行检索/获取动作 → 环境反馈高分辨率 Token → 证据聚合至工作记忆 → 最终生成答案。

#### 3. 实验设计
*   **数据集：**
    *   **域内（In-Domain）：** MP-DocVQA, DUDE。
    *   **域外（Out-of-Domain）：** SlideVQA（幻灯片）、LongDocURL（网页文档）、MMLongBench-Doc（超长文档，最高 468 页）。
*   **对比方法：**
    *   **开源基线：** 包括端到端模型（如 InternVL3, mPLUG-DocOwl2）、RAG 方法（如 M3DocRAG, VisRAG, URaG）以及其他智能体系统（如 CogDoc）。
    *   **闭源模型：** GPT-4o, Gemini-1.5-Pro, Claude-3.7-Sonnet 等。
*   **评价指标：** ANLS（针对 DUDE/MP-DocVQA）、F1（SlideVQA）、Accuracy（MMLongBench/LongDocURL）。

#### 4. 资源与算力
*   **硬件：** 使用了 **8 台 NVIDIA A100 (80GB) GPU**。
*   **模型架构：** 基于 **Qwen-2.5-VL-7B-Instruct** 初始化。
*   **训练细节：** 采用 BF16 混合精度。分为两个阶段：
    1.  **SFT 阶段：** 使用 GPT-4o 蒸馏的 9,019 条高质量交互轨迹进行监督微调。
    2.  **RL 阶段：** 使用 **GRPO（群体相对策略优化）** 算法，在 2,048 个样本上进行强化学习，优化目标结合了答案准确性、证据召回率和格式规范性。

#### 5. 实验数量与充分性
*   **实验规模：** 在 5 个主流基准测试上进行了全面评估，涵盖了从标准文档到超长文档的多种场景。
*   **消融实验：** 针对认知模块（缩略图、页面分析、记忆）和动作空间（检索 vs 获取）进行了详细的消融研究。
*   **鲁棒性分析：** 测试了不同检索器（ColQwen, BGE, BM25）、不同文档长度、不同迭代步数对性能的影响。
*   **充分性评价：** 实验设计客观、公平，通过与相同参数规模的 RAG 基线对比，清晰地展示了智能体策略的优越性。

#### 6. 主要结论与发现
*   **性能卓越：** Doc-V* 在开源模型中达到 SOTA 水平，甚至在某些指标上超越了 GPT-4o 和 Claude-3.7。
*   **域外泛化强：** 在 OOD 场景下，相比静态 RAG 基线提升了高达 **47.9%**。
*   **证据聚合优于长上下文：** 实验证明，理解长文档的关键在于**选择性注意力和有效的证据聚合**，而非单纯增加输入的页面数量。
*   **主动补偿机制：** 即使初始检索器较弱，Doc-V* 也能通过主动的 `fetch_page` 动作纠正检索错误。

#### 7. 优点
*   **主动感知：** 成功将人类的“由粗到精”阅读策略引入多模态大模型。
*   **效率平衡：** 通过缩略图和按需获取高分辨率图像，在计算成本和感知精度之间找到了极佳的平衡点。
*   **无 OCR 依赖：** 避免了 OCR 识别错误导致的级联失效，保留了原始视觉和布局信息。
*   **训练策略先进：** 结合了高质量轨迹蒸馏与 GRPO 强化学习，提升了智能体的决策效率。

#### 8. 不足与局限
*   **模型底座单一：** 实验主要基于 Qwen2.5-VL，尚未在其他视觉语言模型底座上验证普适性。
*   **单文档限制：** 目前仅针对单文档问答进行了优化，尚未扩展到需要跨多个异构文档聚合信息的场景。
*   **推理延迟：** 相比于单次推理的 RAG，多轮交互智能体在推理延迟（Latency）上相对较高。
*   **幻觉风险：** 尽管有工作记忆，但在极端复杂的文档中仍可能存在证据误判或幻觉现象。
