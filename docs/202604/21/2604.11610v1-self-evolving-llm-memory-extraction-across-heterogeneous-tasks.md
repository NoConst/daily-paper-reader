## 摘要
随着基于大语言模型（LLM）的助手变得持久化和个性化，它们必须从过去的对话中提取并保留有用信息作为记忆。然而，值得记住的信息类型在不同任务之间存在显著差异。我们形式化了“异构记忆提取”任务，并引入了 BEHEMOTH，这是一个重新利用了涵盖个性化、问题解决和智能体任务的 18 个现有数据集的基准，并使用下游效用驱动的指标进行系统评估。我们的实证分析证实，没有单一的静态提取提示词能在所有任务类别中占据主导地位，且原本为同构分布设计的现有自进化提示词优化框架在训练任务具有异构性时性能会下降。为解决这一问题，我们提出了 CluE，这是一种基于聚类的自进化策略，它根据提取场景将训练样本分组为簇，独立分析每个簇，并综合跨簇的见解来更新提取提示词。在 BEHEMOTH 上的实验表明，CluE 在异构任务中具有有效的泛化能力（相对增益达 +9.04%），始终优于先前的自进化框架。

## Abstract
As LLM-based assistants become persistent and personalized, they must extract and retain useful information from past conversations as memory. However, the types of information worth remembering vary considerably across tasks. We formalize the \textit{heterogeneous memory extraction} task and introduce \textbf{BEHEMOTH}, a benchmark that repurposes 18 existing datasets spanning personalization, problem-solving, and agentic tasks, using a downstream utility-driven metric for systematic evaluation. Our empirical analysis confirms that no single static extraction prompt dominates across all task categories, and that existing self-evolving prompt optimization frameworks, originally designed for homogeneous distributions, degrade when training tasks are heterogeneous. To address this, we propose \textbf{CluE}, a cluster-based self-evolving strategy that groups training examples into clusters by extraction scenarios, analyzes each cluster independently, and synthesizes cross-cluster insights to update the extraction prompt. Experiments on BEHEMOTH show that CluE generalizes effectively across heterogeneous tasks ($+$9.04\% relative gain), consistently outperforming prior self-evolving frameworks.

### 1. 核心问题与整体含义
*   **研究动机**：随着 LLM 助手向个性化和持久化发展，它们需要从对话中提取有用信息存入记忆。然而，不同任务对“有用信息”的定义截然不同（例如：社交对话关注用户偏好，而编程任务关注解决策略）。
*   **核心挑战**：现有的记忆提取系统多依赖于静态、针对特定领域的规则，难以应对现实世界中多样化的交互场景。此外，现有的提示词（Prompt）自进化框架通常假设任务分布是同构的，在面对异构任务时容易出现“反馈稀释”或“过度拟合近期任务”的问题。
*   **整体含义**：论文旨在建立一个评估异构记忆提取的统一基准，并开发一种能够跨越不同任务类型、稳定进化的记忆提取策略。

### 2. 方法论：CluE (Cluster-based Evolution)
CluE 是一种基于聚类的自进化策略，旨在通过将异构任务分解为相似的场景来优化提取提示词。其核心流程包括：
*   **场景总结（Summarization）**：针对每个训练样本，总结其“提取场景”（如：提取长对话中的程序性知识），而非关注具体的业务领域。
*   **动态聚类（Clustering）**：根据提取场景的相似性将样本分组。聚类管理器可以根据数据分布的变化动态创建、合并或拆分簇。
*   **簇内独立分析（Cluster-based Analysis）**：对每个簇内的成功和失败案例进行独立分析，识别该场景下的特定提取模式和改进建议。
*   **跨簇见解合成（Cross-cluster Proposal）**：将所有簇的分析结果汇总，由一个“提议者”模型识别通用原则，并组织成结构化的记忆分类体系（Taxonomy），生成最终的提取提示词。

### 6. 主要结论与发现
*   **静态提示词的局限性**：没有一个静态提示词能在所有异构任务中表现最优，领域特定的提示词在非目标领域表现较差。
*   **现有进化框架的失效**：在异构分布下，GEPA 和 MemEvolve 等框架往往在提升某一类任务性能时会损害另一类任务。
*   **CluE 的优越性**：CluE 实现了 **+9.04%** 的整体相对增益，且在所有任务类别中均表现出正向提升。
*   **聚类的价值**：通过聚类分析，模型能够构建出更清晰的记忆分类法，从而在处理复杂任务时提供更精准的指导。

### 7. 优点
*   **首创性**：首次形式化了异构记忆提取任务，并提出了 BEHEMOTH 这一综合性基准。
*   **稳定性**：CluE 通过聚类机制解决了自进化过程中反馈信号相互抵消的问题，使学习过程更加稳定。
*   **结构化输出**：进化出的提示词具有良好的结构（分类法+通用准则），可解释性强。

### 8. 不足与局限
*   **基准来源**：BEHEMOTH 是由现有数据集重新组合而成，虽然覆盖广，但可能无法完全模拟真实世界中极其复杂、长程的对话动态。
*   **单步评估限制**：主要评估集中在单步提取上，虽然补充了持续记忆实验，但对于记忆的长期管理（如遗忘机制、冲突解决）探讨较少。
*   **计算开销**：虽然 CluE 的时间效率与 MemEvolve 相当，但相比静态提示词，其进化过程仍需要多次 LLM 调用，存在一定的优化成本。
