## 摘要
长期对话智能体需要能够捕捉事件间关系（而非仅仅是孤立事实）的记忆系统，以支持时间推理和多跳问答。目前的方法面临着一个基本的权衡：扁平化记忆虽然高效，但无法建模关系结构；而基于图的记忆虽然能实现结构化推理，但其构建过程昂贵且脆弱。为了解决这些问题，我们提出了 StructMem，这是一种结构增强的分层记忆框架，它保留了事件级的绑定并诱导了跨事件的连接。通过在时间上锚定双重视角并执行周期性的语义巩固，StructMem 提升了在 LoCoMo 上的时间推理和多跳性能，同时与之前的记忆系统相比，显著减少了 Token 使用量、API 调用次数和运行时间。代码见 https://github.com/zjunlp/LightMem 。

## Abstract
Long-term conversational agents need memory systems that capture relationships between events, not merely isolated facts, to support temporal reasoning and multi-hop question answering. Current approaches face a fundamental trade-off: flat memory is efficient but fails to model relational structure, while graph-based memory enables structured reasoning at the cost of expensive and fragile construction. To address these issues, we propose \textbf{StructMem}, a structure-enriched hierarchical memory framework that preserves event-level bindings and induces cross-event connections. By temporally anchoring dual perspectives and performing periodic semantic consolidation, StructMem improves temporal reasoning and multi-hop performance on \texttt{LoCoMo}, while substantially reducing token usage, API calls, and runtime compared to prior memory systems, see https://github.com/zjunlp/LightMem .

#### 1. 核心问题与整体含义（研究动机和背景）
长期对话智能体（Long-term agents）需要记忆系统来维持连贯性。现有的记忆系统主要分为两类：
*   **扁平记忆（Flat Memory）：** 将事实或摘要存储为独立单元（如向量数据库），效率高但缺乏事件间的关联，导致长历史检索退化为浅层的相似性匹配。
*   **图记忆（Graph Memory）：** 通过实体-关系抽取构建结构，虽支持复杂推理，但构建成本极高、推理链条脆弱且易受噪声影响。
**研究动机：** 论文认为记忆的基本单位不应是孤立的事实或三元组，而应是**具有时间锚点的关系事件**。StructMem 旨在通过分层结构，在不引入复杂图模式的情况下，捕捉事件间的因果、时间和人际关系。

#### 2. 方法论：核心思想与关键技术
StructMem 采用分层设计，分为两个核心阶段：
*   **事件级绑定（Event-Level Binding）：**
    *   **双视角抽取（Dual-Perspective Extraction）：** 对每条对话同时抽取“事实条目”（内容）和“关系条目”（人际动态、因果影响）。
    *   **时间锚定（Temporal Anchoring）：** 所有条目都绑定到原始时间戳，形成一个完整的事件单元，避免了实体消解的开销。
*   **跨事件整合（Cross-Event Consolidation）：**
    *   **语义事件连接：** 当累积事件超过时间阈值时触发。系统利用当前缓存事件作为查询，检索语义相关的历史事件。
    *   **合成式巩固（Synthesis）：** LLM 将当前事件与检索到的历史事件进行综合，生成高层级的关系假设（Summary），从而在保留原始细节的同时诱导跨时间的因果链条。

#### 7. 优点
*   **分层架构：** 巧妙地结合了事件内的细节保留和跨事件的语义合成。
*   **低成本结构化：** 避免了硬性的图模式（Schema）设计和复杂的实体对齐，利用自然语言描述关系，更具灵活性。
*   **时间感知：** 强化了时间锚定，有效解决了长程对话中的时间推理难题。

#### 8. 不足与局限
*   **提示词依赖：** 双视角抽取的质量高度依赖于 Prompt 设计，设计不当可能导致关系捕捉不全。
*   **缺乏更新机制：** 目前系统主要关注记忆的扩张和合成，缺乏显式的冲突解决和记忆更新（Update/Forget）机制，当用户偏好发生变化时可能存在一致性风险。
*   **实时性挑战：** 虽然比图记忆快，但周期性的整合步骤在极大规模实时交互中仍可能带来瞬时延迟。
