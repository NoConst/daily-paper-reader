## 摘要
大语言模型（LLM）路由在整合不同模型优势的同时平衡效率与性能方面取得了显著成果。然而，为了支持更真实且具挑战性的应用，路由必须扩展到代理型 LLM 场景，其中任务规划、异构智能体间的多轮协作以及记忆利用是不可或缺的。为了填补这一空白，我们提出了 GraphPlanner，这是一种面向多智能体 LLM 的异构图记忆增强型代理路由器，它为每个查询生成路由工作流，并支持归纳式和转导式推理。GraphPlanner 将工作流生成建模为马尔可夫决策过程（MDP），在每一步中同时选择 LLM 主干模型和智能体角色，包括规划者（Planner）、执行者（Executor）和总结者（Summarizer）。通过利用名为 GARNet 的异构图来捕获查询、智能体和响应之间的交互记忆，GraphPlanner 将历史记忆和工作流记忆整合到更丰富的状态表示中。整个流水线通过强化学习进行优化，共同提升了特定任务的性能和计算效率。我们在 14 个不同的 LLM 任务上对 GraphPlanner 进行了评估，结果表明：(1) GraphPlanner 优于强大的单轮和多轮路由器，在将 GPU 成本从 186.26 GiB 降低到 1.04 GiB 的同时，准确率提升高达 9.3%；(2) GraphPlanner 对未见过的任务和 LLM 具有鲁棒的泛化能力，展现出强大的零样本能力；(3) GraphPlanner 有效地利用了历史记忆，支持归纳式和转导式推理，从而实现更具适应性的路由。我们的代码已发布在 https://github.com/ulab-uiuc/GraphPlanner。

## Abstract
LLM routing has achieved promising results in integrating the strengths of diverse models while balancing efficiency and performance. However, to support more realistic and challenging applications, routing must extend into agentic LLM settings, where task planning, multi-round cooperation among heterogeneous agents, and memory utilization are indispensable. To address this gap, we propose GraphPlanner, a heterogeneous graph memory-augmented agentic router for multi-agent LLMs that generates routing workflows for each query and supports both inductive and transductive inference. GraphPlanner formulates workflow generation as a Markov Decision Process (MDP), where at each step it selects both the LLM backbone and the agent role, including Planner, Executor, and Summarizer. By leveraging a heterogeneous graph, denoted as GARNet, to capture interaction memories among queries, agents, and responses, GraphPlanner integrates historical memory and workflow memory into richer state representations. The entire pipeline is optimized with reinforcement learning, jointly improving task-specific performance and computational efficiency. We evaluate GraphPlanner across 14 diverse LLM tasks and demonstrate that: (1) GraphPlanner outperforms strong single-round and multi-round routers, improving accuracy by up to 9.3% while reducing GPU cost from 186.26 GiB to 1.04 GiB; (2) GraphPlanner generalizes robustly to unseen tasks and LLMs, exhibiting strong zero-shot capabilities; and (3) GraphPlanner effectively leverages historical memories, supporting both inductive and transductive inference for more adaptive routing. Our code for GraphPlanner is released at https://github.com/ulab-uiuc/GraphPlanner.

#### 1. 核心问题与整体含义（研究动机和背景）
随着大语言模型（LLM）能力的提升，如何高效整合多个异构模型（在能力、成本、可靠性上各异）成为关键。现有的路由方法主要分为**单轮路由**（一次性分配模型）和**多轮路由**（顺序调用模型），但它们在处理复杂任务时存在局限：
*   **缺乏代理能力**：无法进行任务分解和多角色协作（如规划、执行、总结）。
*   **记忆利用不足**：难以系统性地利用历史交互轨迹来优化当前的决策。
*   **协作效率低**：独立调用模型往往导致上下文冲突或冗余计算。
**核心问题**：如何将路由扩展到“代理化（Agentic）”设置中，使路由器能根据查询动态生成包含不同角色和模型的工作流，并有效利用长效记忆。

#### 2. 方法论：核心思想与技术细节
GraphPlanner 将代理路由建模为**马尔可夫决策过程（MDP）**，通过动态生成图结构的工作流来解决查询。
*   **核心思想**：路由器不仅决定使用哪个 LLM，还决定该 LLM 承担什么角色（Planner, Executor, Summarizer）。
*   **关键技术：GARNet（异构图神经网络）**：
    *   **双图结构**：整合了 `Gworkflow`（当前任务的实时工作流记忆）和 `Ghistory`（跨查询的历史交互记忆）。
    *   **角色中心节点（Role Hub Nodes）**：作为桥梁连接历史和当前图，聚合特定“模型-角色”对的性能与成本特征。
    *   **嵌套编码**：先编码历史图提取经验，再将其注入当前工作流图进行决策。
*   **算法流程**：
    1.  **状态表示**：基于当前子查询和 GARNet 提取的图特征。
    2.  **动作选择**：从 $|A| = 3 \times K$（3个角色 $\times$ K个模型）的空间中选择动作，并应用动态掩码确保逻辑合理（如第一步不能是总结）。
    3.  **强化学习优化**：使用 **PPO（近端策略优化）** 算法，奖励函数 $r_t$ 平衡了任务准确率（Utility）和计算成本（Cost），公式为 $r_t = U - \alpha C$。

*   **实验规模**：论文进行了大规模实验，包括 14 个任务的性能对比、帕累托前沿分析、针对未见任务和未见模型的泛化性测试。

#### 6. 主要结论与发现
*   **性能卓越**：在 Phase 2 中，准确率比最强基线提升了 **9.3%**，在数学和代码等复杂推理任务上优势尤为明显。
*   **成本极低**：在提升性能的同时，大幅降低了训练所需的算力和显存。
*   **强泛化性**：在完全未见的任务上，准确率比之前最好的路由器高出 20-40%；能直接适应未参与训练的新 LLM。
*   **记忆价值**：异构图 GARNet 能有效提取历史交互中的成功模式，转导式推理（利用历史）比归纳式推理性能更强。

#### 7. 优点
*   **轻量化设计**：相比于基于大模型进行路由决策的方法，GraphPlanner 采用轻量级 GNN，极大地降低了部署门槛。
*   **动态工作流生成**：打破了固定路由的限制，能够根据查询复杂度自动决定是“直接回答”还是“分解-执行-总结”。
*   **异构记忆建模**：通过图结构巧妙地解决了多轮对话中上下文冲突和长效记忆利用的问题。

#### 8. 不足与局限
*   **角色定义依赖**：虽然支持扩展，但核心实验仍基于预定义的三个角色，对于更细粒度的专家角色划分（如特定领域的专家）探讨较少。
*   **奖励函数敏感性**：超参数 $\alpha$（成本系数）需要根据具体预算手动调整，缺乏一种完全自动化的成本-性能平衡机制。
*   **冷启动问题**：虽然有零样本能力，但在完全没有历史交互数据的情况下，其转导式推理的优势无法发挥。
