## 摘要
视觉文档理解通常需要独立的检索和生成模型，这导致内存占用和系统复杂度翻倍。我们提出了 Hydra，这是一种双头方法，通过单一视觉语言模型（VLM）同时提供 ColBERT 风格的后期交互（late-interaction）检索和自回归生成。一个仅针对检索训练的单一 LoRA 适配器在推理时进行切换：启用它可产生多向量嵌入；禁用它则可恢复基础模型的生成质量——在 10,500 个贪婪和随机采样样本中，输出与基础模型 100% 字节一致；在四个 VQA 基准测试的 15,301 个样本中（其中三个具有信息量；在贪婪解码下，两个模型的 ChartQA 结果均接近于零），与独立的基础模型流水线相比，最大 delta-ANLS 仅为 0.0044。我们确定了三个工程要求（注意力模式恢复、lm_head 保留、KV 缓存感知解码），如果忽略这些要求，尽管权重恢复正确，也会在无声中破坏生成性能。在 ViDoRe V1 上，Hydra (4B) 在单次训练运行中与受控的单头基准模型差距在 1 个百分点以内，在 V2 和 V3 上的综合得分更高，且集中在部分任务上；需要多种子实验来确认这些趋势。这种单模型设计将 GPU 显存峰值降低了 41%，尽管在并发服务负载下，适配器切换会引入吞吐量开销。消融实验表明，在基于 LoRA (r=16) 的训练方案中，GritLM 风格的联合训练没有带来收益。对 Qwen2.5-Omni-3B 的概念验证扩展表明，该机制可推广到音频检索和视频嵌入，并支持语音生成。

## Abstract
Visual document understanding typically requires separate retrieval and generation models, doubling memory and system complexity. We present Hydra, a dual-head approach that provides both ColBERT-style late-interaction retrieval and autoregressive generation from a single vision-language model (VLM). A single LoRA adapter, trained only for retrieval, is toggled at inference: enabling it produces multi-vector embeddings; disabling it recovers the base model's generation quality -- byte-identical outputs in 100% of 10,500 greedy and stochastic samples, with max delta-ANLS = 0.0044 across 15,301 samples on four VQA benchmarks (three informative; ChartQA is near-zero for both models under greedy decoding) when compared against an independent base-model pipeline. We identify three engineering requirements (attention-mode restoration, lm_head preservation, KV-cache-aware decoding) whose omission silently breaks generation despite correct weight recovery. On ViDoRe V1, Hydra (4B) is within 1 percentage point of a controlled single-head baseline in a single training run, with higher aggregate scores on V2 and V3 that are concentrated on a subset of tasks; multi-seed experiments are needed to confirm these trends. The single-model design reduces peak GPU memory by 41%, though adapter switching introduces throughput overhead under concurrent serving loads. An ablation shows that GritLM-style joint training provides no benefit within the LoRA-based (r=16) training regime. A proof-of-concept extension to Qwen2.5-Omni-3B demonstrates that the mechanism generalizes to audio retrieval and video embedding, with speech generation.

现代视觉文档 AI 系统通常采用“双模型”架构：一个检索模型（如 ColPali）用于从海量页面中寻找相关信息，另一个生成式视觉语言模型（VLM，如 Qwen2-VL）用于理解并回答问题。这种做法导致 GPU 内存占用翻倍、系统部署复杂。
**Hydra 的核心动机**是消除这种冗余。作者观察到，检索模型通常是基于 VLM 微调而来的，通过使用 **LoRA（低秩自适应）** 技术，可以在推理时动态开启或关闭适配器，从而让同一个模型在“检索模式”和“生成模式”之间无缝切换，实现内存的大幅节省。

#### 2. 论文提出的方法论
Hydra 采用了一种**双头（Dual-head）架构**，基于单一 VLM（如 Qwen3.5-4B）：
*   **核心思想**：利用 LoRA 的加性特性（$W_{adapted} = W_{base} + BA$）。开启 LoRA 时，模型变为检索器；关闭 LoRA 时，模型完全恢复为原始的生成式 VLM。
*   **检索模式（LoRA-on）**：启用 LoRA 适配器，将注意力机制改为**双向（Bidirectional）**，通过一个线性投影层（`custom_text_proj`）输出 320 维的多向量嵌入，利用 ColBERT 风格的后期交互进行评分。
*   **生成模式（LoRA-off）**：禁用 LoRA 适配器，恢复**因果（Causal）**注意力，使用原始的 `lm_head` 进行自回归文本生成。
*   **关键工程要求**：
    1.  **注意力模式恢复**：必须在切换模式时严格恢复因果掩码，否则自回归生成会失效。
    2.  **lm_head 保留**：防止训练过程中的梯度污染（如权重共享或 DDP 同步导致的数值漂移）破坏原始生成头。
    3.  **KV 缓存感知解码**：实现支持 KV 缓存的生成流程，相比朴素的前向传播提速约 38 倍。

#### 6. 论文的主要结论与发现
*   **性能无损切换**：通过 LoRA 切换，Hydra 在检索性能上与专门的检索模型相当（差距 < 1%），同时在生成任务上与原始基础模型完全一致（ANLS 差异极小）。

#### 7. 优点
*   **极高的资源效率**：显著降低了 RAG 系统的显存门槛，适合端侧或资源受限的部署环境。
*   **工程落地性强**：识别并解决了注意力模式恢复、权重污染等隐蔽的工程坑，为工业界提供了清晰的实现路径。
*   **架构优雅**：无需复杂的模型合并或多适配器管理，仅需一个检索适配器即可解锁双重功能。

#### 8. 不足与局限
*   **模型家族局限**：主要在 Qwen 系列模型上验证，尚未在 InternVL 或 LLaVA 等其他架构上广泛测试。

*   **LoRA 秩的影响**：实验仅限于 r=16，更高秩的情况下联合训练是否会有不同表现尚不明确。
