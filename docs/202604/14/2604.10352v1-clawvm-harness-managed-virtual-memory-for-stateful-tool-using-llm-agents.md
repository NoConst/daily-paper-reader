## 摘要
有状态的工具调用 LLM 智能体将上下文窗口视为工作内存，然而目前的智能体框架对驻留性（residency）与持久性（durability）的管理仅是尽力而为，导致了反复出现的故障：压缩后的状态丢失、重置时跳过刷新以及破坏性写回。我们提出了 ClawVM，这是一个虚拟内存层，它将状态管理为具有最小保真度不变性的有类型页，在 Token 预算下提供多分辨率表示，并在每个生命周期边界进行经过验证的写回。由于框架已经负责组装提示词、协调工具并观察生命周期事件，因此它是天然的执行点；将契约置于此处可使驻留性与持久性变得具有确定性且可审计。在合成工作负载、12 个真实会话追踪以及对抗性压力测试中，经离线真值（oracle）确认，只要最小保真度集合符合 Token 预算，ClawVM 就能消除所有策略可控的故障，且每轮增加的策略引擎开销中位数小于 50 微秒。

## Abstract
Stateful tool-using LLM agents treat the context window as working memory, yet today's agent harnesses manage residency and durability as best-effort, causing recurring failures: lost state after compaction, bypassed flushes on reset, and destructive writeback. We present \textsc{ClawVM}, a virtual memory layer that manages state as typed pages with minimum-fidelity invariants, multi-resolution representations under a token budget, and validated writeback at every lifecycle boundary. Because the harness already assembles prompts, mediates tools, and observes lifecycle events, it is the natural enforcement point; placing the contract there makes residency and durability deterministic and auditable. Across synthetic workloads, 12 real-session traces, and adversarial stress tests, \textsc{ClawVM} eliminates all policy-controllable faults whenever the minimum-fidelity set fits within the token budget, confirmed by an offline oracle, and adds median <50 microseconds of policy-engine overhead per turn.

### 2. 方法论：ClawVM 架构
ClawVM 是一个集成在智能体框架中的虚拟内存层，其核心思想包括：
*   **类型化页面（Typed Pages）**：将智能体状态划分为不同类型的“页”（如引导策略、约束、计划、偏好、证据、对话片段），每种页面都有其“最小保真度不变性”（Minimum-fidelity Invariants）。
*   **多分辨率表示（Multi-resolution Representations）**：页面在 Token 预算压力下可以按四个等级降级：**完整（Full）** $\rightarrow$ **压缩（Compressed）** $\rightarrow$ **结构化（Structured）** $\rightarrow$ **指针（Pointer）**。这避免了全有或全无的丢弃。
*   **确定性选择策略**：
    *   **阶段 1**：安装所有硬固定（Hard-pinned）页面和满足最小保真度的表示。
    *   **阶段 2**：根据边际效用（考虑重计算成本、近期性等）贪婪地升级页面保真度，直到耗尽 Token 预算。
*   **验证写回协议（Validated Writeback）**：采用三阶段事务处理（结构化暂存、确定性验证、作用域提交），确保在生命周期边界（如重置、压缩）时，脏数据被非破坏性地持久化。
*   **可观测故障模型**：定义了如“重新获取故障”（Refetch Fault）、“重复工具故障”等指标，使内存管理决策可审计、可回放。

### 6. 主要结论与发现
*   **消除故障**：在 Token 预算允许的情况下，ClawVM 消除了所有“策略可控故障”（如引导丢失、刷新遗漏），将故障数从基准方法的平均 67.8 降至 **0**。

### 7. 优点
*   **系统性严谨**：将成熟的 OS 虚拟内存概念引入 LLM 智能体领域，解决了长程会话中状态管理的混乱。
*   **多分辨率设计**：通过“降级而非丢弃”的思路，极大提高了 Token 利用率和系统弹性。
*   **确定性与可审计**：通过故障模型和写回日志，使智能体的“失忆”问题变得可诊断、可复现。
*   **极低开销**：微秒级的处理延迟使其非常适合生产环境。

### 8. 不足与局限
*   **语义正确性**：ClawVM 只能保证状态“被保存”且“格式正确”，但无法验证 LLM 生成的更新在语义上是否真实（即 LLM 可能会写回错误的事实）。
*   **回放依赖**：实验中的工具调用是通过签名模拟的，在处理具有非确定性输出的真实外部服务时，可能需要更复杂的录制/回放机制。
*   **外部有效性**：虽然验证了 Hook 的正确性，但大规模的多会话在线部署效果仍有待进一步验证。
