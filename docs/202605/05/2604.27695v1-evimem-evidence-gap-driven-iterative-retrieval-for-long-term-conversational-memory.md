## 摘要
长期对话记忆需要检索散布在多个会话中的证据，然而单次检索在处理时间相关和多跳问题时往往会失败。现有的迭代方法通过生成内容或文档级信号来优化查询，但没有一种方法能明确诊断证据缺口（即累积检索集中缺失的内容），导致查询优化缺乏针对性。我们提出了 EviMem，它结合了 IRIS（通过不足信号进行的迭代检索）和 LaceMem（对话证据记忆的分层架构）。IRIS 是一个闭环框架，通过充分性评估检测证据缺口，诊断缺失内容并驱动有针对性的查询优化；LaceMem 是一种从粗到细的记忆层级结构，支持细粒度的缺口诊断。在 LoCoMo 数据集上，EviMem 在时间相关问题（从 73.3% 提升至 81.6%）和多跳问题（从 65.9% 提升至 85.2%）上的评判准确率均优于 MIRIX，且延迟降低了 4.5 倍。代码：https://github.com/AIGeeksGroup/EviMem。

## Abstract
Long-term conversational memory requires retrieving evidence scattered across multiple sessions, yet single-pass retrieval fails on temporal and multi-hop questions. Existing iterative methods refine queries via generated content or document-level signals, but none explicitly diagnoses the evidence gap, namely what is missing from the accumulated retrieval set, leaving query refinement untargeted. We present EviMem, combining IRIS (Iterative Retrieval via Insufficiency Signals), a closed-loop framework that detects evidence gaps through sufficiency evaluation, diagnoses what is missing, and drives targeted query refinement, with LaceMem (Layered Architecture for Conversational Evidence Memory), a coarse-to-fine memory hierarchy supporting fine-grained gap diagnosis. On LoCoMo, EviMem improves Judge Accuracy over MIRIX on temporal (73.3% to 81.6%) and multi-hop (65.9% to 85.2%) questions at 4.5x lower latency. Code: https://github.com/AIGeeksGroup/EviMem.

#### 1. 核心问题与整体含义（研究动机和背景）
在处理跨越多个会话的长期对话时，LLM 智能体需要从海量历史中提取证据。现有的检索增强生成（RAG）系统面临两大挑战：
*   **单次检索失效**：对于需要时间推理（如“某人何时改变了计划”）或多跳合成（跨会话关联事实）的问题，单次检索难以覆盖所有散布的证据。
*   **迭代优化盲目**：现有的迭代检索方法（如 FLARE、Self-RAG）虽然引入了反馈，但通常基于生成内容的置信度或单个文档的质量，**缺乏对“检索集合整体是否充分”的显式诊断**。这导致后续的查询优化（Query Refinement）缺乏针对性，容易产生语义漂移。

#### 2. 方法论
EviMem 由两个相互协作的核心组件构成：

*   **LaceMem（分层记忆架构）**：
    *   **索引层 (Index Layer)**：将对话转换为原子语义元组 `(主语, 谓语, 宾语, 时间)`，支持精确搜索。
    *   **边缘层 (Edge Layer)**：通过同源连接和语义相似度构建稀疏图，支持跨会话的多跳扩展。
    *   **原始层 (Raw Layer)**：保留原始对话文本，确保生成的答案有据可查。
*   **IRIS（基于不足信号的迭代检索）**：
    *   **双路径检索**：同时运行“锚点路径”（原始问题）和“优化路径”（诊断驱动的查询），平衡覆盖面与针对性。
    *   **充分性评估（核心思想）**：LLM 将累积的证据集整体分类为三个等级：**EXACT**（直接证据）、**INFERRABLE**（可推断证据）、**PARTIAL**（部分证据）。
    *   **缺口诊断与优化**：如果证据不足，LLM 会生成自然语言描述的“证据缺口”（诊断缺失了什么），并据此生成下一次迭代的精准查询语句。
    *   **实体追踪**：监控特定实体的证据覆盖率，防止因某个关键实体信息缺失导致的推理失败。

*   **性能领先**：EviMem 在时序推理（81.6%）和多跳问题（85.2%）上的准确率显著超过 MIRIX。

#### 7. 优点
*   **闭环反馈**：将检索从“开环”变为“闭环”，通过诊断缺失信息来驱动检索，逻辑清晰且符合人类认知。
*   **分层架构设计**：LaceMem 巧妙地平衡了结构化数据的检索效率与原始文本的生成质量。
*   **时序敏感**：专门针对对话中的时间属性进行了归一化和阈值优化，解决了 RAG 系统常见的时序混乱问题。
*   **拒绝幻觉**：具备“未提及”检测机制，在证据确实不足时选择拒绝回答，降低了幻觉风险。

#### 8. 不足与局限
*   **离线构建限制**：目前的 LaceMem 记忆层是基于对话快照离线构建的，尚未在论文中展示实时流式更新（Streaming Update）的性能表现。
*   **API 依赖**：系统性能高度依赖于底层 LLM 的评估能力，如果使用较弱的开源模型，充分性诊断的准确性可能会下降。
*   **成本考量**：虽然延迟降低了，但多次迭代和双路径检索意味着更多的 Token 消耗，在极大规模应用中需权衡成本。
