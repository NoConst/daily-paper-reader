## 摘要
随着基于大语言模型（LLM）的智能体越来越多地被部署用于执行复杂的现实任务，它们面临着一类日益增多的攻击，这些攻击利用用户-智能体-环境之间延长的交互来追求在单轮对话设置中难以实现的恶意目标。这种长程威胁对关键领域中 LLM 智能体的安全部署构成了重大风险。在本文中，我们提出了 MAGE（Memory As Guardrail Enforcement），这是一个旨在应对各种长程威胁的新型防御框架。受系统安全中“影子栈”抽象概念的启发，MAGE 维护了一个专门的、以安全为中心的智能体内存，用于提取并保留智能体整个执行轨迹中的安全关键上下文，并利用该影子内存在待执行操作执行前主动评估其风险。广泛的评估表明，MAGE 在多种长程威胁的检测准确率上显著优于现有防御措施，实现了对大多数攻击的早期检测，且对智能体效用的开销微乎其微。据我们所知，MAGE 是首个利用智能体内存方法检测和缓解长程威胁的框架，为这一关键挑战建立了新范式，并为未来的研究开辟了充满前景的方向。

## Abstract
As large language model (LLM)-powered agents are increasingly deployed to perform complex, real-world tasks, they face a growing class of attacks that exploit extended user-agent-environment interactions to pursue malicious objectives improbable in single-turn settings. Such long-horizon threats pose significant risks to the safe deployment of LLM agents in critical domains. In this paper, we present MAGE (Memory As Guardrail Enforcement), a novel defensive framework designed to counter a wide range of long-horizon threats. Inspired by the "shadow stack" abstraction in systems security, MAGE maintains a dedicated, safety-focused agentic memory that distills and retains safety-critical context across the agent's full execution trajectory, leveraging this shadow memory to proactively assess the risk of pending actions prior to their execution. Extensive evaluation demonstrates that MAGE substantially outperforms existing defenses across diverse long-horizon threats in detection accuracy, achieves early-stage detection for the majority of attacks, and introduces only negligible overhead to agent utility. To our best knowledge, MAGE represents the first framework to detect and mitigate long-horizon threats using an agentic memory approach, establishing a new paradigm for this critical challenge and opening promising directions for future research.

#### 1. 核心问题与整体含义
*   **研究背景**：随着大语言模型（LLM）智能体被部署于多轮、复杂的现实任务中，它们面临着一种新型安全挑战——**长程威胁（Long-Horizon Threats）**。
*   **核心问题**：传统的防御手段（如关键词过滤、单轮分类器）主要针对短上下文设计。在长程交互中，攻击者可以通过多轮诱导、工具链组合或持久化提示注入，使恶意意图在单轮动作中显得“无害”，但在整个执行轨迹（Trajectory）层面产生破坏性后果。
*   **研究动机**：智能体往往缺乏对执行轨迹的回溯性安全检查能力，且其安全对齐在长上下文中容易被稀释。因此，需要一种能够跨轮次提炼并保留安全关键信息的机制。

#### 2. 方法论
*   **核心思想**：受系统安全中“影子栈”（Shadow Stack）概念启发，提出**影子内存（Shadow Memory）**机制。它在智能体工作内存之外维护一个独立的、专门存储安全相关上下文的存储层。
*   **关键组件**：
    *   **内存管理器 ($M$)**：负责在每一轮交互后，将最新的动作、观察和指令提炼为紧凑的安全状态 $m_t$。
    *   **裁判 ($J$)**：在智能体执行下一个动作前，根据影子内存中的上下文评估该动作的风险，决定“批准”或“拒绝”，并给出理由。
*   **算法流程**：
    1.  智能体提出待执行动作 $a_t$。
    2.  内存管理器更新影子内存，识别潜在的风险信号（如敏感文件访问、外部联系人关联）。
    3.  裁判基于影子内存进行风险评估。若拒绝，则拦截动作并向用户报告理由；若批准，则执行动作并进入下一轮。
*   **模型优化**：使用**分轮强化学习（Turn-wise RL）**，基于 GRPO 算法联合优化 $M$ 和 $J$。奖励函数设计包括：裁判准确性奖励、内存的未来影响折现奖励、格式惩罚以及内存长度惩罚（鼓励简洁）。

#### 3. 实验设计
*   **实验场景与数据集**：
    *   **用户作为对手（User-as-Adversary）**：使用 **SHADE-Arena** 基准，针对“顺序工具攻击链”（STAC）进行测试。

*   **评估指标**：攻击成功率（ASR）、良性效用（BU，即正常任务完成率）、Token 开销（TC）。

*   **模型配置**：

    *   MAGE 防御组件模型：**Qwen3-4B-Instruct**（作为轻量化外部防御模型）。
*   **训练参数**：使用 AdamW 优化器，学习率为 1e-6，Batch Size 为 32。

#### 5. 实验数量与充分性
*   **实验规模**：
    *   在 SHADE-Arena 中使用了 247 个恶意任务和 88 个良性任务。

*   **充分性评价**：实验设计非常充分。除了标准基准测试，还进行了：
    *   **消融实验**：验证了影子内存、RL 训练、用户意图参考等组件的贡献。
    *   **自适应攻击测试**：模拟攻击者已知晓 MAGE 存在并试图通过特定指令污染影子内存或绕过裁判的情况。

#### 6. 主要结论与发现
*   **卓越的防御性能**：在工具链攻击中，MAGE 将 ASR 从 100% 降低到 **8.3%**，同时保持了 **94.4%** 的良性效用。
*   **早期检测能力**：MAGE 能够在攻击发生的早期（第一轮或动作执行前）识别风险，平均检测延迟接近于 0。
*   **高效性**：相比于将全量轨迹输入裁判的方法，影子内存机制将 Token 开销降低了约 **1.8 倍**。
*   **鲁棒性**：面对专门针对防御机制设计的自适应攻击，经过 RL 训练的 MAGE 依然表现出极强的韧性。

#### 7. 优点
*   **架构创新**：首次将智能体记忆机制转化为安全原语，解决了长程交互中安全信号被稀释的问题。
*   **即插即用**：作为外部拦截层，无需重新训练或微调核心智能体模型。
*   **可解释性强**：裁判不仅做决策，还基于影子内存提供人类可读的违规理由。
*   **性能均衡**：在 ASR、BU 和计算成本之间取得了目前最优的平衡（Pareto Optimal）。

#### 8. 不足与局限
*   **攻击覆盖面**：虽然涵盖了主流的长程攻击，但对于内存中毒（Memory Poisoning）或目标漂移（Goal Drifting）的细分变体仍有待进一步验证。
*   **模型依赖**：防御效果高度依赖于外部小模型（如 Qwen3-4B）的指令遵循和提炼能力。
*   **多智能体局限**：目前的研究集中在单智能体系统，尚未探讨在多智能体协同环境下的影子内存同步与安全问题。
