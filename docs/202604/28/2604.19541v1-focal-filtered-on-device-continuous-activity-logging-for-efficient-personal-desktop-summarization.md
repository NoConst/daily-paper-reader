## 摘要
桌面交互流提供了交织的用户任务的连续且隐私敏感的记录。在端侧将这些流转换为按任务组织的个人日志面临两个主要挑战：详尽的视觉语言模型（VLM）处理会耗尽本地资源，而全局流处理会导致跨任务的上下文污染。我们提出了 FOCAL（过滤式端侧连续活动日志记录），这是一个采用统一“过滤-规划-记录”架构的隐私优先多智能体系统。它级联了一个用于噪声抑制的轻量级过滤智能体（Filter Agent）、一个仅处理文本的大脑智能体（Brain Agent）用于任务归因、一个记录智能体（Record Agent）用于选择性视觉推理，以及一个任务隔离的记忆智能体（Memory Agent）用于上下文连贯的摘要。在 DesktopBench（包含 420 个复杂会话中的 2,572 张截图）上的实验表明，与基准方案相比，FOCAL 将总 Token 消耗降低了 60.4%，VLM 调用次数减少了 72.3%，同时将关键信息召回率（KIR）从 0.38 提升至 0.61。至关重要的是，在 A→B→A 任务中断情况下，FOCAL 保持了 0.81 的任务准确率（Task Acc）和 0.80 的 KIR，而基准方案的任务准确率则崩溃至 0.03。FOCAL 开创了将无指令桌面流高效、端侧地摘要为多视角个人日志的先河。

## Abstract
Desktop interaction streams provide a continuous, privacy-sensitive record of interleaved user tasks. Transforming these streams into task-organized personal logs on-device faces two main challenges: exhaustive Vision-Language Model (VLM) processing strains local resources, and global stream processing causes cross-task context pollution. We present FOCAL (Filtered On-device Continuous Activity Logging), a privacy-first multi-agent system utilizing a unified filter-plan-log architecture. It cascades a lightweight Filter Agent for noise suppression, a text-only Brain Agent for task attribution, a Record Agent for selective visual reasoning, and a task-isolated Memory Agent for context-coherent summarization. Experiments on DesktopBench (comprising 2,572 screenshots across 420 complex sessions) show FOCAL reduces total token consumption by 60.4% and VLM call count by 72.3% versus a baseline, while boosting Key Information Recall (KIR) from 0.38 to 0.61. Crucially, under $A{\to}B{\to}A$ task interruptions, FOCAL maintains Task Acc 0.81 and KIR 0.80, whereas the baseline collapses to Task Acc 0.03. FOCAL pioneers the efficient, on-device summarization of instruction-free desktop streams into multi-perspective personal logs.

### 1. 核心问题与整体含义（研究动机和背景）
论文关注的是**个人桌面活动日志的自动化生成**。在日常办公中，用户的操作流是多任务交织的（如写文档时穿插查资料、回复邮件）。将这些原始交互流转化为按任务组织的摘要具有极高的生产力价值，但面临两大挑战：
*   **端侧资源受限（Efficiency）**：详尽地调用视觉语言模型（VLM）处理每一帧截图会严重消耗本地 CPU/GPU 资源，导致系统卡顿。
*   **跨任务上下文污染（Context Pollution）**：用户频繁切换任务，传统的全局处理方式会将不同任务的语义混淆，导致摘要逻辑混乱。
*   **隐私保护（Privacy）**：桌面截图包含敏感信息，必须在端侧（On-device）完成处理，不能依赖云端大模型。

### 2. 方法论：核心思想与技术细节
FOCAL 提出了一个**“过滤-规划-记录”（Filter-Plan-Log）**的多智能体架构，核心思想是**“在昂贵的视觉推理之前进行任务感知的控制”**。

*   **多智能体分工**：
    1.  **过滤智能体 (Filter Agent)**：前端模块，负责捕获截图并剔除无意义的噪声事件。
    2.  **大脑智能体 (Brain Agent)**：**核心创新点**。仅利用轻量级的文本元数据（如 App 名称、窗口标题）对整个会话进行一次性规划，决定哪些帧需要 VLM 采样（Adaptive Sampling），并将每一帧路由到特定的任务 ID。
    3.  **记录智能体 (Record Agent)**：仅对被采样的关键帧调用 VLM，结合当前任务的局部记忆生成语义描述。
    4.  **记忆智能体 (Memory Agent)**：维护**任务隔离的记忆存储**。每个任务拥有独立的上下文空间，防止 A 任务的信息干扰 B 任务。
    5.  **摘要智能体 (Summary Agent)**：在会话结束时，基于各任务隔离的记忆生成最终的结构化个人日志。
*   **算法流程**：系统通过元数据诱导任务边界，将连续流 $S$ 划分为 $K$ 个潜在任务，通过公式化的任务路由机制实现 $\hat{\Sigma} = \bigoplus_{k=1}^{\hat{K}} \hat{\sigma}_k$（即各任务摘要的有序拼接）。

### 3. 实验设计
*   **数据集 (DesktopBench)**：基于 VideoGUI 重构。包含 2,572 张截图，共 420 个复杂会话。
    *   **Multitask 场景**（320个）：模拟跨应用的工作流（如：视频编辑 -> 网页浏览 -> 视频编辑）。
    *   **Interruption 场景**（100个）：专门测试 $A \to B \to A$ 模式下的任务恢复能力。
*   **对比方法**：
    1.  **Naive LLM Agent**：全量处理，无采样过滤，无任务隔离记忆。
    2.  **FOCAL-GM**：保留采样机制，但使用全局共享记忆（Global Memory）。
*   **评价指标**：效率指标（VLM 调用次数 VCC、Token 消耗 TCS）；质量指标（关键信息召回率 KIR、任务计数准确率 Task Acc、BERTScore、GPT-5 评分 G-Eval）。

*   **硬件平台**：所有实验均在配备 **Apple M4 芯片、16GB 统一内存**的 MacBook 上完成，符合“端侧设备”的设定。

### 5. 实验数量与充分性
*   **实验规模**：共 420 组会话实验，涵盖了 5 大类应用家族（视频编辑、图像处理、AI 生成、幻灯片、网页参考）。

### 6. 主要结论与发现
*   **高效性**：相比 Naive 基准，FOCAL 减少了 **60.4% 的 Token 消耗**和 **72.3% 的 VLM 调用**。
*   **高质量**：关键信息召回率（KIR）从 0.38 提升至 **0.61**，G-Eval 评分显著提高。

*   **发现**：单纯的采样（减少帧数）若不配合隔离记忆，反而可能因丢失证据导致摘要质量下降；**任务隔离是提升摘要忠实度的关键**。

### 7. 优点与亮点
*   **架构优雅**：将“感知”与“认知”分离，用低成本的元数据规划高成本的视觉推理。
*   **任务隔离机制**：有效解决了长程交互中的上下文污染问题，这在桌面日志领域是开创性的。
*   **隐私与实用并重**：完全基于端侧小模型实现，具有很强的工程落地潜力。
*   **Benchmark 贡献**：构建了 DesktopBench，填补了无指令桌面活动总结缺乏标准测试集的空白。
