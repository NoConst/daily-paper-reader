## 摘要
基于大语言模型（LLM）的购物智能体日益依赖长购买历史和多轮交互来实现个性化，但由于噪声、长度和相关性不匹配，直接将原始历史记录附加到提示词中往往效果不佳。我们提出了 MemRerank，这是一个偏好记忆框架，它将用户购买历史提炼为简洁且独立于查询的信号，用于个性化商品重排序。为了研究这一问题，我们构建了一个端到端的基准测试和评估框架，其核心是一个基于 LLM 的“五选一”（1-in-5）选择任务，该任务同时衡量记忆质量和下游重排序效用。我们进一步利用强化学习（RL）训练记忆提取器，并以下游重排序性能作为监督信号。在两个基于 LLM 的重排序器上进行的实验表明，MemRerank 始终优于无记忆、原始历史和现成的记忆基准，在“五选一”准确率上最高提升了 10.61 个绝对百分点。这些结果表明，显式偏好记忆是代理式电子商务系统中实现个性化的实用且有效的构建模块。

## Abstract
LLM-based shopping agents increasingly rely on long purchase histories and multi-turn interactions for personalization, yet naively appending raw history to prompts is often ineffective due to noise, length, and relevance mismatch. We propose MemRerank, a preference memory framework that distills user purchase history into concise, query-independent signals for personalized product reranking. To study this problem, we build an end-to-end benchmark and evaluation framework centered on an LLM-based \textbf{1-in-5} selection task, which measures both memory quality and downstream reranking utility. We further train the memory extractor with reinforcement learning (RL), using downstream reranking performance as supervision. Experiments with two LLM-based rerankers show that MemRerank consistently outperforms no-memory, raw-history, and off-the-shelf memory baselines, yielding up to \textbf{+10.61} absolute points in 1-in-5 accuracy. These results suggest that explicit preference memory is a practical and effective building block for personalization in agentic e-commerce systems.

### 1. 核心问题与整体含义
*   **研究动机**：随着大语言模型（LLM）在电商智能体中的应用，如何利用长期的用户购买历史实现个性化推荐成为关键。
*   **核心挑战**：直接将原始购买历史（Raw History）放入 Prompt 会引入大量噪声、消耗过长的上下文窗口，且历史记录与当前查询的相关性往往不匹配，导致重排序效果不佳甚至下降。
*   **研究目标**：提出一种名为 **MemRerank** 的框架，将冗长的购买历史提炼为简洁、结构化且独立于查询的“偏好记忆”（Preference Memory），以提升个性化商品重排序的准确性。

### 2. 方法论
*   **核心思想**：将个性化重排序分为“记忆提取”和“下游重排序”两个阶段，并通过强化学习（RL）使提取器生成的记忆直接服务于重排序任务。
*   **关键技术细节**：
    *   **结构化记忆**：提取器生成两种记忆：**类目内记忆**（Within-category，针对当前商品类目的特定偏好）和**跨类目记忆**（Cross-category，通用的购物风格、预算、品牌倾向等）。
    *   **记忆提取器**：使用 Qwen2.5-7B-Instruct 作为基础模型，通过提示词引导其从商品元数据（标题、描述）中总结偏好。
    *   **RL 训练（GRPO）**：采用群体相对策略优化（Group Relative Policy Optimization, GRPO）算法。
        *   **奖励函数**：由“格式奖励”（确保输出符合特定的 XML 标签结构）和“重排序奖励”（如果生成的记忆能帮助下游重排序器选对正确商品，则给予正奖励）组成。
    *   **1-in-5 任务**：将重排序简化为从 5 个候选商品（1 个正样本，4 个同类目硬负样本）中选出最相关的一个，作为评估和训练的核心目标。

### 6. 主要结论与发现
*   **MemRerank 效果显著**：相比无记忆基准，在 o4-mini 重排序器上准确率提升了 **10.61%**。
*   **RL 训练是核心**：未经训练的提取器效果远不如经过 RL 优化的提取器，证明了“为任务而生”的记忆比单纯的摘要更有用。
*   **提示词设计至关重要**：半结构化且要求提供“证据片段”（Evidence Grounding）的提示词（v3）效果最好，因为它在灵活性和约束性之间取得了平衡。
*   **原始历史的局限性**：简单增加原始历史长度并不总能提升效果，甚至可能因为噪声增加而导致性能下降。

### 7. 优点
*   **任务导向的记忆**：不同于通用的摘要，MemRerank 通过 RL 将记忆提取与下游任务目标对齐。
*   **高效性**：将长历史压缩为短文本，极大地节省了 LLM 的上下文窗口，降低了推理成本。
*   **可解释性**：通过结构化的 Aspect-Preference-Evidence 格式，用户或开发者可以清晰地看到系统认为的偏好依据。

### 8. 不足与局限
*   **领域覆盖有限**：实验主要局限于电子产品类目，不同类目（如服装、食品）的偏好特征差异巨大，其通用性有待验证。
*   **规模限制**：目前仅在 1-in-5 的小规模重排序上验证，尚未测试在大规模候选集（如 Top-100）中的表现。
*   **信号单一**：仅利用了文本元数据，未考虑价格波动、图片信息或用户的实时点击流等动态信号。
