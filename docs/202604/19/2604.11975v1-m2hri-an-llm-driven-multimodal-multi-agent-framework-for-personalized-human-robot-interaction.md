## 摘要
多机器人系统在家庭和医院等社会环境中具有巨大的应用前景，然而现有的多机器人研究往往将机器人视为功能相同的个体，忽视了机器人的个体身份如何塑造用户感知，以及在存在这种个体差异时，协调机制如何影响多机器人的行为。为解决这一问题，我们提出了 M2HRI，这是一个基于大语言模型构建的多模态多智能体框架，它为每个机器人赋予了独特的个性和长期记忆，并建立了一个基于这些差异的协调机制。在一项针对多智能体人机交互（HRI）场景的受控用户研究（n = 105）中，我们发现大语言模型驱动的性格特征具有显著的可区分性并能提升交互质量，长期记忆改善了个性化和偏好感知，而中心化协调显著减少了行为重叠，同时提高了整体交互质量。总之，这些结果表明，智能体的个体特性和结构化协调对于实现连贯且符合社交规范的多智能体人机交互至关重要。项目网站和代码可在 https://project-m2hri.github.io/ 获取。

## Abstract
Multi-robot systems hold significant promise for social environments such as homes and hospitals, yet existing multi-robot works treat robots as functionally identical, overlooking how robots individual identity shape user perception and how coordination shapes multi-robot behavior when such individuality is present. To address this, we introduce M2HRI, a multimodal multi-agent framework built on large language models that equips each robot with distinct personality and long-term memory, alongside a coordination mechanism conditioned on these differences. In a controlled user study (n = 105) in a multi-agent human-robot interaction (HRI) scenario, we find that LLM-driven personality traits are significantly distinguishable and enhance interaction quality, long-term memory improves personalization and preference awareness, and centralized coordination significantly reduces overlap while improving overall interaction quality. Together, these results demonstrate that both agent individuality and structured coordination are essential for coherent and socially appropriate multi-agent HRI. Project website and code are available at https://project-m2hri.github.io/.

*   **研究目标**：探索机器人性格、长期记忆以及中心化协调机制如何共同作用，提升多机器人交互的连贯性、个性化和社交适宜性。

#### 2. 方法论与核心技术
M2HRI 框架采用大语言模型（LLM）和视觉语言模型（VLM）驱动，核心架构分为两个层面：

*   **智能体架构（Agent Architecture）**：每个机器人 $a_i$ 由“身份配置”和“认知循环”组成。
    *   **身份配置 ($\phi_i$)**：包含**记忆（Memory）**（语义、情境、工作记忆）、**性格（Personality）**（基于大五人格模型 FFM）和**动作库**。

*   **中心化协调机制（Multi-Agent Coordination）**：
    *   为了防止多个机器人同时说话，引入了一个中心化协调器。
    *   **算法流程**：协调器收集所有机器人的局部观察、对话上下文和性格向量，利用 LLM 计算每个机器人的**响应似然得分 ($r_{i,t}$)**。只有得分超过阈值的机器人才能按顺序执行动作。

#### 6. 主要结论与发现
*   **性格感知（RQ1）**：用户能显著区分机器人的性格（尤其是开放性和神经质），且性格表现具有高度一致性。性格能增强交互参与感，但“尽责性”在短期交互中较难被察觉。
*   **记忆作用（RQ2）**：长期记忆显著提升了机器人的召回准确率和用户偏好感知，使交互更具个性化，但对“自然度”的提升相对有限。
*   **协调价值（RQ3）**：中心化协调对减少行为重叠（抢话）效果极其显著（效应量 $d=2.04$），并能显著提升对话流的顺畅度和响应的适宜性。

#### 7. 优点与亮点
*   **系统性集成**：首次在一个统一的 LLM 框架下整合了多模态感知、长期记忆、显式性格模型和多机协调机制。
*   **身份解耦设计**：将“身份配置”与“认知循环”分离，使得开发者可以轻松通过修改 Prompt 或参数来定制不同个性的机器人。
*   **解决冲突**：提出的基于 LLM 的响应似然得分机制，为多机器人环境下的轮次转接（Turn-taking）提供了有效的社交计算方案。

#### 8. 不足与局限
*   **交互形式限制**：采用视频观察而非实地交互，可能削弱了用户对机器人具身性（Embodiment）的真实感知。
*   **性格维度的不平衡**：实验发现某些性格特征（如尽责性）在短时间内难以通过对话体现，框架对这类长周期行为特征的建模尚显不足。
*   **扩展性挑战**：目前仅在两台机器人上测试，当机器人数量增加时，中心化协调器的计算开销和对话逻辑的复杂性可能会呈指数级增长。
*   **实时性风险**：依赖云端 LLM API 可能会带来交互延迟，影响社交互动的即时反馈。
