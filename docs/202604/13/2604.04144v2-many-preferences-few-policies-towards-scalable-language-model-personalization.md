## 摘要
大语言模型（LLM）个性化的终极目标是为每个用户提供一个单独的 LLM，使其与该用户的偏好完美对齐。然而，由于计算、内存和系统复杂性的限制，为每个用户维护一个独立的 LLM 是不切实际的。我们通过开发一种有原则的方法来应对这一挑战，该方法选择一小组 LLM 组合（portfolio），以捕捉异质用户群体中的代表性行为。我们通过多维权重向量对用户在多个特征（如安全性、幽默感、简洁性）上的偏好进行建模。在给定这些维度的奖励函数的情况下，我们的算法 PALM（Portfolio of Aligned LLMs）生成一小组 LLM 组合，使得对于任何权重向量，该组合中都包含一个针对相应标量化目标的近乎最优的 LLM。据我们所知，这是首个在 LLM 个性化组合的规模和近似质量方面都提供理论保证的研究结果。它刻画了系统成本与个性化之间的权衡，以及覆盖用户偏好全景所需的 LLM 多样性。我们提供的实证结果验证了这些保证，并展示了相比于常见基准模型更出色的输出多样性。

## Abstract
The holy grail of LLM personalization is a single LLM for each user, perfectly aligned with that user's preferences. However, maintaining a separate LLM per user is impractical due to constraints on compute, memory, and system complexity. We address this challenge by developing a principled method for selecting a small portfolio of LLMs that captures representative behaviors across heterogeneous users. We model user preferences across multiple traits (e.g., safety, humor, brevity) through a multi-dimensional weight vector. Given reward functions across these dimensions, our algorithm PALM (Portfolio of Aligned LLMs) generates a small portfolio of LLMs such that, for any weight vector, the portfolio contains a near-optimal LLM for the corresponding scalarized objective. To the best of our knowledge, this is the first result that provides theoretical guarantees on both the size and approximation quality of LLM portfolios for personalization. It characterizes the trade-off between system cost and personalization, as well as the diversity of LLMs required to cover the landscape of user preferences. We provide empirical results that validate these guarantees and demonstrate greater output diversity over common baselines.

*   **核心问题**：如何在不为每个用户训练独立模型的前提下，通过维护一个**极小规模的模型组合（Portfolio）**，来覆盖海量用户在安全性、简洁性、幽默感等多个维度上的异构偏好？

#### 2. 方法论
论文提出了 **PALM（Portfolio of Aligned LLMs）** 算法，旨在构建一个具有理论保证的小型模型库。
*   **核心思想**：利用多目标对齐中的线性标量化框架。虽然用户偏好权重（Weight Vector）是连续且无限的，但一个针对特定权重优化的策略往往在一定权重范围内都是近乎最优的。
*   **关键技术细节**：
    *   **$(\epsilon, \delta)$-近似保证**：定义了组合的质量，即对于单纯形（Simplex）中的任意权重向量，组合中至少存在一个模型，其性能与该权重下的理论最优模型相比，差距在乘性误差 $\epsilon$ 和加性误差 $\delta$ 之内。
    *   **网格构造（CONSTRUCT GRID）**：PALM 结合了“乘性网格”（处理内部区域）和“均匀网格”（处理边界区域），在保证覆盖全空间的同时，避免了单纯使用某种网格导致的规模爆炸或边界失效。
    *   **剪枝（PRUNE）**：在生成初始候选模型集后，通过求解“最小集合覆盖”问题（贪心算法实现），剔除冗余模型，进一步压缩组合规模。
*   **算法流程**：输入精度参数 $\mu$ 和 $\alpha$ $\rightarrow$ 在权重空间生成结构化采样点 $\rightarrow$ 训练/获取对应模型 $\rightarrow$ 剪枝压缩 $\rightarrow$ 输出最终组合。

#### 3. 实验设计
论文在三个具有代表性的任务上验证了 PALM 的有效性：
*   **数据集/场景**：

    2.  **Safety Alignment**：安全对齐任务，平衡“有用性”与“安全性”。
    3.  **Helpful Assistants**：助手任务，平衡“有用性”、“无害性”与“幽默感”（三维目标）。

    *   **指标**：近似间隙（Approximation Gap）、策略使用分布的困惑度（Perplexity，衡量模型利用率和多样性）。

#### 5. 实验数量与充分性
*   **实验规模**：涵盖了从 2 维到 3 维奖励目标的扩展，模型规模从 3B 到 7B。
*   **充分性**：
    *   进行了**消融实验**（如 Uniform PALM），验证了 PALM 特有的权重生成方案优于简单的均匀采样后剪枝。
    *   提供了**定性分析**，展示了不同模型在同一提示词下的具体回复差异。
*   **客观性**：实验对比了相同规模下的不同组合构建方法，并使用了多种随机种子取平均值（针对随机基准），评估过程较为严谨。

#### 6. 主要结论与发现
*   **规模无关性**：个性化成本不再随用户数量增长，而是取决于奖励函数的维度。在实验中，**少于 7 个模型**的组合即可在全空间实现小于 1.5% 的次优度。
*   **性能优越性**：在相同组合规模下，PALM 的近似误差显著低于均匀采样和随机采样。
*   **高利用率**：PALM 构建的模型在权重空间中分布更合理，避免了基准方法中常见的“某些模型从未被选中”的冗余现象，模型使用率（困惑度）更高。

#### 7. 优点
*   **理论完备性**：这是首个为 LLM 个性化组合规模和近似质量提供显式理论界限的工作。
*   **系统友好**：通过极小的模型集合实现广泛的偏好覆盖，极大地降低了模型部署、托管和安全审计的复杂性。
*   **算法鲁棒**：PALM 的网格构造方法考虑了权重在边界处的退化问题，比传统 MORL 方法更适合 LLM 场景。

#### 8. 不足与局限
*   **线性假设**：研究基于线性标量化（Linear Scalarization）偏好模型，可能无法完全捕捉用户复杂的非线性偏好（如阈值偏好）。
*   **奖励模型依赖**：组合的质量高度依赖于底层奖励模型（Reward Models）的准确性，若奖励模型存在偏差，生成的组合也会失效。
*   **推理代理风险**：在 7B 模型实验中使用了 MOD 推理时对齐作为微调的代理，虽然有理论支撑，但与实际全量微调的效果可能存在细微偏差。
*   **维度灾难**：虽然模型数量不随用户增加，但随奖励维度 $d$ 指数级增长（尽管 $d$ 通常很小），在处理极多目标（如 >10 个）时可能面临挑战。
