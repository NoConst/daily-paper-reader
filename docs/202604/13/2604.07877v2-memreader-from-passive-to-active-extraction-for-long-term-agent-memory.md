## 摘要
长期记忆是实现个性化和自主智能体的基础，但如何填充记忆仍是一个瓶颈。现有系统将记忆提取视为从上下文到结构化条目的一次性、被动转录，这在处理噪声对话、缺失引用和跨轮次依赖时表现不佳，导致记忆污染、低价值写入和不一致性。在本文中，我们推出了用于智能体系统主动长期记忆提取的 MemReader 系列：MemReader-0.6B 是一个紧凑且具有成本效益的被动提取器，通过蒸馏实现准确且符合模式一致性的结构化输出；MemReader-4B 是一个通过群体相对策略优化（GRPO）进行优化的主动提取器，用于做出记忆写入决策。在 ReAct 风格的范式下，MemReader-4B 在采取行动前会明确评估信息价值、引用歧义和完整性，并能选择性地写入记忆、推迟处理不完整输入、检索历史上下文或丢弃无关闲聊。在 LOCOMO、LongMemEval 和 HaluMem 上的实验表明，MemReader 的表现始终优于现有的基于提取的基准模型。特别是，MemReader-4B 在涉及知识更新、时间推理和减少幻觉的任务上达到了最先进的性能。这些结果表明，有效的智能体记忆不仅需要提取更多信息，还需要进行推理驱动的选择性记忆提取，以构建低噪声且动态演进的长期记忆。此外，MemReader 已集成到 MemOS 中，并正在实际应用中部署。为了支持未来的研究和应用，我们发布了模型并提供公开 API 访问。

## Abstract
Long-term memory is fundamental for personalized and autonomous agents, yet populating it remains a bottleneck. Existing systems treat memory extraction as a one-shot, passive transcription from context to structured entries, which struggles with noisy dialogue, missing references, and cross-turn dependencies, leading to memory pollution, low-value writes, and inconsistency. In this paper, we introduce the MemReader family for active long-term memory extraction in agent systems: MemReader-0.6B, a compact and cost-efficient passive extractor distilled for accurate and schema-consistent structured outputs, and MemReader-4B, an active extractor optimized with Group Relative Policy Optimization (GRPO) to make memory writing decisions. Under a ReAct-style paradigm, MemReader-4B explicitly evaluates information value, reference ambiguity, and completeness before acting, and can selectively write memories, defer incomplete inputs, retrieve historical context, or discard irrelevant chatter. Experiments on LOCOMO, LongMemEval, and HaluMem show that MemReader consistently outperforms existing extraction-based baselines. In particular, MemReader-4B achieves state-of-the-art performance on tasks involving knowledge updating, temporal reasoning, and hallucination reduction. These results suggest that effective agent memory requires not merely extracting more information, but performing reasoning-driven and selective memory extraction to build low-noise and dynamically evolving long-term memory. Furthermore, MemReader has been integrated into MemOS and is being deployed in real-world applications. To support future research and adoption, we release the models and provide public API access.

这篇论文由 MemTensor (上海) 科技公司与中国电信研究院的研究团队合作完成，旨在解决智能体长期记忆构建中的“被动提取”瓶颈。以下是对该论文的结构化总结：

### 1. 核心问题与整体含义（研究动机和背景）
长期记忆是实现个性化智能体的关键，但现有的记忆系统（如 Mem0, Zep, MemOS）大多采用**被动提取（Passive Extraction）**模式。这种模式将记忆提取视为从对话到结构化条目的一次性转录，存在以下痛点：
*   **噪声污染：** 无法区分闲聊与高价值信息，导致记忆库充斥低价值内容。
*   **指代歧义：** 难以处理代词（如“他”、“那个项目”）和跨轮次的依赖关系。
*   **更新困难：** 难以处理用户状态的动态演进和信息覆盖。
论文提出将记忆提取从“被动转录”转变为**“主动管理（Active Management）”**，即智能体在写入记忆前，应先思考信息的价值、完整性及是否需要检索历史背景。

### 2. 方法论：核心思想与技术细节
论文推出了 **MemReader** 系列模型，核心在于引入 **ReAct（推理+行动）** 范式和 **GRPO（群体相对策略优化）** 强化学习。

*   **模型家族：**
    *   **MemReader-0.6B：** 紧凑型模型，通过高质量数据蒸馏，专注于低成本、高准确率的结构化 JSON 输出。
    *   **MemReader-4B：** 主动提取器，基于 Qwen3-4B，具备“思考-行动-观察”循环能力。
*   **核心动作空间（Tools）：**
    *   `add_memory`：信息完整且有价值时直接写入。
    *   `buffer_memory`：信息有价值但不完整（如用户话没说完）时暂存。
    *   `search_memory`：遇到歧义时检索历史记忆进行消歧。
    *   `ignore_memory`：丢弃闲聊或重复信息。
*   **训练流程：**
    1.  **SFT 预热：** 使用 Gemini-3-Flash 生成的 7k 条 ReAct 轨迹数据进行监督微调。
    2.  **GRPO 强化学习：** 针对记忆任务设计了多维度奖励函数：
        *   **格式奖励（Format）：** 确保输出符合 JSON 和标签规范。
        *   **动作对齐奖励（Action-Align）：** 纠正决策路径，特别是对错误的 `add` 动作施加重罚。
        *   **LLM-Judge 奖励：** 评估记忆的准确性、完整性和幻觉抑制。
        *   **效率奖励（Efficiency）：** 惩罚过长的推理轨迹，鼓励简洁表达。

### 3. 实验设计
*   **数据集/基准测试：**
    *   **LOCOMO：** 评估长对话记忆（单跳、多跳、时间推理）。
    *   **LongMemEval：** 评估长期交互记忆（知识更新、跨会话一致性）。
    *   **HaluMem-Medium：** 专门评估记忆系统中的幻觉、遗漏和更新准确率。
*   **对比方法：** 包括 Mem0, Zep, MemOS (基于 GPT-4o-mini), MemU, Memobase 等主流记忆框架。

### 4. 资源与算力
*   **硬件环境：** 使用了 **8 张 NVIDIA A800 (80GB) GPU**。
*   **软件框架：** 采用了 LlamaFactory 进行 SFT，使用 verl 框架进行 GRPO 训练。
*   **训练细节：** SFT 阶段学习率为 1e-5，3 个 Epoch；GRPO 阶段学习率为 1e-6，组大小（Group Size）为 8。

### 5. 实验数量与充分性
*   **实验规模：** 论文在三个权威基准上进行了全面测试，涵盖了准确率、F1 值、Token 消耗、幻觉率等多个维度。
*   **消融与对比：** 对比了 SFT 版本与 GRPO 版本的性能差异，验证了强化学习在决策对齐上的作用。
*   **客观性：** 实验使用了统一的 GPT-4o-mini 作为评估器（LLM-as-a-judge），并记录了 Token 消耗以评估成本效益，实验设计较为公平、客观。

### 6. 主要结论与发现
*   **主动管理优于被动提取：** MemReader-4B 在知识更新和时间推理任务上显著超过了基于 GPT-4o-mini 的被动提取基准。
*   **小模型也有大作为：** 0.6B 版本通过专项蒸馏，在结构化准确性上甚至在某些指标上优于更大的通用模型。
*   **强化学习的必要性：** GRPO 显著提升了模型在复杂决策（如先搜索再写入）中的成功率，并有效抑制了推理过程中的幻觉。
*   **低噪声记忆：** 通过 `ignore` 和 `buffer` 机制，MemReader 构建的记忆库更加精炼，减少了下游任务的干扰。

### 7. 优点（亮点）
*   **范式创新：** 首次将记忆提取明确定义为“主动状态管理问题”，而非简单的 NLP 提取任务。
*   **工具化思维：** 将记忆操作（增、查、存、弃）工具化，使记忆系统具备了自省和消歧能力。
*   **训练方案成熟：** 成功将 DeepSeek 提出的 GRPO 算法应用在 Agent 记忆决策领域，并设计了精细的奖励模型。
*   **实用性强：** 提供了 0.6B 和 4B 两个版本，兼顾了成本与性能，且已集成到开源项目 MemOS 中。

### 8. 不足与局限
*   **推理开销：** ReAct 范式涉及多次“思考”和“工具调用”，相比一次性提取，其推理延迟和 Token 消耗更高（尽管论文通过效率奖励进行了优化）。
*   **工具集扩展性：** 目前仅支持 4 种基础动作，对于更复杂的记忆编辑（Edit）、冲突检测（Conflict Resolution）或层级抽象（Abstraction）尚待开发。
*   **评估依赖：** 实验高度依赖 LLM-as-a-judge，可能存在评估模型本身的偏见风险。
