## 摘要
随着基于大语言模型（LLM）的多智能体系统（MAS）从实验性试点演变为复杂且持久的生态系统，智能体之间直接通信的局限性变得日益明显。当前的架构面临上下文碎片化、随机幻觉、僵化的安全边界以及低效的拓扑管理等问题。本文介绍了认知织网节点（Cognitive Fabric Nodes, CFN），这是一种新型中间件层，在智能体之间创建了一个无处不在的“认知织网”。与传统的消息队列或服务网格不同，CFN 不仅仅是透传机制，而是主动的智能中介。该架构的核心是将“记忆”从简单的存储提升为一种主动的功能基质，为其他四项关键能力提供支持：拓扑选择、语义对齐（Semantic Grounding）、安全策略执行和提示词转换。我们建议这些功能中的每一项都由利用强化学习（RL）和优化算法的学习模块来管理，以动态提高系统性能。通过拦截、分析和重写智能体间的通信，认知织网确保了单个智能体保持轻量化，同时使生态系统实现连贯性、安全性和语义对齐。我们在多智能体环境下的 HotPotQA 和 MuSiQue 数据集上评估了 CFN 的有效性，并证明与智能体间的直接通信相比，CFN 在这两个数据集上的性能提升均超过了 10%。

## Abstract
As Large Language Model (LLM) based Multi-Agent Systems (MAS) evolve from experimental pilots to complex, persistent ecosystems, the limitations of direct agent-to-agent communication have become increasingly apparent. Current architectures suffer from fragmented context, stochastic hallucinations, rigid security boundaries, and inefficient topology management. This paper introduces Cognitive Fabric Nodes (CFN), a novel middleware layer that creates an omnipresent "Cognitive Fabric" between agents. Unlike traditional message queues or service meshes, CFNs are not merely pass-through mechanisms; they are active, intelligent intermediaries.   Central to this architecture is the elevation of Memory from simple storage to an active functional substrate that informs four other critical capabilities: Topology Selection, Semantic Grounding, Security Policy Enforcement, and Prompt Transformation. We propose that each of these functions be governed by learning modules utilizing Reinforcement Learning (RL) and optimization algorithms to improve system performance dynamically. By intercepting, analyzing, and rewriting inter-agent communication, the Cognitive Fabric ensures that individual agents remain lightweight while the ecosystem achieves coherence, safety, and semantic alignment.   We evaluate the effectiveness of the CFN on the HotPotQA and MuSiQue datasets in a multi-agent environment and demonstrate that the CFN improves performance by more than 10\% on both datasets over direct agent to agent communication.

### 2. 方法论：核心思想与关键技术
*   **核心思想**：引入 **CFN（认知织网节点）** 作为智能中介，拦截、分析并重写所有智能体间的通信。它将“记忆”视为主动的功能基质，而非被动的存储库。
*   **五大功能支柱**：
    1.  **主动记忆（Active Memory）**：作为系统的“世界模型”，为所有操作提供上下文。
    2.  **拓扑选择（Topology Selection）**：基于意图的路由。利用强化学习（RL）解决“上下文多臂老虎机”问题，将任务动态分配给表现最优、成本最低的智能体。
    3.  **语义对齐（Semantic Grounding）**：验证消息是否符合共享本体，防止“幽灵实体”（不存在的对象）在系统中传播。
    4.  **安全策略执行（Security Enforcement）**：混合防御机制。结合确定性规则（如正则过滤隐私信息）和概率性防御（利用 RL 检测复杂的提示词注入攻击）。
    5.  **提示词转换与重写（Transformation & Re-writing）**：执行层。根据记忆和安全策略，将模糊的原始请求重写为清晰、安全、符合接收方格式的指令。

### 6. 主要结论与发现
*   **性能提升**：CFN 显著提高了 MAS 的协作效率。在 HotPotQA 上，准确率从 80.1% 提升至 91.5%；在 MuSiQue 上，从 72.7% 提升至 86.1%。
*   **消除幻觉**：通过语义对齐模块，CFN 有效拦截了因信息不对称导致的“幽灵实体”传播。
*   **中间件的价值**：证明了“智能网络”可以弥补“平庸智能体”的不足，通过中间件的重写和干预，系统整体表现能接近甚至达到单体最强模型的水平。

### 7. 优点
*   **架构创新**：从“哑网络+强智能体”转向“认知网络+专业化智能体”，符合分布式系统演进逻辑。
*   **解耦意图与实现**：拓扑选择模块实现了基于技能的路由，使系统具备极强的动态扩展性。
*   **状态化安全**：相比传统的无状态过滤器，CFN 能够通过记忆追踪多轮对话中的攻击轨迹，识别碎片化攻击。

### 8. 不足与局限
*   **认知税（Cognitive Tax）**：引入中间件必然增加延迟（网络跳数+推理时间+向量检索），论文虽有提及但未给出详细的延迟损耗数据。
*   **同步开销**：在分布式部署中，维持各节点间记忆和策略的一致性（最终一致性协议）可能在高并发场景下产生挑战。
*   **评估维度单一**：实验主要关注 QA 准确率，未对“提示词重写”带来的额外 Token 成本进行经济性分析。
*   **依赖骨干模型**：CFN 本身也依赖 LLM 进行重写和判断，若中间件模型发生幻觉，可能导致整个通信网格崩溃。
