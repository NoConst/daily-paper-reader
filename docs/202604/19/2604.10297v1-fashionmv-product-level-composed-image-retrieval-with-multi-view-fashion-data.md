## 摘要
组合图像检索（CIR）利用参考图像与修改文本配对来检索目标图像。尽管取得了快速进展，但现有方法和数据集均在图像层面运行——即输入单个参考图像加修改文本，输出单个目标图像——而真实的电子商务用户通常会根据从多个视角展示的产品进行推理。我们将这种不匹配称为“视角不完整性”（View Incompleteness），并正式定义了一项新的多视角 CIR 任务，将标准 CIR 从图像级检索推广到产品级检索。为了支持这一任务，我们构建了 FashionMV，这是首个用于产品级 CIR 的大规模多视角时尚数据集，包含 12.7 万个产品、47.2 万张多视角图像以及超过 22 万个 CIR 三元组，是通过利用大型多模态模型的全自动流水线构建的。我们进一步提出了 ProCIR（产品级组合图像检索），这是一个基于多模态大语言模型的建模框架，采用了三种互补机制：两阶段对话、基于描述的对齐和思维链引导，以及一个可选的有监督微调（SFT）阶段，用于在对比训练之前注入结构化的产品知识。在三个时尚基准测试上对 16 种配置进行的系统消融实验表明：（1）对齐是最关键的单一机制；（2）两阶段对话架构是有效对齐的前提条件；以及（3）SFT 和思维链作为部分冗余的知识注入路径。我们表现最好的 0.8B 参数模型优于所有基准模型，包括尺寸为其 10 倍的通用嵌入模型。数据集、模型和代码已在 https://github.com/yuandaxia2001/FashionMV 公开发布。

## Abstract
Composed Image Retrieval (CIR) retrieves target images using a reference image paired with modification text. Despite rapid advances, all existing methods and datasets operate at the image level -- a single reference image plus modification text in, a single target image out -- while real e-commerce users reason about products shown from multiple viewpoints. We term this mismatch View Incompleteness and formally define a new Multi-View CIR task that generalizes standard CIR from image-level to product-level retrieval. To support this task, we construct FashionMV, the first large-scale multi-view fashion dataset for product-level CIR, comprising 127K products, 472K multi-view images, and over 220K CIR triplets, built through a fully automated pipeline leveraging large multimodal models. We further propose ProCIR (Product-level Composed Image Retrieval), a modeling framework built upon a multimodal large language model that employs three complementary mechanisms -- two-stage dialogue, caption-based alignment, and chain-of-thought guidance -- together with an optional supervised fine-tuning (SFT) stage that injects structured product knowledge prior to contrastive training. Systematic ablation across 16 configurations on three fashion benchmarks reveals that: (1) alignment is the single most critical mechanism; (2) the two-stage dialogue architecture is a prerequisite for effective alignment; and (3) SFT and chain-of-thought serve as partially redundant knowledge injection paths. Our best 0.8B-parameter model outperforms all baselines, including general-purpose embedding models 10x its size. The dataset, model, and code are publicly available at https://github.com/yuandaxia2001/FashionMV.

### FashionMV: 产品级多视角组合图像检索深度总结

#### 1. 核心问题与研究动机
*   **核心问题**：现有的组合图像检索（CIR）任务和数据集均基于“单参考图 + 修改文本 $\rightarrow$ 单目标图”的模式。然而在电商实际场景中，产品通常由多个视角（正面、侧面、背面、细节）展示。
*   **视角不完整性（View Incompleteness）**：单张图像无法捕捉产品的全貌（例如，深V领口仅正面可见，而交叉露背设计仅背面可见）。如果修改指令涉及未观察到的区域，检索将变得不可靠。
*   **研究动机**：将 CIR 从“图像级”推广到“产品级”，定义**多视角 CIR（Multi-View CIR）**任务，使模型能够整合多张视图的信息进行整体推理。

#### 2. 方法论：ProCIR 框架
ProCIR 是一个基于多模态大语言模型（MLLM，如 Qwen3.5-0.8B）的建模框架，核心包含以下技术：
*   **两阶段对话架构（MT）**：
    *   **第一阶段（感知）**：仅输入源产品的多视图图像，生成纯视觉嵌入 $s$。
    *   **第二阶段（推理）**：在第一阶段上下文基础上输入修改文本，生成最终查询嵌入 $q$。这种解耦避免了文本对视觉感知的干扰，并允许源端图像-文本对齐。
*   **描述对齐（Align）**：利用 InfoNCE 损失函数，将产品的视觉嵌入与自动生成的长/短文本描述进行对齐，将结构化产品知识锚定到嵌入空间中。
*   **思维链引导与渐进式移除（CoT）**：在生成嵌入前，让模型在 `<think>` 标签内先生成产品描述。为了防止推理时对文本的依赖，训练过程中采用线性衰减的保留率 $\rho(t)$，逐渐移除这些中间文本。
*   **有监督微调（SFT）**：在对比训练前，先通过描述生成和三元组生成任务对模型进行微调，注入领域特定的结构化知识。

#### 3. 实验设计
*   **数据集**：构建了 **FashionMV**，包含 127K 产品、472K 图像和 220K+ CIR 三元组。数据源自 DeepFashion、Fashion200K 和 FashionGen。
*   **Benchmark**：在上述三个数据集的验证集上进行评估，指标为 Recall@5 和 Recall@10。
*   **对比方法**：
    *   **单图 CIR 模型**：CLIP4CIR, SPRC。
    *   **通用 VLM 嵌入模型**：Qwen3-VL (2B/8B), RzenEmbed (8B), Doubao-Embedding-Vision。
    *   **多视图策略**：对比了 Joint（联合编码）、MeanPool（均值池化）、MaxSim（最大相似度）三种策略。

#### 4. 资源与算力
*   **硬件环境**：使用了 **10 张 NVIDIA RTX 3090 GPU**。
*   **训练时长**：
    *   最快配置（仅 MT）：约 7 小时 20 分钟。
    *   最全配置（MT+Align+CoT+SFT）：约 14 小时 33 分钟（含 4.5 小时 SFT 预开销）。
*   **超参数**：有效 Batch Size 为 160，训练 1 个 Epoch，图像分辨率上限为 512x512。

#### 5. 实验数量与充分性
*   **实验规模**：进行了 **16 种配置**的系统消融实验（8 种机制组合 $\times$ 2 种初始化方式），并在 3 个不同的时尚基准数据集上验证。
*   **客观性与公平性**：
    *   对比了参数量大 10 倍的模型（8B vs 0.8B）。
    *   针对不支持多图输入的模型，适配了 MeanPool 和 MaxSim 策略以确保对比公平。
    *   数据集划分严格遵循产品级去重，确保无训练/验证集重叠。

#### 6. 主要结论与发现
*   **对齐是核心**：描述对齐（Align）是提升性能最关键的单一机制，尤其在两阶段对话架构下效果显著。
*   **架构前提**：两阶段对话（MT）是实现有效对齐的前提，单阶段模型无法进行源端视觉-文本对齐。
*   **知识路径冗余**：SFT 和 CoT 在注入产品知识方面具有部分替代性。一旦模型通过 SFT 内化了语义，CoT 的边际收益会消失甚至产生负面影响。
*   **小模型胜大模型**：经过专门设计的 0.8B 模型在产品级检索任务上全面超越了 8B 规模的通用模型。

#### 7. 优点与亮点
*   **任务创新**：首次正式定义并解决了 CIR 中的“视角不完整性”问题，将任务提升到产品实体层面。
*   **数据质量**：通过全自动流水线（结合 Kimi, Qwen, Gemini 等多个大模型）构建了高质量、带方向纠错的多视图数据集。
*   **工程实用性**：两阶段对话设计允许缓存第一阶段的 KV Cache，显著降低了交互式检索的延迟。

#### 8. 不足与局限
*   **模型规模限制**：受限于算力，目前仅在 0.8B 模型上验证，尚未在大规模（如 7B+）模型上探索性能极限。
*   **领域局限性**：目前仅专注于时尚领域，尚未验证该框架在家具、电子产品等其他多视图电商领域的泛化能力。
*   **SFT 数据依赖**：高性能高度依赖于 SFT 阶段，而 SFT 需要极高质量的结构化描述数据，这在某些垂直领域可能难以获取。
