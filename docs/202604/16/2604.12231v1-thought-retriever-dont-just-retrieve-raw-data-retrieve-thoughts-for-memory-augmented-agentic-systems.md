## 摘要
大语言模型（LLMs）凭借其强大的内部能力和知识改变了人工智能研究。然而，现有的 LLM 在与世界交互时，仍无法有效地整合海量的外部知识。尽管检索增强型 LLM 的提出缓解了这一问题，但它们仍从根本上受限于 LLM 的上下文长度，因为它们只能从通常包含数百万个数据块的外部知识库中检索前 K 个原始数据块。在本文中，我们提出了 Thought-Retriever，这是一种新颖的、与模型无关的算法，旨在帮助 LLM 在不受上下文长度或检索数据块数量限制的情况下，基于任意长度的外部数据生成输出。我们的核心见解是让 LLM 充分利用其在解决过去用户查询时生成的中间响应（即“思维”），过滤掉无意义和冗余的思维，将其组织在思维记忆中，并在处理新查询时检索相关的思维。这有效地为基于 LLM 的智能体配备了自进化的长期记忆，使其能力随着持续的交互而不断增强。除了算法创新，我们还精心准备了一个全新的基准测试 AcademicEval，该基准要求 LLM 忠实地利用超长上下文，基于真实世界的学术论文回答查询。在 AcademicEval 和另外两个公开数据集上的广泛实验验证了 Thought-Retriever 显著优于最先进的基准模型，在各项任务中 F1 分数平均提升至少 7.6%，胜率提升 16%。更重要的是，我们进一步展示了两个令人兴奋的发现：（1）Thought-Retriever 确实能帮助 LLM 在解决更多用户查询后实现自进化；（2）Thought-Retriever 学会了利用更深层次的思维来回答更抽象的用户查询。

## Abstract
Large language models (LLMs) have transformed AI research thanks to their powerful internal capabilities and knowledge. However, existing LLMs still fail to effectively incorporate the massive external knowledge when interacting with the world. Although retrieval-augmented LLMs are proposed to mitigate the issue, they are still fundamentally constrained by the context length of LLMs, as they can only retrieve top-K raw data chunks from the external knowledge base which often consists of millions of data chunks. Here we propose Thought-Retriever, a novel model-agnostic algorithm that helps LLMs generate output conditioned on arbitrarily long external data, without being constrained by the context length or number of retrieved data chunks. Our key insight is to let an LLM fully leverage its intermediate responses generated when solving past user queries (thoughts), filtering meaningless and redundant thoughts, organizing them in thought memory, and retrieving the relevant thoughts when addressing new queries. This effectively equips LLM-based agents with a self-evolving long-term memory that grows more capable through continuous interaction. Besides algorithmic innovation, we further meticulously prepare a novel benchmark, AcademicEval, which requires an LLM to faithfully leverage ultra-long context to answer queries based on real-world academic papers. Extensive experiments on AcademicEval and two other public datasets validate that Thought-Retriever remarkably outperforms state-of-the-art baselines, achieving an average increase of at least 7.6% in F1 score and 16% in win rate across various tasks. More importantly, we further demonstrate two exciting findings: (1) Thought-Retriever can indeed help LLM self-evolve after solving more user queries; (2) Thought-Retriever learns to leverage deeper thoughts to answer more abstract user queries.

# Thought-Retriever 论文深度总结

## 1. 核心问题与整体含义
*   **研究动机**：尽管大语言模型（LLM）具备强大的内部知识，但在处理海量外部知识（如数百万个数据块）时，仍受限于有限的上下文窗口。
*   **现有技术痛点**：
    *   **标准 RAG (RALM)**：仅能检索 Top-K 个原始数据块，难以覆盖回答复杂问题所需的全局信息。
    *   **分层 RAG (Hierarchical RALM)**：通过预先对数据块进行总结构建树状结构，但这种总结是静态的、与查询无关的，且构建成本高、灵活性差。
*   **核心目标**：提出一种名为 **Thought-Retriever** 的算法，让模型不仅检索原始数据，还检索过去解决问题时产生的“思维（Thoughts）”，从而构建一个能够随交互持续进化的长期记忆系统。

## 2. 方法论
### 核心思想
将 LLM 的中间响应转化为持久化的、由查询驱动的“思维单元”。这些思维是原始数据的抽象提炼，能够跨越多个数据块建立逻辑联系。

### 关键技术流程
1.  **思维检索 (Thought Retrieval)**：接收用户查询 $Q_i$，从原始知识库 $K$ 和思维记忆库 $T$ 中通过向量相似度检索最相关的 Top-K 内容。
2.  **答案生成 (Answer Generation)**：LLM 基于检索到的原始数据和思维生成最终答案 $A_i$。
3.  **思维与置信度生成 (Thought & Confidence Generation)**：LLM 将查询和答案提炼为一个简洁的“知识点”（即思维 $T_i$），并输出一个二元置信度 $c_i$（1 表示有效，0 表示可能存在幻觉或无意义）。
4.  **思维合并与去重 (Thought Merge)**：计算新思维与现有记忆的相似度 $s_i$，若超过阈值 $\epsilon$ 则视为冗余。
5.  **记忆更新 (Memory Update)**：仅当思维有效（$c_i=1$）且不冗余（$s_i=0$）时，将其存入思维记忆库 $T$。

### 抽象层级定义
论文递归地定义了思维的抽象级别：原始数据级别为 1；思维的级别为其来源（原始块或旧思维）的平均级别加 1。这使得模型能处理更复杂的抽象查询。

## 3. 实验设计
### 数据集与基准 (Benchmark)
*   **AcademicEval (本文构建)**：基于 arXiv 论文的真实学术任务。
    *   *Abstract-single/multi*：根据论文正文生成单篇或多篇论文的摘要。
    *   *Related-multi*：根据目标论文的标题和摘要，从大量参考文献中检索并撰写“相关工作”章节。
*   **公共数据集**：
    *   *GovReport*：政府报告的长文本摘要。
    *   *WCEP*：多篇新闻文章的事件摘要。
    *   *LooGLE*：长文本理解与推理基准。

### 对比方法
*   **启发式检索**：BM25, TF-IDF。
*   **深度学习检索器**：Contriever, DPR, DRAGON, Qwen3-Embed-8b。
*   **高级策略**：IRCoT（迭代检索）、RECOMP（上下文压缩）。
*   **长上下文模型**：OpenOrca-8k, Nous Hermes-32k。
*   **结构化方法**：MemWalker（树状记忆导航）。

*   **硬件**：实验在单张 **NVIDIA A6000 GPU** 上运行。
*   **API 支持**：使用了 Together AI 的 API 调用 Mistral-8x7B 模型。

## 5. 实验数量与充分性
*   **实验规模**：涵盖了 5 个主要数据集任务，并进行了多维度的对比。
*   **消融实验**：测试了不同检索器底座（Contriever vs NV-Embed）、不同 LLM 底座（Qwen-7B, Llama-3-70B）、以及有无过滤机制的影响。
*   **充分性评价**：实验设计较为全面，不仅对比了 F1 分数，还引入了基于强模型（Qwen1.5-72B）的“胜率（Win Rate）”评估和人工评估。通过“Oracle（金标准）”对比证明了思维检索在信息密度上的优势，实验逻辑严密、客观。

## 6. 主要结论与发现
1.  **性能卓越**：在所有任务中，Thought-Retriever 的 F1 分数平均提升 7.6% 以上，胜率提升 16% 以上。
2.  **自进化能力**：随着用户查询数量的增加，思维记忆库不断丰富，模型的表现呈现出一种“记忆层面的 Scaling Law”。
3.  **深度匹配**：模型能够根据查询的抽象程度自动匹配相应层级的思维。越抽象的问题，检索到的思维层级越高。
4.  **高效性**：相比于 MemWalker，推理速度提升了 25 倍以上；相比于处理超长上下文的 Nous Hermes，速度提升了 8 倍。

## 7. 优点
*   **模型无关性**：作为一个通用的记忆模块，可以轻松集成到任何 LLM 智能体系统中。
*   **动态性**：思维是随交互动态生成的，比预先构建的静态摘要更贴合实际使用场景。
*   **高信息密度**：通过将冗余的原始数据压缩为高质量思维，在有限的上下文窗口内提供了更丰富的信息覆盖。
*   **过滤机制**：引入置信度检查和去重，有效抑制了 RAG 系统常见的幻觉和冗余问题。

## 8. 不足与局限
*   **领域偏差**：AcademicEval 主要集中在计算机科学（AI 相关）领域，其在其他学科（如人文、法律）的泛化能力尚待验证。
*   **语言限制**：目前实验仅针对英文，未探讨在多语言或低资源语言环境下的表现。
*   **冷启动依赖**：系统的“进化”依赖于初始阶段高质量的查询和交互，如果初始交互质量较低，可能会影响后续思维库的质量。
*   **动态更新成本**：虽然推理高效，但随着思维库无限增长，长期下的检索压力和存储管理（如思维的遗忘机制）在文中未深入讨论。
