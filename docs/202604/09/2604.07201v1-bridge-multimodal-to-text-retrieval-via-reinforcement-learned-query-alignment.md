## 摘要
多模态检索系统在处理针对纯文本语料库的图像-文本查询时面临挑战：最先进的视觉-语言编码器在 MM-BRIGHT 数据集上仅达到 27.6 nDCG@10，表现逊于强大的纯文本检索器。我们认为瓶颈不在于检索器，而在于查询——原始多模态查询将视觉描述、对话噪声和检索意图交织在一起，从而系统性地降低了嵌入相似度。我们提出了 BRIDGE，这是一个由两个组件组成的系统，无需多模态编码器即可解决这种不匹配问题。FORGE（Focused Retrieval Query Generator，聚焦检索查询生成器）是一个通过强化学习训练的查询对齐模型，它将噪声多模态查询提炼为紧凑且经过检索优化的搜索字符串。LENS（Language-Enhanced Neural Search，语言增强神经搜索）是一个推理增强型稠密检索器，在推理密集型检索数据上进行了微调，以处理 FORGE 生成的富含意图的查询。在 MM-BRIGHT（包含 2,803 个查询，覆盖 29 个领域）上的评估结果显示，BRIDGE 达到了 29.7 nDCG@10，超过了包括 Nomic-Vision (27.6) 在内的所有多模态编码器基准。当 FORGE 作为即插即用的对齐器应用于 Nomic-Vision 之上时，组合系统的 nDCG@10 达到了 33.3，超过了表现最好的纯文本检索器 (32.2)，这证明了查询对齐是多模态到文本检索的关键瓶颈。

## Abstract
Multimodal retrieval systems struggle to resolve image-text queries against text-only corpora: the best vision-language encoder achieves only 27.6 nDCG@10 on MM-BRIGHT, underperforming strong text-only retrievers. We argue the bottleneck is not the retriever but the query -- raw multimodal queries entangle visual descriptions, conversational noise, and retrieval intent in ways that systematically degrade embedding similarity. We present \textbf{BRIDGE}, a two-component system that resolves this mismatch without multimodal encoders. \textbf{FORGE} (\textbf{F}ocused Retrieval Query Generato\textbf{r}) is a query alignment model trained via reinforcement learning, which distills noisy multimodal queries into compact, retrieval-optimized search strings. \textbf{LENS} (\textbf{L}anguage-\textbf{E}nhanced \textbf{N}eural \textbf{S}earch) is a reasoning-enhanced dense retriever fine-tuned on reasoning-intensive retrieval data to handle the intent-rich queries FORGE produces. Evaluated on MM-BRIGHT (2,803 queries, 29 domains), BRIDGE achieves \textbf{29.7} nDCG@10, surpassing all multimodal encoder baselines including Nomic-Vision (27.6). When FORGE is applied as a plug-and-play aligner on top of Nomic-Vision, the combined system reaches \textbf{33.3} nDCG@10 -- exceeding the best text-only retriever (32.2) -- demonstrating that \textit{query alignment} is the key bottleneck in multimodal-to-text retrieval. https://github.com/mm-bright/multimodal-reasoning-retrieval

### 1. 核心问题与整体含义（研究动机和背景）
论文关注的是**多模态到文本检索（Multimodal-to-Text Retrieval）**任务，即用户输入“图像+文本”查询，从大规模“纯文本”语料库中检索相关文档。
*   **研究动机**：现有的视觉-语言模型（VLM）在处理此类任务时表现不佳，甚至不如强力的纯文本检索器。
*   **核心瓶颈**：作者认为瓶颈不在于检索器的视觉编码能力，而在于**查询表示的噪声**。原始多模态查询将视觉细节、对话噪声和检索意图交织在一起，导致生成的嵌入向量（Embedding）无法精准匹配文本语料。
*   **整体含义**：通过将多模态查询“对齐”并“提炼”为纯文本搜索字符串，可以消除模态鸿沟，充分利用现有的高性能文本检索技术。

### 2. 方法论：BRIDGE 系统
BRIDGE 由两个核心组件构成，采用“先描述、再提炼、后检索”的流程：

*   **视觉描述生成（Visual Captioning）**：
    使用 GPT-4o 将查询图像转换为详细的文本描述 $\delta(q_v)$，将视觉信息转化为文本模态。
*   **FORGE（聚焦检索查询生成器）**：
    *   **核心思想**：一个查询对齐模型，负责将“原始问题 + 图像描述”提炼为不超过 200 字的、针对检索优化的搜索字符串。
    *   **关键技术**：基于 Qwen2.5-7B-Instruct，采用**强化学习（RL）**进行训练。
    *   **算法流程**：使用**组相对策略优化（GRPO）**算法。模型生成的查询 $\hat{q}$ 会直接输入检索器，根据检索结果的 **nDCG@k** 作为奖励信号（Reward）。这意味着模型学习的是“如何写出让检索器更容易找到答案的查询”，而非简单模仿人类的重写习惯。
*   **LENS（语言增强神经搜索）**：
    *   **核心思想**：一个推理增强型稠密检索器（Bi-encoder）。
    *   **技术细节**：基于 Qwen3-Embedding-4B，在包含数学、科学、医学、法律等推理密集型检索数据上进行对比学习（Contrastive Learning）微调，以处理 FORGE 生成的高意图密度查询。

### 3. 实验设计
*   **数据集**：**MM-BRIGHT**。这是首个推理密集型多模态检索基准，包含 2,803 个查询，涵盖 29 个技术领域（如生物信息学、量子计算、法律、游戏等）。
*   **Benchmark 指标**：主要使用 **nDCG@10**。
*   **对比方法**：

*   **硬件环境**：使用了 **4 × NVIDIA H100 80GB GPU**。
*   **训练细节**：
    *   **FORGE**：在 MM-BRIGHT 训练集上使用 GRPO 训练 3 个 epoch，学习率 $1 \times 10^{-6}$。
    *   **LENS**：在推理密集型数据上训练 3 个 epoch，batch size 为 512，学习率 $1 \times 10^{-5}$。
    *   **推理阶段**：图像描述生成（GPT-4o）是离线完成并缓存的。

*   **实验规模**：
    *   在 29 个细分领域进行了详尽的性能对比。
    *   **消融实验**：验证了 LENS 检索器、图像描述、监督微调（SFT）对齐与强化学习（RL）对齐各自的贡献。
    *   **模型规模实验**：测试了 FORGE 在不同参数量（3B 到 72B）下的表现。
    *   **通用性实验**：将 FORGE 作为“插件”配合 BM25、Nomic-Vision 等不同检索器使用。
*   **充分性评价**：实验设计非常充分且具有说服力。特别是“即插即用”实验证明了 FORGE 的增益是跨模型的，而不仅仅局限于自家的 LENS 检索器。

### 6. 主要结论与发现
*   **超越基准**：BRIDGE 达到了 29.7 nDCG@10，显著超过了最强的多模态编码器 Nomic-Vision (27.6)。
*   **打破瓶颈**：当 FORGE 配合 Nomic-Vision 使用时，得分升至 **33.3**，甚至超过了纯文本检索器的最高水平（32.2）。这证明了**查询对齐（Query Alignment）**才是多模态检索的真正瓶颈。
*   **RL 的必要性**：强化学习训练的 FORGE 优于监督微调（SFT）和零样本 GPT-4o，说明针对检索指标直接优化比模仿人类重写更有效。
*   **模型规模并非唯一**：在 FORGE 的骨干模型选择上，11B 的模型表现优于 72B，说明简洁的指令遵循能力比原始参数量更重要。

### 7. 优点
*   **架构创新**：将复杂的多模态检索简化为“文本对齐+文本检索”，无需在推理时运行昂贵的视觉编码器，部署更轻量。
*   **训练策略**：巧妙地将检索指标（nDCG）作为 RL 的奖励信号，使模型生成的查询具有极强的“检索导向性”。
*   **模块化/兼容性**：FORGE 可以作为任何现有检索系统的预处理插件，具有极高的实用价值。

### 8. 不足与局限
*   **依赖外部大模型**：初始的图像描述依赖于 GPT-4o，这在离线处理大规模数据时会产生较高的 API 成本，且描述质量直接影响后续对齐效果。
*   **领域差异**：在某些视觉信息仅起装饰作用的领域（如法律、数学），BRIDGE 的提升并不明显，甚至略逊于某些基准。
*   **实时性挑战**：虽然检索阶段是纯文本的，但“描述生成 + FORGE 重写”这两个步骤增加了推理链路的延迟。
