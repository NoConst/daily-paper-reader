## 摘要
基于大语言模型（LLM）的检索器提升了在复杂查询上的性能，但其实际价值除了准确率外，还取决于效率、鲁棒性以及可靠的置信度信号。我们在 12 个任务和 14 个检索器上复现了一个推理密集型检索基准（BRIGHT），并扩展了评估维度，包括冷启动索引成本、查询延迟分布与吞吐量、语料库扩展性、对受控查询扰动的鲁棒性，以及用于预测查询成功的置信度应用（AUROC）。我们还通过将标准查询与五种提供的推理增强变体进行比较，衡量了相对于增加延迟的准确率增益，从而量化了“推理开销”。我们发现，一些专门针对推理优化的检索器在保持竞争性吞吐量的同时实现了强大的有效性，而一些基于大型 LLM 的双编码器（bi-encoders）为了微小的增益却产生了巨大的延迟。推理增强对于 1B 参数以下的编码器产生的延迟极小，但对于顶级检索器则表现出收益递减，并可能降低在正式数学/代码领域的性能。各类模型家族的置信度校准普遍较弱，这表明在没有额外校准的情况下，原始检索分数对于下游路由是不可靠的。我们发布了所有代码和人工制品以供复现。

## Abstract
Large language model retrievers improve performance on complex queries, but their practical value depends on efficiency, robustness, and reliable confidence signals in addition to accuracy. We reproduce a reasoning-intensive retrieval benchmark (BRIGHT) across 12 tasks and 14 retrievers, and extend evaluation with cold-start indexing cost, query latency distributions and throughput, corpus scaling, robustness to controlled query perturbations, and confidence use (AUROC) for predicting query success. We also quantify \emph{reasoning overhead} by comparing standard queries to five provided reasoning-augmented variants, measuring accuracy gains relative to added latency. We find that some reasoning-specialized retrievers achieve strong effectiveness while remaining competitive in throughput, whereas several large LLM-based bi-encoders incur substantial latency for modest gains. Reasoning augmentation incurs minimal latency for sub-1B encoders but exhibits diminishing returns for top retrievers and may reduce performance on formal math/code domains. Confidence calibration is consistently weak across model families, indicating that raw retrieval scores are unreliable for downstream routing without additional calibration. We release all code and artifacts for reproducibility.

### 1. 核心问题与研究动机
*   **核心问题**：随着 LLM 在检索领域的应用，虽然在复杂推理查询上取得了显著的准确率提升，但其背后的**计算成本（延迟、内存、索引时间）**、**鲁棒性**以及**置信度可靠性**是否与其性能增益相匹配？
*   **研究动机**：现有的基准测试（如 MS MARCO, BEIR）主要关注排名效果（nDCG），忽略了实际部署中的效率权衡。开发者难以判断为了微小的性能提升而引入巨大的 LLM 推理开销是否值得。

### 2. 方法论
研究者建立了一个多维度的评估框架，涵盖以下核心思想：
*   **效率比（Efficiency Ratio, ER）**：定义为 `nDCG@10 的增益 / 增加的延迟 (ms)`。用于量化推理增强查询（Reasoning-augmented queries）带来的性价比。
*   **推理增强分析**：利用五种 LLM（GPT-4, Claude-3, Llama-3-70B, Gemini, GritLM）生成带有思维链（CoT）的增强查询，对比其相对于原始查询的性能提升与延迟代价。
*   **鲁棒性测试**：通过四种受控扰动（改写、同义词替换、对抗性标记插入、长度变化）评估检索器的稳定性。
*   **置信度校准**：使用 AUROC 指标，将检索得分作为预测查询是否成功（Top-k 是否包含正确答案）的信号，评估模型自我判断的能力。

### 3. 实验设计
*   **基准测试**：采用 **BRIGHT** 基准，包含 12 个推理密集型任务（涵盖 StackExchange 科学/技术论坛、编程 LeetCode、数学定理等）。
*   **对比方法**：共 14 种检索器，分为四类：

    4.  **推理专用检索器**：ReasonIR, RaDeR, Diver。
*   **场景**：对比了短上下文（分块）和长上下文（原始网页）两种检索设置。

### 6. 主要结论与发现
*   **推理专用模型胜出**：如 **Diver** 模型在帕累托前沿上表现最优，以更小的参数量和更高的吞吐量（47.3 QPS）超越了 7B 规模的通用 LLM 检索器（如 GTE-Qwen2, 6.0 QPS）。
*   **推理增强的性价比极不均衡**：
    *   对于 1B 以下的小模型，推理增强几乎是“免费的午餐”（延迟增加极小但增益显著）。
    *   对于大型 LLM 检索器，推理增强会导致巨大的延迟惩罚（ER 极低），且在数学和代码领域反而可能因引入噪声而降低性能。

*   **多维度评估**：跳出了单一的准确率指标，将工业界关心的延迟、吞吐量和鲁棒性作为核心评估维度。
*   **揭示“隐藏成本”**：量化了 LLM 检索器在索引和查询时的真实开销，为从业者提供了选型指南。
*   **发现任务特异性**：明确指出推理增强不适用于所有领域（如正式的数学/代码任务）。

### 8. 不足与局限
*   **模型架构局限**：主要关注双编码器（Bi-encoders），未深入探讨交叉编码器（Cross-encoders）或重排序器（Rerankers）在推理任务中的成本效益。
