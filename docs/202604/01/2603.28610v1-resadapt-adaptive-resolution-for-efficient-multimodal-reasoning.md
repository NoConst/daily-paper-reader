## 摘要
多模态大语言模型 (MLLMs) 通过提高输入保真度实现了更强的视觉理解，但由此产生的视觉 token 增长使得同时维持高空间分辨率和长时序上下文变得代价高昂。我们认为瓶颈不在于编码后的表示如何压缩，而在于编码器接收的像素量，并提出了 ResAdapt，这是一种输入端自适应框架，用于在编码前学习每帧应分配多少视觉预算。ResAdapt 将轻量级分配器 (Allocator) 与未改变的 MLLM 主干网络耦合，使主干网络在接收算子转换后的输入时，仍保留其原生的视觉 token 接口。我们将分配过程建模为上下文老虎机 (contextual bandit)，并使用成本感知策略优化 (Cost-Aware Policy Optimization, CAPO) 训练分配器，该方法将稀疏的展开反馈转化为稳定的准确率-成本学习信号。在受预算控制的视频问答、时序定位和图像推理任务中，ResAdapt 改善了低预算运行点，且通常处于或接近效率-准确率前沿，在激进压缩下的推理密集型基准测试中提升最为明显。值得注意的是，在相同的视觉预算下，ResAdapt 支持多达 16 倍的帧数，同时提供超过 15% 的性能提升。代码可在 https://github.com/Xnhyacinth/ResAdapt 获取。

## Abstract
Multimodal Large Language Models (MLLMs) achieve stronger visual understanding by scaling input fidelity, yet the resulting visual token growth makes jointly sustaining high spatial resolution and long temporal context prohibitive. We argue that the bottleneck lies not in how post-encoding representations are compressed but in the volume of pixels the encoder receives, and address it with ResAdapt, an Input-side adaptation framework that learns how much visual budget each frame should receive before encoding. ResAdapt couples a lightweight Allocator with an unchanged MLLM backbone, so the backbone retains its native visual-token interface while receiving an operator-transformed input. We formulate allocation as a contextual bandit and train the Allocator with Cost-Aware Policy Optimization (CAPO), which converts sparse rollout feedback into a stable accuracy-cost learning signal. Across budget-controlled video QA, temporal grounding, and image reasoning tasks, ResAdapt improves low-budget operating points and often lies on or near the efficiency-accuracy frontier, with the clearest gains on reasoning-intensive benchmarks under aggressive compression. Notably, ResAdapt supports up to 16x more frames at the same visual budget while delivering over 15% performance gain. Code is available at https://github.com/Xnhyacinth/ResAdapt.

### 1. 核心问题与整体含义（研究动机和背景）
多模态大语言模型（MLLM）在处理高分辨率图像和长视频时面临“视觉标记（Token）爆炸”的问题。现有的效率优化方案存在局限性：
*   **模型侧压缩（如 Token 合并/剪枝）：** 在编码后进行，无法恢复已丢失的细粒度信息，且往往破坏推理引擎的优化内核。
*   **输出侧智能体推理（如 迭代缩放/检索）：** 引入多轮交互延迟，且依赖初始的粗糙视图，可能遗漏关键线索。
*   **核心动机：** 作者认为瓶颈在于编码器接收的**原始像素总量**。论文提出了 **ResAdapt**，通过在编码前动态分配每帧的视觉预算（分辨率），实现输入侧的自适应调整，从而在有限的计算预算下兼顾高空间分辨率和长时序上下文。

### 2. 方法论
ResAdapt 的核心是一个轻量级的**分配器（Allocator）**，它在主干网络处理前决定每帧的缩放比例。
*   **核心思想：** 将视觉预算分配建模为一个**上下文老虎机（Contextual Bandit）**问题。分配器根据文本查询（Query）和粗糙的视觉特征，为每帧预测一个缩放因子 $s_t$。
*   **关键技术细节：**
    *   **分配器架构：** 采用 SmolVLM 架构，通过时间自注意力（Temporal Self-attention）和查询交叉注意力（Cross-attention）捕捉时空冗余和查询相关性。输出参数化为 **Beta 分布**，以支持连续的缩放动作采样。
    *   **成本感知策略优化（CAPO）：** 针对强化学习中准确率与成本权衡不稳定的问题，CAPO 引入了**非对称奖励塑形**。它根据动态成本基准（Dynamic Pivot）对高效的正确回答给予奖励，对高能耗的错误回答给予更重的惩罚，防止策略坍缩至最小预算。
    *   **时序相似性正则化（Temporal-similarity Regularizer）：** 惩罚相邻相似帧同时获得高预算的情况，鼓励模型在时序上进行差异化分配。
    *   **算子无关性：** 虽然实验主要使用双线性插值（Resize），但该框架理论上支持帧选择等其他输入侧算子。

### 5. 实验数量与充分性
*   **实验规模：** 论文涵盖了 15 个以上的基准测试，跨越了视频理解、时序定位和静态图像三大领域。
*   **消融实验：** 针对奖励设计（CAPO vs 直接惩罚）、分布类型（Beta vs 正态）、时序正则化项、以及算子泛化性（零样本迁移至帧选择）进行了详尽的消融研究。
*   **客观性：** 实验通过 Pareto 前沿曲线展示了不同 Token 保留率下的性能对比，确保了在相同计算预算下的公平比较。

### 6. 主要结论与发现
*   **效率提升：** 在相同视觉预算下，ResAdapt 可支持多达 **16 倍**的帧数，性能提升超过 **15%**。
*   **Pareto 前沿：** ResAdapt 在低预算运行点上显著优于现有的 Token 压缩方法，尤其是在推理密集型任务（如 VideoMMMU）上。
*   **主动感知：** 学习到的策略表现出“开环主动感知”能力，能自动将预算集中在信息密集帧（如包含文字、动作剧烈或场景切换的帧）。
*   **延迟优势：** 随着视频长度增加，ResAdapt 节省的计算量远超分配器的开销。在 128 帧设置下，端到端延迟可降低约 **60%**。

### 7. 优点
*   **架构兼容性：** 无需修改主干网络架构，完全保留了原生 Token 接口，兼容 FlashAttention 等硬件加速内核。
*   **端到端学习：** 相比手工设计的启发式规则，ResAdapt 能通过任务奖励学习到查询相关的分配策略。
*   **时空权衡：** 成功证明了“节省空间分辨率以换取更长时序覆盖”在长视频理解中的优越性。

### 8. 不足与局限
*   **开环限制：** 分配决策是在编码前一次性完成的（开环），模型无法在推理过程中根据不确定性动态调整或回溯已压缩的信息。
*   **粗糙特征依赖：** 分配器依赖于冻结的粗糙视觉特征，可能在处理极细微的视觉线索（如极小的文字或短暂的闪烁）时判断失误。
*   **跨域迁移：** 虽然在视频任务上表现强劲，但在静态图像任务上的效率增益相对不明显，且跨模型迁移时存在一定的性能波动。
