## 摘要
长期对话记忆需要检索分散在多个会话中的证据，然而单次检索在处理时间相关和多跳问题时往往会失败。现有的迭代方法通过生成的内容或文档级信号来优化查询，但没有一种方法能明确诊断出证据缺口（即累积检索集中缺失的内容），导致查询优化缺乏针对性。我们提出了 EviMem，它结合了 IRIS（通过不足信号进行迭代检索）和 LaceMem（对话证据记忆的分层架构）。IRIS 是一个闭环框架，通过充分性评估检测证据缺口，诊断缺失内容并驱动有针对性的查询优化；LaceMem 是一种由粗到精的记忆层级结构，支持细粒度的缺口诊断。在 LoCoMo 数据集上，EviMem 在时间相关问题（从 73.3% 提升至 81.6%）和多跳问题（从 65.9% 提升至 85.2%）上的评判准确率（Judge Accuracy）均优于 MIRIX，且延迟降低了 4.5 倍。代码：https://github.com/AIGeeksGroup/EviMem。

## Abstract
Long-term conversational memory requires retrieving evidence scattered across multiple sessions, yet single-pass retrieval fails on temporal and multi-hop questions. Existing iterative methods refine queries via generated content or document-level signals, but none explicitly diagnoses the evidence gap, namely what is missing from the accumulated retrieval set, leaving query refinement untargeted. We present EviMem, combining IRIS (Iterative Retrieval via Insufficiency Signals), a closed-loop framework that detects evidence gaps through sufficiency evaluation, diagnoses what is missing, and drives targeted query refinement, with LaceMem (Layered Architecture for Conversational Evidence Memory), a coarse-to-fine memory hierarchy supporting fine-grained gap diagnosis. On LoCoMo, EviMem improves Judge Accuracy over MIRIX on temporal (73.3% to 81.6%) and multi-hop (65.9% to 85.2%) questions at 4.5x lower latency. Code: https://github.com/AIGeeksGroup/EviMem.

#### 1. 核心问题与整体含义（研究动机和背景）
在长期对话系统中，大语言模型（LLM）需要处理跨越多个会话的交互历史。现有的检索增强生成（RAG）系统在处理简单事实查询时表现尚可，但在面对**时序推理**（如“某人何时改变了计划？”）或**多跳合成**（需要整合分散在不同会话且无关键词重叠的证据）时经常失败。
*   **研究动机**：传统检索是“开环”的单次过程，无法根据检索结果调整策略。现有的迭代检索方法（如 FLARE 或 Self-RAG）虽然引入了反馈，但主要关注生成内容的置信度或单个文档的质量，**缺乏对“累积证据集”整体充分性的显式诊断**。
*   **核心目标**：通过显式诊断“证据缺口”（即当前已检索到的信息还缺少什么），驱动有针对性的迭代查询优化。

#### 2. 方法论：核心思想与关键技术
EviMem 由两个相互协作的组件构成：

*   **LaceMem（分层记忆架构）**：
    *   **Index 层（索引层）**：将对话转换为原子语义三元组（主-谓-宾-时间），支持精确的语义搜索。
    *   **Edge 层（边缘层）**：通过同源连接和语义相似度建立稀疏图，支持跨会话的多跳扩展。
    *   **Raw 层（原始层）**：保留原始对话文本，确保生成答案时的忠实度。
*   **IRIS（基于不足信号的迭代检索）**：
    *   **双路径检索**：每轮迭代同时运行“锚点路径”（原始问题）和“优化路径”（基于诊断生成的针对性查询），防止语义漂移。
    *   **三层充分性评估**：LLM 将累积证据分为 **EXACT**（直接答案）、**INFERRABLE**（可推断）和 **PARTIAL**（部分相关但不足）。
    *   **证据缺口诊断**：如果评估为不充分，LLM 会生成自然语言描述（诊断报告），明确指出缺失的信息。
    *   **实体跟踪与置信度校准**：监控每个实体的证据覆盖率，防止因某个关键实体信息缺失而导致的误判。
    *   **算法流程**：预处理 -> 迭代检索（检索-扩展-评估-诊断-优化） -> 终止判断 -> 答案生成（或在证据不足时拒绝回答）。

*   **闭环反馈机制**：将“诊断缺失什么”作为核心驱动力，使检索过程具有极强的目的性。
