## 摘要
大语言模型（LLMs）在复杂推理方面表现出色，但其效率受到长思维轨迹激增的认知开销限制。在本文中，我们提出了 LightThinker，这是一种使 LLMs 能够将中间思维动态压缩为紧凑语义表示的方法。然而，静态压缩在处理复杂推理时往往面临困难，因为中间细节的不可逆损失可能导致逻辑瓶颈。为了解决这一问题，我们将该框架演进为 LightThinker++，引入了显式自适应内存管理。该范式通过整合显式内存原语转向行为级管理，并由专门的轨迹合成流水线支持，以训练有目的的内存调度。广泛的实验证明了该框架在三个维度上的通用性：(1) LightThinker 在精度损失极小的情况下，将峰值 token 使用量减少了 70%，推理时间缩短了 26%；(2) 在标准推理中，LightThinker++ 在相同上下文预算下将峰值 token 使用量削减了 69.9%，同时在追求最大性能时获得了 +2.42% 的准确率提升；(3) 最值得注意的是，在长程智能体任务中，它在超过 80 轮的运行中保持了稳定的资源占用（减少了 60%-70%），在不同复杂场景下实现了 14.8% 的平均性能提升。总的来说，我们的工作为在最小开销下维持长程深度 LLM 推理提供了一个可扩展的方向。

## Abstract
Large language models (LLMs) excel at complex reasoning, yet their efficiency is limited by the surging cognitive overhead of long thought traces. In this paper, we propose LightThinker, a method that enables LLMs to dynamically compress intermediate thoughts into compact semantic representations. However, static compression often struggles with complex reasoning where the irreversible loss of intermediate details can lead to logical bottlenecks. To address this, we evolve the framework into LightThinker++, introducing Explicit Adaptive Memory Management. This paradigm shifts to behavioral-level management by incorporating explicit memory primitives, supported by a specialized trajectory synthesis pipeline to train purposeful memory scheduling. Extensive experiments demonstrate the framework's versatility across three dimensions. (1) LightThinker reduces peak token usage by 70% and inference time by 26% with minimal accuracy loss. (2) In standard reasoning, LightThinker++ slashes peak token usage by 69.9% while yielding a +2.42% accuracy gain under the same context budget for maximum performance. (3) Most notably, in long-horizon agentic tasks, it maintains a stable footprint beyond 80 rounds (a 60%-70% reduction), achieving an average performance gain of 14.8% across different complex scenarios. Overall, our work provides a scalable direction for sustaining deep LLM reasoning over extended horizons with minimal overhead.

#### 1. 核心问题与整体含义（研究动机和背景）
随着大语言模型（LLM）向“慢思考”（如 OpenAI o1 模式）演进，模型通过多步推理、反思和纠错显著提升了复杂问题的解决能力。然而，这种性能提升伴随着巨大的代价：
*   **计算与存储开销**：冗长的思维链（CoT）导致 KV Cache 线性增长，注意力机制计算量呈平方级增加。
*   **推理瓶颈**：长上下文不仅占用大量显存，还增加了推理延迟，限制了模型在长程任务（如深度研究智能体）中的实用性。
*   **认知冗余**：人类在思考时并不会在工作记忆中保留每一个字，而是存储关键结论。论文旨在模拟这种“认知经济”，通过压缩和管理思维轨迹来提升 LLM 的推理效率。

#### 2. 方法论：核心思想与关键技术
论文提出了两个阶段的演进框架：

*   **LightThinker（隐式思维压缩）**：
    *   **核心思想**：利用“信息瓶颈”原理，将冗长的思维段落压缩为极少数的隐藏状态（Gist Tokens）。
    *   **技术细节**：引入特殊的**基于思维的注意力掩码（Thought-based Attention Mask）**。在训练时，强制模型将当前思维段落的信息汇总到几个特定的 `[c]` token 中，随后的推理仅能访问这些压缩后的 token，而不能访问原始思维文本。
*   **LightThinker++（显式自适应内存管理）**：
    *   **核心思想**：将内存管理从表示层提升到行为层，引入显式的内存操作原语。
    *   **关键原语**：
        *   `commit`：将当前详细推理归档为语义摘要（Summary）。
        *   `expand`：当遇到逻辑瓶颈时，重新展开某个历史步骤的原始细节。
        *   `fold`：使用完细节后将其重新折叠，保持上下文整洁。
    *   **轨迹合成**：开发了一套环境感知的专家轨迹合成流水线，通过多智能体协作生成包含“思考-行动-观察-内存管理”的训练数据。

#### 6. 主要结论与发现
*   **极高的压缩比**：LightThinker 在 Qwen 模型上减少了 **70% 的峰值 Token 使用量**，并缩短了 **26% 的推理时间**，准确率损失仅约 1%。
*   **性能反超**：在复杂推理（如 GPQA）中，LightThinker++ 在相同上下文预算下比 Vanilla 基准提升了 **2.42% 的准确率**。这表明“去噪”后的紧凑上下文比冗长原始上下文更有利于模型聚焦逻辑。
*   **长程稳定性**：在智能体任务中，Vanilla 模型的上下文在 60 轮内会膨胀到 100k Token，而 LightThinker++ 能稳定在 **30k-40k**，且在硬样本上的性能提升高达 **2.51 倍**。

#### 7. 优点（亮点）
*   **解耦推理深度与显存**：通过显式管理，模型不再受物理上下文窗口的死板限制，理论上可以支持无限深度的思考。
*   **可解释性强**：LightThinker++ 的 `expand/fold` 动作让模型的内存调度过程透明化，便于分析模型在何处遇到了逻辑困难。
*   **自适应性**：模型能根据任务难度自主决定压缩频率，简单任务快速通过，复杂任务频繁归档与回溯。

#### 8. 不足与局限
*   **数值敏感度**：案例研究显示，隐式压缩（LightThinker）在处理极度精确的数值计算时仍可能丢失关键数字信息。
*   **数据依赖**：LightThinker++ 的效果高度依赖于高质量“内存管理轨迹”的合成，这种数据的获取成本较高。
*   **Prefill 开销**：由于涉及多轮推理和动作触发，虽然减少了 Decoding 阶段的 KV Cache，但在频繁切换动作时会引入额外的 Prefill（预填充）计算开销。
