## 摘要
用户描述任务的方式与工具文档记录的方式之间存在语义鸿沟。随着 API 生态系统扩展到数万个端点，仅靠初始查询的静态检索无法弥合这一鸿沟：智能体对所需内容的理解在执行过程中不断演化，但其工具集却并非如此。我们介绍了 FitText，这是一个无需训练的框架，通过将检索直接嵌入到智能体的推理循环中，使检索过程动态化。FitText 生成自然语言伪工具描述作为检索探针，利用检索反馈进行迭代优化，并通过随机生成探索多样化的替代方案。模因检索（Memetic Retrieval）在工具记忆的引导下，对候选描述施加演化选择压力，从而避免冗余搜索。在 ToolRet（包含 4 个领域的 4.3 万个工具）上，FitText 将平均检索排名从 8.81 提升至 2.78；在 StableToolBench（包含 16,464 个 API）上，它实现了 0.73 的平均通过率，相比静态查询检索获得了 24 个百分点的绝对提升。这种增益可以迁移到能够胜任语义算子的基础模型上；在较弱的基础模型下，模因演化搜索会发生反转——放大噪声而非提炼信号——这表明模型能力是进行演化工具探索的前提条件。

## Abstract
A semantic gap separates how users describe tasks from how tools are documented. As API ecosystems scale to tens of thousands of endpoints, static retrieval from the initial query alone cannot bridge this gap: the agent's understanding of what it needs evolves during execution, but its tool set does not. We introduce FitText, a training-free framework that makes retrieval dynamic by embedding it directly in the agent's reasoning loop. FitText generates natural-language pseudo-tool descriptions as retrieval probes, refines them iteratively using retrieval feedback, and explores diverse alternatives through stochastic generation. Memetic Retrieval adds evolutionary selection pressure over candidate descriptions, guided by a tool memory that avoids redundant search. On ToolRet (43k tools, 4 domains), FitText improves average retrieval rank from 8.81 to 2.78; on StableToolBench (16,464 APIs), it achieves a 0.73 average pass rate--a 24-point absolute gain over static query retrieval. The gains transfer across base models capable of acting as competent semantic operators; under weaker base models, Memetic's evolutionary search inverts--amplifying noise rather than refining signal--surfacing model capacity as a prerequisite for evolutionary tool exploration.

### 2. 方法论：核心思想与关键技术
FitText 是一个**无需训练（Training-free）**的框架，其核心是将工具检索视为一种在工具生态系统中的“演化搜索”。

*   **伪工具描述（Pseudo-tool descriptions）：** 智能体不直接使用用户查询检索，而是根据当前推理状态生成一个假设的工具功能描述作为“探针”。
*   **四种递进策略：**
    1.  **Single-Pass（单次）：** 生成一个伪描述并检索。
    2.  **Multi-Turn Refinement（多轮优化）：** 利用检索到的工具示例作为反馈，迭代修正伪描述。
    3.  **Scattershot（散射）：** 高温采样生成多个多样化的伪描述，通过投票机制聚合结果。
    4.  **Memetic Retrieval（模因检索）：** 核心算法。引入进化算法，将伪描述视为“模因（Meme）”，通过**交叉（Crossover）**、**变异（Mutation）**和**适应度选择**进行多代演化。
*   **关键机制：**
    *   **适应度函数：** 结合检索质量（余弦相似度）和**工具记忆（Tool Memory）**惩罚。记忆机制记录已探索区域，避免冗余搜索。
    *   **语义算子：** 利用 LLM 在语义层面执行交叉和变异，而非简单的 Token 级扰动。

### 6. 主要结论与发现

*   **瓶颈转移：** 实验发现**检索质量而非规划能力**是当前智能体性能的绑定约束。提升检索侧的模型能力比提升规划侧更有利于端到端成功。

### 7. 优点
*   **动态自适应：** 解决了静态检索无法应对任务中途需求变化的问题。
*   **无需训练：** 模块化设计，可直接作为插件应用于任何现有的 LLM 智能体，无需昂贵的微调。
*   **弥合语义鸿沟：** 通过伪描述将检索空间从“查询-文档”对齐到“文档-文档”，显著提升了召回精度。
*   **演化鲁棒性：** 模因检索通过群体探索避免了陷入局部最优。

*   **推理成本：** 模因检索涉及多代演化和多次 LLM 调用，虽然作者声称成本可摊销，但在实时性要求极高的场景下可能存在延迟和 Token 成本压力。
*   **模型依赖性：** 该框架对基础模型的语义理解能力高度敏感，弱模型无法发挥其优势，限制了其在低端硬件或小模型上的应用。
*   **随机性风险：** 演化过程具有随机性，虽然有投票机制，但在极端情况下可能导致输出的不稳定。
