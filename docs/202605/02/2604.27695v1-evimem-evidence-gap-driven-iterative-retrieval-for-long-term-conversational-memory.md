## 摘要
长期对话记忆需要检索分散在多个会话中的证据，然而单次检索在处理时间相关和多跳问题时往往会失败。现有的迭代方法通过生成的内容或文档级信号来优化查询，但没有一种方法能明确诊断证据鸿沟（即累积检索集中缺失的内容），导致查询优化缺乏针对性。我们提出了 EviMem，它结合了 IRIS（通过不足信号进行迭代检索）和 LaceMem（对话证据记忆的分层架构）。IRIS 是一个闭环框架，通过充分性评估检测证据鸿沟，诊断缺失内容并驱动有针对性的查询优化；LaceMem 是一种由粗到精的记忆层次结构，支持细粒度的鸿沟诊断。在 LoCoMo 数据集上，EviMem 在时间相关问题（从 73.3% 提升至 81.6%）和多跳问题（从 65.9% 提升至 85.2%）上的评判准确率（Judge Accuracy）均优于 MIRIX，且延迟降低了 4.5 倍。代码：https://github.com/AIGeeksGroup/EviMem。

## Abstract
Long-term conversational memory requires retrieving evidence scattered across multiple sessions, yet single-pass retrieval fails on temporal and multi-hop questions. Existing iterative methods refine queries via generated content or document-level signals, but none explicitly diagnoses the evidence gap, namely what is missing from the accumulated retrieval set, leaving query refinement untargeted. We present EviMem, combining IRIS (Iterative Retrieval via Insufficiency Signals), a closed-loop framework that detects evidence gaps through sufficiency evaluation, diagnoses what is missing, and drives targeted query refinement, with LaceMem (Layered Architecture for Conversational Evidence Memory), a coarse-to-fine memory hierarchy supporting fine-grained gap diagnosis. On LoCoMo, EviMem improves Judge Accuracy over MIRIX on temporal (73.3% to 81.6%) and multi-hop (65.9% to 85.2%) questions at 4.5x lower latency. Code: https://github.com/AIGeeksGroup/EviMem.

### 1. 核心问题与整体含义（研究动机和背景）
在长期对话场景中，大语言模型（LLM）需要处理跨越多个会话的历史信息。现有的检索增强生成（RAG）系统在处理简单事实查询时表现尚可，但在面对**时序推理**（如“某人何时改变了计划”）或**多跳合成**（需要整合分散在不同会话中的事实）时经常失败。
*   **研究动机**：传统的检索是“开环”的单次过程，无法根据检索结果调整策略。现有的迭代检索方法（如 FLARE 或 Self-RAG）虽然引入了反馈，但主要关注生成内容的置信度或单个文档的质量，**缺乏对“证据集整体充分性”的显式诊断**。
*   **核心目标**：通过诊断“证据鸿沟”（即当前已检索到的证据还缺少什么），实现有针对性的迭代检索。

### 2. 方法论：核心思想与关键技术
EviMem 由两个相互协作的组件构成：**IRIS**（检索逻辑）和 **LaceMem**（存储架构）。

#### **LaceMem：分层记忆架构**
为了支持细粒度的证据诊断，记忆被组织为三层：
*   **Index 层（索引层）**：将对话转换为原子语义三元组（主语、谓语、宾语、时间），支持快速语义搜索。
*   **Edge 层（边缘层）**：通过同源连接和语义相似度建立图连接，支持多跳扩展。
*   **Raw 层（原始层）**：保留原始对话文本，确保生成的答案有据可查。

#### **IRIS：证据鸿沟驱动的迭代检索**
这是一个闭环反馈框架，其算法流程如下：
1.  **双路径检索**：同时运行“锚点路径”（基于原问题，保证覆盖面）和“优化路径”（基于诊断后的新查询，针对性补全）。
2.  **实体追踪**：监控每个关键实体的相关事实数量，防止因某个实体信息缺失导致的推理失败。
3.  **充分性评估（核心）**：LLM 将累积证据分为三个层级：
    *   **EXACT**：已有直接答案。
    *   **INFERRABLE**：有足够线索进行合理推理。
    *   **PARTIAL**：相关但不足。
4.  **鸿沟诊断与查询优化**：如果证据不足，LLM 会生成一段自然语言描述（诊断报告），说明缺失的具体信息，并据此生成下一轮的优化查询。
5.  **自适应生成**：根据证据层级调整生成策略，若最终仍无证据则选择“拒绝回答”以减少幻觉。

*   **数据集**：使用 **LoCoMo** 基准测试，这是评估长期对话记忆的标准数据集，包含 10 组超长对话（平均 1.6 万 token），涵盖五类问题：单跳、多跳、时序、开放域、对抗性问题。
*   **对比方法**：
    *   **Single-pass**：基于 LaceMem 的单次检索基准。
    *   **MIRIX**：当前最先进的基于多智能体协作的记忆系统（主要竞争对手）。
*   **评估指标**：G-EVAL（GPT-4 评分）、Judge Accuracy（法官准确率）、F1、ROUGE-L 和 BERTScore。

### 6. 主要结论与发现
*   **性能领先**：EviMem 在时序推理（81.6%）和多跳问题（85.2%）上的准确率大幅超过 MIRIX。
*   **效率极高**：平均响应延迟为 **9.54s**，而 MIRIX 为 **42.71s**，实现了 **4.5 倍的加速**。这得益于其自适应循环（简单问题一轮结束）而非复杂的多智能体编排。
*   **诊断的价值**：实验证明，基于“缺失信息诊断”生成的查询比盲目重写查询在召回率和排序质量上都有显著提升。

### 7. 优点
*   **闭环诊断机制**：将“证据充分性”显式化，解决了以往 RAG 系统盲目迭代的问题。
*   **架构与逻辑协同**：LaceMem 的原子化存储与 IRIS 的细粒度诊断完美契合。
*   **高性价比**：通过轻量级模型（GPT-4o-mini）处理中间步骤，在提升性能的同时大幅降低了延迟和成本。
*   **可靠性**：引入了“拒绝回答”机制和置信度校准，显著降低了长文本场景下的幻觉风险。

### 8. 不足与局限
*   **离线构建限制**：目前的 LaceMem 索引是基于对话快照离线构建的，尚未在论文中展示如何实时、流式地处理动态涌入的新对话。
*   **API 依赖**：系统性能高度依赖于底层 LLM 的推理和诊断能力，在极小规模或能力较弱的模型上可能效果受限。
*   **检索开销**：虽然比多智能体系统快，但对于极简单的查询，多次 LLM 调用（评估+优化）仍比单次检索昂贵。
