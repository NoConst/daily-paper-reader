## 摘要
用户在向基于大语言模型（LLM）的智能体发出请求时，往往会省略关键细节，导致工具使用的输入信息不足。这对工具增强型智能体提出了根本性挑战，因为 API 执行通常需要完整的参数，从而凸显了个性化工具调用的必要性。为了研究这一问题，我们引入了 MPT，这是一个包含 265 个多会话对话的基准测试，涵盖了三个挑战：偏好召回、偏好归纳和偏好迁移。我们还提出了 PRefine，这是一种测试时内存增强方法，将用户偏好表示为不断演化的假设。通过“生成-验证-细化”循环，它从历史记录中提取可重用的约束，在提高工具调用准确性的同时，仅消耗全历史提示词所需 token 数量的 1.24%。这些结果表明，智能体系统中稳健的个性化取决于能够捕捉用户选择背后原因的记忆，而不仅仅是选择本身。

## Abstract
Users often omit essential details in their requests to LLM-based agents, resulting in under-specified inputs for tool use. This poses a fundamental challenge for tool-augmented agents, as API execution typically requires complete arguments, highlighting the need for personalized tool calling. To study this problem, we introduce MPT, a benchmark comprising 265 multi-session dialogues that cover three challenges: Preference Recall, Preference Induction, and Preference Transfer. We also propose PRefine, a test-time memory-augmented method that represents user preferences as evolving hypotheses. Through a generate--verify--refine loop, it extracts reusable constraints from history and improves tool-calling accuracy while using only 1.24% of the tokens required by full-history prompting. These results indicate that robust personalization in agentic systems depends on memory that captures the reasons behind user choices, not just the choices themselves.

### 2. 方法论：PRefine
论文提出了 **PRefine**，一种基于记忆增强的测试时（test-time）方法，其核心思想是将用户偏好视为“不断演化的假设”。
*   **核心流程：生成-验证-细化（Generate-Verify-Refine）循环**：
    1.  **生成（Generate）**：根据当前会话、执行的 API 调用和先验记忆，生成候选的偏好假设（如“用户倾向于选择廉价选项”）。
    2.  **验证（Verify）**：由验证器根据四个维度评估假设：**证据支持**（是否有多次记录支持）、**抽象质量**（是否超越了简单的槽位复述）、**可操作性**（是否能指导未来决策）、**时间一致性**（是否符合最新行为）。
    3.  **细化（Refine）**：若验证未通过，根据反馈修正假设，剔除过度特定或无证据的描述。
*   **关键技术细节**：
    *   **模式无关记忆（Schema-Agnostic Memory）**：记忆存储的是抽象的文本约束（如“预算敏感”），而非特定 API 的参数名。这使得记忆在 API 架构发生变化（Dynamic Schema）时依然有效。
    *   **轻量化**：仅存储精炼后的偏好假设，而非原始对话全文。

    1.  **偏好召回（Preference Recall）**：直接复用同一领域内的历史选择。
    2.  **偏好归纳（Preference Induction）**：聚合跨会话的证据来推断潜在偏好。
    3.  **偏好迁移（Preference Transfer）**：将一个领域的偏好（如租车选便宜的）应用到新领域（如订机票选经济舱）。

### 6. 主要结论与发现
*   **性能提升**：PRefine 在偏好归纳和迁移任务上显著优于全历史提示（Base Prompting）和 RAG。
*   **极高的效率**：PRefine 仅使用了全历史提示所需 Token 量的 **1.24%**，且记忆长度不随会话增加而线性膨胀。
*   **推理能力差异**：现有的模型在“偏好召回”上表现尚可，但在需要抽象推理的“归纳”和“迁移”任务上存在明显短板，而 PRefine 弥补了这一差距。

### 7. 优点
*   **抽象化建模**：不只是存储事实，而是存储“行为背后的原因”，更符合人类的个性化逻辑。
*   **架构鲁棒性**：支持动态 API 架构，具有很强的实际应用潜力。
*   **资源友好**：极大地降低了长对话背景下的上下文窗口压力和推理成本。

*   **保守性风险**：实验发现某些模型在应用 PRefine 后变得过于保守，可能会漏掉一些必要的参数生成。
