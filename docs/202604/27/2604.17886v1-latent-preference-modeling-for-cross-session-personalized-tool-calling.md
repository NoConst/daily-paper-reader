## 摘要
用户在向基于大语言模型（LLM）的智能体提出请求时，往往会省略关键细节，导致工具使用的输入信息不足。这对工具增强型智能体提出了根本性挑战，因为 API 执行通常需要完整的参数，从而凸显了个性化工具调用的必要性。为了研究这一问题，我们引入了 MPT，这是一个包含 265 个多会话对话的基准测试，涵盖了三个挑战：偏好召回、偏好归纳和偏好迁移。我们还提出了 PRefine，这是一种测试时内存增强方法，将用户偏好表示为不断演化的假设。通过“生成-验证-细化”循环，它从历史记录中提取可重用的约束，在提高工具调用准确性的同时，仅消耗全历史提示所需 token 数量的 1.24%。这些结果表明，智能体系统中稳健的个性化取决于能够捕捉用户选择背后原因（而非仅仅是选择本身）的记忆。

## Abstract
Users often omit essential details in their requests to LLM-based agents, resulting in under-specified inputs for tool use. This poses a fundamental challenge for tool-augmented agents, as API execution typically requires complete arguments, highlighting the need for personalized tool calling. To study this problem, we introduce MPT, a benchmark comprising 265 multi-session dialogues that cover three challenges: Preference Recall, Preference Induction, and Preference Transfer. We also propose PRefine, a test-time memory-augmented method that represents user preferences as evolving hypotheses. Through a generate--verify--refine loop, it extracts reusable constraints from history and improves tool-calling accuracy while using only 1.24% of the tokens required by full-history prompting. These results indicate that robust personalization in agentic systems depends on memory that captures the reasons behind user choices, not just the choices themselves.

*   **研究动机**：个性化工具调用不应只是简单的“动作重用”（检索过去做了什么），而应是一个“推理问题”，即从跨会话的噪声数据中抽象出用户行为背后的潜在约束（如：该用户总是选择最便宜的选项）。

### 2. 方法论：PRefine
*   **核心思想**：提出一种名为 **PRefine** 的测试时记忆增强方法，将用户偏好视为一种“不断演化的假设”，而非静态的事实。
*   **关键技术细节**：
    *   **生成-验证-细化循环（Generate-Verify-Refine Loop）**：
        1.  **生成（Generate）**：根据当前会话和历史记忆，生成关于用户偏好的抽象假设。
        2.  **验证（Verify）**：由验证器根据四个维度（证据支持、抽象质量、可操作性、时间一致性）评估假设。
        3.  **细化（Refine）**：若验证失败，根据反馈修正假设，剔除过度具体或无证据的描述。
    *   **架构无关的偏好记忆**：存储的是抽象的文本约束（如“用户偏好经济实惠的旅行”），而非特定 API 的参数名。这使得记忆在 API 架构发生变化（Schema 变更）时依然有效。
    *   **推理过程**：在推理时，模型仅需携带这段精简的偏好描述，而非全部历史对话。

### 3. 实验设计

        2.  **偏好归纳 (Induction)**：聚合跨任务、跨领域的证据。
        3.  **偏好迁移 (Transfer)**：将一个领域的偏好应用到从未见过的领域。

*   **推理能力差异**：现有的 RAG 和记忆方法在“偏好召回”上表现尚可，但在需要抽象推理的“归纳”和“迁移”任务上表现糟糕。

### 7. 优点（亮点）
*   **分类科学**：首次将个性化偏好细分为召回、归纳、迁移三种类型，填补了评估空白。
*   **架构无关性**：PRefine 存储的是自然语言约束，这解决了 API 升级或更换时记忆失效的痛点。
*   **自修正机制**：通过验证循环确保了记忆的质量，避免了将偶然的行为误认为长期偏好。

### 8. 不足与局限
*   **保守性风险**：实验发现某些模型（如 R1 蒸馏版）在应用 PRefine 后变得过于保守，生成的参数数量减少，可能导致召回率下降。
*   **偏好演化处理**：虽然提到了时间一致性，但对于用户偏好发生剧烈转变（如从“穷游”突然变为“奢华游”）的复杂动态处理仍有提升空间。
*   **依赖生成质量**：PRefine 的效果高度依赖于“生成器”和“验证器”的推理能力，在较弱的小模型上效果受限。
