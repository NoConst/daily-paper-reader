## 摘要
尽管过去十年中出现了日益复杂的有状态记忆架构，但在受监管领域（如核保、理赔裁定、税务稽查）中，长程决策智能体的企业级部署仍以检索增强流水线为主。我们认为这反映了一个隐含的需求：受监管的部署高度依赖四项系统属性（确定性回放、可审计的依据、多租户隔离以及用于水平扩展的无状态性），而有状态架构在设计上就违背了这些属性。我们提出了确定性投影记忆（Deterministic Projection Memory, DPM）：它由一个仅追加的事件日志和决策时的一次任务条件投影组成。在三种记忆预算下的十个受监管决策案例中，DPM 在预算充裕时与基于摘要的记忆表现相当，而在预算受限时表现显著优于后者：在 20 倍压缩率下，DPM 将事实精准度提高了 0.52（Cohen's h=1.17, p=0.0014），推理连贯性提高了 0.53（h=1.13, p=0.0034），配对置换检验，n=10。此外，在预算受限时，DPM 的速度快 7-15 倍，在决策时仅需调用一次大语言模型（LLM），而非 N 次。一项在零温度下对每个案例进行 10 次回放的确定性研究表明，两种架构都继承了残余的 API 级非确定性，但这种不对称性是结构性的：DPM 仅暴露一次非确定性调用，而摘要法则暴露了 N 次复合调用。审计面也遵循同样的“1 对 N”模式：在 LongHorizon-Bench 上，DPM 每次决策记录两次 LLM 调用，而摘要法则记录 83-97 次。最后，我们提出了 TAMS（一种供从业者选择架构的启发式方法），并对企业运行条件下有状态记忆的失效进行了分析。本文的贡献在于论证了无状态性是解释企业偏好性能较弱但可回放的检索流水线的核心属性，并证明了 DPM 可以在不付出检索所带来的决策性能代价的情况下实现这一属性。

## Abstract
Enterprise deployment of long-horizon decision agents in regulated domains (underwriting, claims adjudication, tax examination) is dominated by retrieval-augmented pipelines despite a decade of increasingly sophisticated stateful memory architectures. We argue this reflects a hidden requirement: regulated deployment is load-bearing on four systems properties (deterministic replay, auditable rationale, multi-tenant isolation, statelessness for horizontal scale), and stateful architectures violate them by construction. We propose Deterministic Projection Memory (DPM): an append-only event log plus one task-conditioned projection at decision time. On ten regulated decisioning cases at three memory budgets, DPM matches summarization-based memory at generous budgets and substantially outperforms it when the budget binds: at a 20x compression ratio, DPM improves factual precision by +0.52 (Cohen's h=1.17, p=0.0014) and reasoning coherence by +0.53 (h=1.13, p=0.0034), paired permutation, n=10. DPM is additionally 7-15x faster at binding budgets, making one LLM call at decision time instead of N. A determinism study of 10 replays per case at temperature zero shows both architectures inherit residual API-level nondeterminism, but the asymmetry is structural: DPM exposes one nondeterministic call; summarization exposes N compounding calls. The audit surface follows the same one-versus-N pattern: DPM logs two LLM calls per decision while summarization logs 83-97 on LongHorizon-Bench. We conclude with TAMS, a practitioner heuristic for architecture selection, and a failure analysis of stateful memory under enterprise operating conditions. The contribution is the argument that statelessness is the load-bearing property explaining enterprise's preference for weaker but replayable retrieval pipelines, and that DPM demonstrates this property is attainable without the decisioning penalty retrieval pays.

### 2. 方法论：确定性投影记忆 (DPM)
论文提出了 **确定性投影记忆 (Deterministic Projection Memory, DPM)**，其核心思想是将记忆处理从“在线更新”转变为“决策时投影”：
*   **追加式事件日志 (Append-only Event Log)**：智能体运行期间，所有原始事件（文档块、工具输出、推理步骤）仅按顺序写入不可变的日志，不进行任何中间摘要或编辑。
*   **任务条件投影 (Task-conditioned Projection)**：仅在最终决策时刻，将整个日志、任务说明和记忆预算（字符限制）一次性输入给 LLM。
*   **结构化提取**：LLM 在零温度（Temperature 0）下执行单次调用，生成包含“事实（Facts）”、“推理（Reasoning）”和“合规说明（Compliance Notes）”的结构化视图。
*   **数学特性**：将记忆视为日志的纯函数 $M = \pi(E, T, B)$。相比有状态架构需要 $N$ 次 LLM 调用（每步更新一次），DPM 仅需 1 次调用，从而将非确定性风险和审计面从 $N$ 降至 1。

### 3. 实验设计
*   **数据集/场景**：使用 **LongHorizon-Bench**，包含两个受监管领域：抵押贷款资格审查（依据 ECOA/Reg B 法规）和保险理赔裁定。
*   **实验设置**：10 个案例，每个案例包含 82-96 个离散事件，轨迹长度约 2.6k-2.8k 字符。
*   **对比方法 (Benchmark)**：
    *   **Summ-only**：增量摘要法（最强的有状态基准），每步更新摘要。
    *   **DPM**：本文提出的无状态投影法。
*   **评估维度**：事实精准度 (FRP)、推理连贯性 (RCS)、决策准确度 (EDA) 和合规重建 (CRR)。
*   **预算变量**：设置了紧凑（20倍压缩）、中等（5倍压缩）和宽松（2倍压缩）三种记忆预算。

*   **模型使用**：所有实验（包括智能体决策和 LLM 裁判评分）均使用 **Claude-Haiku-4.5-20251001**。

*   **成本与效率**：DPM 在决策时比有状态基准快 **7-15 倍**，且成本大幅降低，因为它避免了轨迹中频繁的中间 LLM 调用。

### 5. 实验数量与充分性
*   **实验规模**：每个配置下进行了 10 组案例实验（n=10），并进行了 10,000 次重采样的配对置换检验。
*   **确定性研究**：对每个案例进行了 10 次重复回放实验，以测量字节级的漂移。
*   **充分性评价**：实验设计较为严谨，涵盖了不同压缩比的对比和确定性分析。虽然样本量（10个案例）在机器学习领域不算大，但考虑到长程决策案例构建的复杂性和统计检验的使用，其结论具有较强的说服力。

### 6. 主要结论与发现
*   **性能优势**：在记忆预算充足时，DPM 与摘要法表现相当；但在**预算紧凑（20倍压缩）**时，DPM 显著优于摘要法（事实精准度提升 0.52，推理连贯性提升 0.53）。
*   **结构性优势**：摘要法存在“误差复合效应”，每一步摘要都会丢失微小信息，最终导致决策失效；DPM 通过保留原始日志避开了这一问题。
*   **确定性**：虽然 API 级仍存在残余的非确定性，但 DPM 将审计面从 $N$ 个调用缩减为 1 个，显著降低了企业运维和合规审计的负担。
*   **TAMS 启发式规则**：提出从业者应根据“合规需求”和“压缩比”选择架构，当压缩比超过 10 倍或有合规要求时，应首选 DPM。

### 7. 优点
*   **系统思维**：不仅关注算法准确率，更从企业工程（确定性、多租户、扩展性）角度审视记忆架构。
*   **架构简洁**：DPM 极其简单（日志+单次投影），却解决了复杂有状态架构难以解决的合规痛点。
*   **高效率**：大幅减少了 LLM 调用次数，显著提升了推理速度并降低了成本。

### 8. 不足与局限
*   **上下文窗口限制**：DPM 依赖于单次调用处理全量日志，受限于 LLM 的上下文窗口（当前约为 100k-1M 字符）。对于超长轨迹（如超过 100 万字符），仍需引入分层投影，这会重新引入部分有状态架构的弊端。
*   **模型依赖**：实验仅基于 Claude 系列模型，在更弱或更小的模型上，单次投影的提取能力可能会下降。
*   **领域覆盖**：虽然涵盖了金融和保险，但尚未在医疗（HIPAA合规）或法律等其他严苛领域验证。
*   **非绝对确定性**：论文承认 DPM 无法解决推理引擎底层的随机性，只能通过减少调用次数来缓解。
