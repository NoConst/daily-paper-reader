## 摘要
长程对话智能体依赖于具有日益复杂检索机制的记忆系统。然而，检索到的片段通常以非结构化文本的形式提供给语言模型，缺乏复杂推理所必需的关系、时间和主题结构。为了弥补这一推理差距，我们引入了 GRAVITY（通过注入拓扑记忆实现的生成时关系锚定），这是一个即插即用的结构化记忆模块。GRAVITY 从原始对话话语中提取三种互补的知识表示：基于关系图的实体概况、链接到因果轨迹的时间事件元组，以及跨会话的主题摘要。在生成时，它将这些表示作为结构化锚定上下文注入到宿主系统的提示词中。这种方法有效地将分散的证据合成为连贯且与查询相关的上下文，而无需对宿主模型进行任何架构修改。在 LongMemEval 和 LoCoMo 基准测试中对五种不同记忆系统的广泛评估证明了我们方法的有效性。平均而言，GRAVITY 将 LLM 评判准确率提高了 7.5%--10.1%。收益与基准强度呈负相关：最弱的宿主提高了 12.2%，而最强的宿主仍获得了 3.8%--5.7% 的提升。这些发现确立了结构化上下文锚定作为一种广泛有效、架构无关的长程对话记忆增强范式。

## Abstract
Long-horizon conversational agents rely on memory systems with increasingly sophisticated retrieval mechanisms. However, retrieved fragments are typically fed to the language model as unstructured text, lacking the relational, temporal, and thematic structures essential for complex reasoning. To bridge this reasoning gap, we introduce GRAVITY (\textbf{G}eneration-time \textbf{R}elational \textbf{A}nchoring \textbf{V}ia \textbf{I}njected \textbf{T}opological Memor\textbf{Y}), a plug-and-play structured memory module. GRAVITY extracts three complementary knowledge representations from raw conversational utterances: entity profiles grounded in relational graphs, temporal event tuples linked into causal traces, and cross-session topic summaries. At generation time, it injects these representations into the host system's prompt as structured anchoring contexts. This approach effectively synthesizes scattered evidence into a coherent, query-relevant context without requiring any architectural modifications to the host model. Extensive evaluations across five diverse memory systems on the LongMemEval and LoCoMo benchmarks demonstrate the efficacy of our approach. On average, GRAVITY improves LLM-judge accuracy by 7.5--10.1%. Gains are inversely correlated with baseline strength: the weakest host improves by 12.2% while the strongest still gains 3.8--5.7%. These findings establish structured context anchoring as a broadly effective, architecture-agnostic augmentation paradigm for long-horizon conversational memory.

### 1. 核心问题与研究动机
*   **核心问题**：现有的长程对话记忆系统（如基于向量检索的系统）在处理跨越数百个会话的对话时，往往只能检索到“碎片化”的原始文本。
*   **研究动机**：作者发现检索和生成之间存在“推理差距”（Reasoning Gap）。即使检索到了正确的证据，由于缺乏显式的**关系（Relational）**、**时序（Temporal）**和**主题（Thematic）**结构，生成模型仍难以进行复杂的逻辑推理。
*   **目标**：在不修改现有记忆系统架构的前提下，通过注入结构化的“锚点”上下文来弥补这一差距。

### 2. 方法论
GRAVITY 的核心思想是**“生成时结构化锚定”**，分为两个阶段：
*   **离线构建阶段（Build Phase）**：从原始对话中提取三种互补的结构化表示：
    1.  **实体锚点（Entity Anchors）**：构建动态实体画像，包含属性、与其他实体的关系图以及状态演变时间线。
    2.  **事件锚点（Event Anchors）**：提取 4W1O（Who, What, When, Where, Outcome）元组，并将相关事件链接成因果/时序轨迹（Traces）。
    3.  **主题锚点（Topic Anchors）**：进行跨会话的主题聚合，生成宏观叙事摘要，捕捉碎片化记忆无法表达的长程弧线。
*   **在线推理阶段（Inference Phase）**：
    1.  **锚点检索与重排**：根据查询（Query）检索最相关的 Top-K 锚点，并引入时序保护机制。
    2.  **查询扩展**：利用锚点内容生成扩展查询，帮助宿主系统检索更广泛的原始记忆。
    3.  **上下文注入**：将结构化锚点作为补充信息注入提示词（Prompt），引导模型推理。

*   **对比方法（宿主系统）**：将 GRAVITY 插入 5 种不同架构的 SOTA 记忆系统：
    *   **LightMem**（层次化压缩）、**A-Mem**（原子笔记组织）、**Mem0**（向量+实体图混合）、**LiCoMemory**（认知图）、**ZEP**（时序知识图）。

*   **实验规模**：在 2 个基准数据集上对 5 种主流架构进行了交叉验证，实验覆盖面广。
*   **消融实验**：分别验证了实体、事件、主题三个模块的独立贡献，并对比了“非结构化摘要”基准，证明了“结构化”本身的价值。
*   **Oracle 实验**：通过“完美检索”设定下的对比，量化证明了推理差距的存在以及 GRAVITY 的修复作用。
*   **客观性**：实验设计考虑了公平性（固定检索条数），并进行了参数敏感性分析（如 Batch Size、Top-K 等），结果具有较强的说服力。

### 6. 主要结论与发现
*   **普适性提升**：GRAVITY 在所有宿主系统上均有显著提升，平均准确率提高 7.5%–10.1%。
*   **补短板效应**：收益与宿主系统本身的强度呈负相关。基准越弱，提升越明显（最高达 13.1%）；即使是最强的系统（LightMem）也有 3.8%–5.7% 的增益。
*   **结构胜过内容**：实验证明，提供结构化的实体/事件/主题信息比提供同等长度的纯文本摘要效果更好。
*   **架构无关**：证明了结构化锚定可以作为一种通用的增强范式，无需触动宿主系统的内部索引或存储逻辑。

### 7. 优点
*   **即插即用**：无需重新训练模型或修改记忆系统的底层架构，集成成本低。
*   **多维建模**：同时捕捉了关系、时序和主题三个关键维度，比单一的知识图谱更全面。
*   **推理引导**：通过将分散的证据预先组织好，降低了 LLM 在生成时的认知负荷。

### 8. 不足与局限
*   **提取阶段的损失**：错误分析显示，约 38% 的失败源于构建阶段的“过度总结”，导致细微事实丢失。
*   **时序精度问题**：在处理模糊的时间引用（如“上周末”）时，提取模块可能产生错误的绝对日期标注。
*   **离线成本**：虽然比某些系统便宜，但对于极长对话，离线提取结构化锚点仍需消耗大量的 LLM API Token。
*   **应用限制**：主要针对长程对话，对于短文本或实时性极高的流式对话，其构建延迟可能成为瓶颈。
