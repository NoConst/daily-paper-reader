## 摘要
用户描述任务的方式与工具文档记录的方式之间存在语义鸿沟。随着 API 生态系统扩展到数万个端点，仅依靠初始查询的静态检索无法弥合这一鸿沟：智能体对所需工具的理解在执行过程中不断演化，但其工具集却并非如此。我们提出了 FitText，这是一个无需训练的框架，通过将检索直接嵌入到智能体的推理循环中，实现了检索的动态化。FitText 生成自然语言伪工具描述作为检索探针，利用检索反馈进行迭代优化，并通过随机生成探索多样化的替代方案。模因检索（Memetic Retrieval）在工具记忆的引导下，对候选描述施加演化选择压力，从而避免冗余搜索。在 ToolRet（包含 4 个领域的 4.3 万个工具）上，FitText 将平均检索排名从 8.81 提升至 2.78；在 StableToolBench（包含 16,464 个 API）上，它实现了 0.73 的平均通过率，相比静态查询检索获得了 24 个百分点的绝对提升。这种增益可以迁移到能够胜任语义算子的基础模型上；在较弱的基础模型下，模因演化搜索会发生反转——放大噪声而非提炼信号——这表明模型能力是进行演化工具探索的前提条件。

## Abstract
A semantic gap separates how users describe tasks from how tools are documented. As API ecosystems scale to tens of thousands of endpoints, static retrieval from the initial query alone cannot bridge this gap: the agent's understanding of what it needs evolves during execution, but its tool set does not. We introduce FitText, a training-free framework that makes retrieval dynamic by embedding it directly in the agent's reasoning loop. FitText generates natural-language pseudo-tool descriptions as retrieval probes, refines them iteratively using retrieval feedback, and explores diverse alternatives through stochastic generation. Memetic Retrieval adds evolutionary selection pressure over candidate descriptions, guided by a tool memory that avoids redundant search. On ToolRet (43k tools, 4 domains), FitText improves average retrieval rank from 8.81 to 2.78; on StableToolBench (16,464 APIs), it achieves a 0.73 average pass rate--a 24-point absolute gain over static query retrieval. The gains transfer across base models capable of acting as competent semantic operators; under weaker base models, Memetic's evolutionary search inverts--amplifying noise rather than refining signal--surfacing model capacity as a prerequisite for evolutionary tool exploration.

*   **核心目标**：将工具检索从“一次性预设”转变为“推理循环中的动态演化”，使智能体能根据当前执行上下文主动发现并优化所需的工具集。

#### 2. 方法论
FitText 是一个无需训练（Training-free）的框架，其核心思想是将工具检索建模为一种**演化搜索过程**：
*   **伪工具描述（Pseudo-tool Descriptions）**：LLM 不直接使用用户查询检索，而是根据任务需求生成一个假设性的“伪工具描述”作为探针，以弥合语义鸿沟。
*   **四种递进策略**：
    1.  **Single-Pass**：生成单个伪描述进行检索。
    2.  **Multi-Turn Refinement**：利用检索到的真实工具作为反馈，迭代修正伪描述。
    3.  **Scattershot**：并行生成多个多样化的伪描述，通过投票机制确定最终工具。
    4.  **模因检索（Memetic Retrieval）**：核心算法。引入进化计算概念，维护一个伪描述种群，通过**交叉（Crossover）**、**变异（Mutation）**和**选择（Selection）**进行多代演化。
*   **关键技术细节**：
    *   **适应度函数（Fitness Function）**：结合检索质量（余弦相似度）和**工具记忆（Tool Memory）**惩罚项。记忆机制记录已探索区域，强制种群向未探索的功能空间演化。
    *   **语义算子**：利用 LLM 作为“语义算子”执行文本层面的交叉和变异，而非简单的 token 扰动。
    *   **分层投票**：对种群结果进行聚合，优先考虑在多个演化分支中共同出现的工具。

#### 6 主要结论与发现

*   **语义对齐**：通过伪工具描述有效解决了用户语言与技术文档之间的术语不一致问题。
*   **记忆引导**：引入工具记忆避免了演化过程中的冗余搜索，提高了探索效率。

*   **推理成本**：模因检索涉及多代演化和多个候选描述的生成，相比单次检索，其 **Token 消耗和响应延迟**显著增加。
*   **模型依赖**：该方法对基础模型的语义理解能力要求极高，在轻量级模型上可能失效。
*   **评估环境**：虽然在模拟环境（StableToolBench）表现优异，但在真实、高动态且带有复杂权限校验的生产 API 环境中的表现仍需验证。
