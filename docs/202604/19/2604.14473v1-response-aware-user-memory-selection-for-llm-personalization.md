## 摘要
大语言模型（LLM）个性化的一种常用方法是在推理时将用户记忆的一个子集整合到提示词中，以引导模型的生成。现有方法主要利用用户记忆项与输入查询之间的相似度来选择这些子集，而忽略了特征实际上如何影响模型的响应分布。我们提出了响应效用优化记忆选择（RUMS），这是一种通过衡量记忆子集与模型输出之间的互信息来选择用户记忆项的新方法，旨在识别那些能够降低响应不确定性并比语义相似度更有效地增强预测确定性的项。我们证明，这种基于信息论的基础能够实现更具原则性的用户记忆选择，与现有最先进方法以及规模大 400 倍的模型相比，其选择结果与人类的选择更加一致。此外，我们还表明，与现有方法相比，使用 RUMS 选择的记忆项能带来更好的响应质量，同时计算成本降低了高达 95%。

## Abstract
A common approach to personalization in large language models (LLMs) is to incorporate a subset of the user memory into the prompt at inference time to guide the model's generation. Existing methods select these subsets primarily using similarity between user memory items and input queries, ignoring how features actually affect the model's response distribution. We propose Response-Utility optimization for Memory Selection (RUMS), a novel method that selects user memory items by measuring the mutual information between a subset of memory and the model's outputs, identifying items that reduce response uncertainty and sharpen predictions beyond semantic similarity. We demonstrate that this information-theoretic foundation enables more principled user memory selection that aligns more closely with human selection compared to state-of-the-art methods, and models $400\times$ larger. Additionally, we show that memory items selected using RUMS result in better response quality compared to existing approaches, while having up to $95\%$ reduction in computational cost.

*   **核心矛盾**：语义相似并不等同于对生成结果有益。有些相似的记忆可能是冗余的，而有些看似不相关的记忆（如用户的职业或偏好风格）却能显著改变模型的响应分布。
*   **研究目标**：提出一种“响应感知”的选择机制，直接衡量记忆项对模型输出分布的影响，从而实现更精准、高效的个性化。

#### 2. 方法论
*   **核心思想**：利用信息论中的**条件互信息（Conditional Mutual Information）**。如果加入某项记忆能显著降低模型生成响应时的不确定性（即降低熵），则认为该记忆具有高“响应效用”。

    *   针对 H2（记忆选择准确性）进行了人工标注对比（n=60，每条由 4 名标注员评估）。
    *   针对 H3（下游生成质量）测试了数百个样本，并使用 GPT-4 作为裁判（LLM-as-a-Judge）计算胜率。
*   **充分性评价**：实验设计较为全面，涵盖了从理论验证、选择准确性到最终生成质量的完整链路。通过消融实验分析了模型大小、采样数量、解码方式等超参数对熵计算的影响，实验结果具有较强的说服力。

#### 6. 主要结论与发现
*   **效用信号有效性**：预测熵的降低确实能有效区分哪些请求需要个性化，哪些不需要。
*   **选择质量**：RUMS 选择的记忆子集比规模大 400 倍的 GPT-4 选得更准，与人类标注的 F1 分数更高。
*   **生成提升**：使用 RUMS 选择的记忆，在 LLaMA-70B 和 GPT-4 上的生成胜率均优于所有基准线。
*   **成本优势**：由于 RUMS 倾向于选择更精简、更有用的记忆，相比“全量加入”的方法，推理成本（Token 数）降低了 **95%**。

#### 7. 优点（亮点）
*   **视角独特**：跳出了传统的“语义匹配”框架，从模型输出分布的改变出发，更符合生成式模型的本质。
*   **理论与实践结合**：既有严谨的信息论证明，又通过知识蒸馏手段解决了实际落地中的计算延迟问题。
*   **鲁棒性强**：在真实世界数据集（WildChat）上表现稳健，证明了其泛化能力。

#### 8. 不足与局限
*   **潜在偏差**：在生成训练数据时，第一步使用了 GPT-4 进行初步筛选（从 50 项减至 10 项），这可能引入了 GPT-4 本身的偏见。
*   **静态假设**：理论证明假设用户偏好是静态的，但在长程对话中，用户意图可能会随模型输出而演变。
*   **模型依赖**：熵的计算依赖于一个参考模型（如 Llama-3-8B），如果参考模型与最终生成模型能力差距过大，效用信号可能失效。
*   **多轮对话覆盖不足**：目前主要集中在单轮指令的个性化，尚未深入探讨复杂多轮对话中的记忆动态更新。
