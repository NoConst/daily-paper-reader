## 摘要
LLM 重排序（LLM-reranking）受限于通过向量相似度检索到的前 k 个文档，这既无法实现查询与文档之间的上下文 Token 交互，也无法捕捉多模态相关性分布。虽然 LLM 查询重构尝试通过生成改进的或额外的查询来提高召回率，但其后续仍采用向量相似度检索。因此，我们提出了 ReBOL 来解决这些前 k 阶段检索失败的问题。ReBOL 1) 利用 LLM 查询重构来初始化关于文档相关性的多模态贝叶斯优化（BO）后验分布，2) 迭代地获取文档批次进行 LLM 查询-文档相关性评分，随后更新后验分布以优化相关性。在探索了查询重构和文档批次多样化技术后，我们在五个 BEIR 数据集上，使用两种 LLM（Gemini-2.5-Flash-Lite, GPT-5.2）将 ReBOL 与 LLM 重排序基准进行了评估。ReBOL 一贯地实现了更高的召回率和具有竞争力的排名，例如在 Robust04 数据集上，与最佳 LLM 重排序器相比，其 recall@100 为 46.5% 对 35.0%，NDCG@10 为 63.6% 对 61.2%。我们还表明，ReBOL 可以达到与 LLM 重排序器相当的延迟。

## Abstract
LLM-reranking is limited by the top-k documents retrieved by vector similarity, which neither enables contextual query-document token interactions nor captures multimodal relevance distributions. While LLM query reformulation attempts to improve recall by generating improved or additional queries, it is still followed by vector similarity retrieval. We thus propose to address these top-k retrieval stage failures by introducing ReBOL, which 1) uses LLM query reformulations to initialize a multimodal Bayesian Optimization (BO) posterior over document relevance, and 2) iteratively acquires document batches for LLM query-document relevance scoring followed by posterior updates to optimize relevance. After exploring query reformulation and document batch diversification techniques, we evaluate ReBOL against LLM reranker baselines on five BEIR datasets and using two LLMs (Gemini-2.5-Flash-Lite, GPT-5.2). ReBOL consistently achieves higher recall and competitive rankings, for example compared to the best LLM reranker on the Robust04 dataset with 46.5% vs. 35.0% recall@100 and 63.6% vs. 61.2% NDCG@10. We also show that ReBOL can achieve comparable latency to LLM rerankers.

#### 1. 核心问题与整体含义（研究动机和背景）
传统的“检索-重排序”流水线（Retrieve-then-Rerank）存在一个根本性的**召回瓶颈**：重排序阶段（如使用 LLM）仅能处理由向量相似度检索出的前 $k$ 个文档。如果相关文档未进入这前 $k$ 个名单，重排序器也无能为力。
*   **向量检索的局限性**：仅依赖编码后的向量点积或余弦相似度，缺乏查询与文档间的 Token 级交互，且其评分函数通常是单模态的，无法捕捉分布在不同语义簇中的多模态相关性。
*   **查询重构的局限性**：虽然 LLM 可以生成多个变体查询来提升召回，但后续依然依赖向量检索，未能从根本上改变检索机制。
*   **研究目标**：提出一种名为 **ReBOL** 的框架，将检索过程建模为**贝叶斯优化（Bayesian Optimization, BO）**任务，通过 LLM 的实时反馈动态引导检索方向，从而突破传统 Top-k 检索的限制。

#### 2. 论文提出的方法论
ReBOL 的核心思想是将文档嵌入空间视为搜索域，将 LLM 的相关性评分视为待优化的黑盒函数。
*   **多模态高斯过程（GP）先验**：使用带有非线性核函数（如 RBF）的 GP 来建模相关性分布。相比向量检索的单峰分布，GP 允许在嵌入空间中存在多个相关性“峰值”。
*   **初始化（Query Reformulation）**：利用 LLM 生成原始查询 $q_0$ 的多个重构版本 $\{q_1, ..., q_Q\}$。将这些查询的嵌入向量视为具有最高相关性（如 3 分）的初始观测点，从而在后验分布中初始化多个潜在的相关区域。
*   **迭代批处理观测**：
    1.  **采集函数（Acquisition Function）**：使用 Greedy（均值最大）、UCB（考虑不确定性）或随机策略选择下一批文档。
    2.  **批处理多样化（MMR-style）**：引入一种受最大边际相关性（MMR）启发的批处理采集方法，在选择高预测相关性文档的同时，确保批次内文档的差异性。
    3.  **LLM 评分**：调用 LLM 对选定的文档批次进行 0-3 分的绝对相关性评分（Pointwise）。
    4.  **后验更新**：将 LLM 的评分作为新的观测值更新 GP 后验，动态调整搜索空间。
