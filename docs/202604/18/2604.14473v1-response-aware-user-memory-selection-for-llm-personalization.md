## 摘要
大语言模型（LLM）个性化的一种常用方法是在推理阶段将用户记忆的子集整合到提示词中，以引导模型的生成。现有方法主要根据用户记忆项与输入查询之间的相似度来选择这些子集，忽略了特征实际上如何影响模型的响应分布。我们提出了响应效用优化记忆选择（Response-Utility optimization for Memory Selection, RUMS），这是一种通过衡量记忆子集与模型输出之间的互信息来选择用户记忆项的新方法，旨在识别那些能够降低响应不确定性并使预测比语义相似度更精准的项。我们证明，这种基于信息论的基础能够实现更具原则性的用户记忆选择，与现有最先进方法以及规模大 400 倍的模型相比，其选择结果与人类的选择更加一致。此外，我们还表明，与现有方法相比，使用 RUMS 选择的记忆项能带来更好的响应质量，同时计算成本降低了高达 95%。

## Abstract
A common approach to personalization in large language models (LLMs) is to incorporate a subset of the user memory into the prompt at inference time to guide the model's generation. Existing methods select these subsets primarily using similarity between user memory items and input queries, ignoring how features actually affect the model's response distribution. We propose Response-Utility optimization for Memory Selection (RUMS), a novel method that selects user memory items by measuring the mutual information between a subset of memory and the model's outputs, identifying items that reduce response uncertainty and sharpen predictions beyond semantic similarity. We demonstrate that this information-theoretic foundation enables more principled user memory selection that aligns more closely with human selection compared to state-of-the-art methods, and models $400\times$ larger. Additionally, we show that memory items selected using RUMS result in better response quality compared to existing approaches, while having up to $95\%$ reduction in computational cost.

大语言模型（LLM）的个性化通常依赖于在推理时将用户相关的记忆（如偏好、历史、个人信息）插入提示词（Prompt）中。然而，现有的记忆选择方法（如 RAG 检索）主要依赖于**输入查询与记忆项之间的语义相似度**。
*   **研究动机**：作者发现语义相似度并不等同于“生成效用”。某些语义相关的记忆可能是冗余的，而某些看似无关的记忆却能显著改变模型的输出分布。
*   **核心问题**：如何选择那些能真正减少模型响应不确定性、提高生成质量的用户记忆，同时避免无关信息带来的干扰和高昂的计算成本？

#### 2. 方法论：RUMS (Response-Utility optimization for Memory Selection)
RUMS 的核心思想是从**信息论**角度出发，选择能最大化降低模型响应熵（Entropy）的记忆子集。
*   **RUMS-Utility（效用函数）**：
    *   定义为输入查询下，模型输出分布与加入记忆子集后的输出分布之间的**条件互信息**。
    *   **核心逻辑**：如果加入某项记忆后，模型生成特定响应的概率显著集中（熵减），则该记忆项具有高预测效用。

    *   训练一个轻量级的编码器-分类器（基于 **DeBERTa-v3-large**），使其在推理时能够快速预测哪些记忆项对当前查询最有价值。

    *   针对 H1（区分能力）、H2（选择对齐度）、H3（生成质量）三个假设进行了详尽验证。
    *   进行了大量的鲁棒性分析（附录中包含对模型大小、采样数量、解码方式、Token 长度等超参数的消融实验）。
*   **充分性评价**：实验设计非常全面。不仅对比了同类检索算法，还跨级对比了比自身大 400 倍的 GPT-4 模型。引入人工评估验证了“LLM 评测”的客观性，实验结果具有较强的说服力。

#### 6. 主要结论与发现
*   **性能卓越**：RUMS 选择的记忆子集在 F1 得分上显著优于 SOTA 检索方法，且在生成质量上超过了使用 Few-shot 提示的 GPT-4。
*   **成本极低**：相比于“全量输入（All）”策略，RUMS 减少了约 **95%** 的计算成本；相比于 GPT-4 选择器，其推理开销也大幅降低。
*   **决策智能**：RUMS 能够有效识别何时**不需要**个性化（如常识问答），从而避免引入噪声，这在非个性化场景下比 GPT-4 更准确。
*   **理论验证**：证明了降低预测熵与最大化用户期望效用之间存在理论关联。

#### 7. 优点与亮点
*   **视角新颖**：跳出了传统的“语义匹配”框架，转向“响应感知”，真正关注记忆对模型决策的影响。
*   **原则性强**：提供了坚实的信息论证明，将直觉上的“有用性”量化为可计算的熵减。
*   **实用性高**：通过“离线计算效用+在线轻量模型预测”的架构，解决了信息论方法难以落地的计算瓶颈。

#### 8. 不足与局限
*   **静态记忆假设**：实验主要针对静态的用户画像特征（如年龄、职业、爱好），对于动态演化的长短期对话记忆处理尚需进一步验证。
*   **候选集预筛选**：在处理极大规模记忆库（如数千条记录）时，仍需依赖 GPT-4 进行初步筛选，这在一定程度上引入了预处理成本。
*   **模型依赖性**：RUMS-Utility 的计算依赖于参考模型（如 Llama-3），如果参考模型与最终生成模型分布差异过大，效用信号可能产生偏差。
