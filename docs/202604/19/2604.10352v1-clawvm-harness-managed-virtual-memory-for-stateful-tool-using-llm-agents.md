## 摘要
有状态的工具使用型 LLM 智能体将上下文窗口视为工作内存，然而目前的智能体框架对驻留性和持久性的管理仅停留在“尽力而为”的层面，导致了频繁发生的故障：压缩后的状态丢失、重置时跳过刷新以及破坏性回写。我们提出了 ClawVM，这是一个虚拟内存层，它将状态管理为具有最小保真度不变性的类型化页面，在 Token 预算下采用多分辨率表示，并在每个生命周期边界进行经过验证的回写。由于框架已经负责组装提示词、协调工具并观察生命周期事件，因此它是天然的执行点；将契约置于此处使得驻留性和持久性变得确定且可审计。通过合成工作负载、12 个真实会话追踪以及对抗性压力测试，ClawVM 证明了只要最小保真度集合符合 Token 预算（由离线真值确认），它就能消除所有策略可控的故障，且每轮增加的策略引擎中值开销小于 50 微秒。

## Abstract
Stateful tool-using LLM agents treat the context window as working memory, yet today's agent harnesses manage residency and durability as best-effort, causing recurring failures: lost state after compaction, bypassed flushes on reset, and destructive writeback. We present \textsc{ClawVM}, a virtual memory layer that manages state as typed pages with minimum-fidelity invariants, multi-resolution representations under a token budget, and validated writeback at every lifecycle boundary. Because the harness already assembles prompts, mediates tools, and observes lifecycle events, it is the natural enforcement point; placing the contract there makes residency and durability deterministic and auditable. Across synthetic workloads, 12 real-session traces, and adversarial stress tests, \textsc{ClawVM} eliminates all policy-controllable faults whenever the minimum-fidelity set fits within the token budget, confirmed by an offline oracle, and adds median <50 microseconds of policy-engine overhead per turn.

论文关注的是**有状态工具使用型 LLM 智能体（Stateful Tool-Using Agents）**在长时运行中的可靠性问题。
*   **研究动机**：当前的智能体框架（如 OpenClaw）将上下文窗口视为“工作内存”，但在管理状态的驻留（哪些内容留在窗口内）和持久化（哪些内容存入硬盘）时，采用的是“尽力而为”的启发式策略。
*   **核心痛点**：这种非确定性的管理导致了三类频繁故障：
    1.  **驻留失败**：上下文压缩（Compaction）时误删关键指令或计划，导致智能体“失忆”。
    2.  **持久化失败**：在会话重置或崩溃时，未及时回写（Flush）导致进度丢失，或发生破坏性覆盖。
    3.  **不可观测性**：当检索失败时，系统无法区分是“没找到”还是“权限拒绝”或“后端错误”。

### 2. 方法论：核心思想与关键技术
ClawVM 的核心思想是将智能体状态视为**受管页面**，通过框架强制执行驻留和回写规则。
*   **类型化页面（Typed Pages）**：将状态划分为六类（如 Bootstrap 策略页、Plan 计划页、Evidence 证据页等），每类页面都有其“最小保真度不变性”。
*   **多分辨率表示（Multi-resolution Representations）**：每个页面预先生成四种粒度的表示：**Full（全文）**、**Compressed（压缩）**、**Structured（结构化字段）**和 **Pointer（指针/句柄）**。当 Token 预算紧张时，系统按需降级表示，而非直接丢弃。
*   **确定性调度算法**：
    *   **阶段 1**：强制安装所有硬固定（Hard-pinned）页面及其最小保真度表示。
    *   **阶段 2**：基于贪心算法，根据“边际效用/Token成本”对比，对页面进行升级（如从指针升级为全文）。
*   **验证回写协议（Validated Writeback）**：在生命周期边界（如重置、压缩）执行三阶段事务：结构化暂存 -> 确定性验证（检查模式、权限、非破坏性） -> 作用域提交。
*   **故障模型**：定义了可观测的故障类型（如 Refetch 重新获取、Flush-miss 刷新丢失等），使内存管理行为可审计、可重放。

### 6. 主要结论与发现
*   **故障消除**：在所有 Token 预算级别下，ClawVM 彻底消除了所有“策略可控故障”（从基准线的平均 67.8 个降至 **0** 个）。

*   **鲁棒性**：ClawVM 对超参数（如效用权重）不敏感，其安全性主要源于**结构化约束**（如强制回写和硬固定）而非启发式调优。
*   **性能**：系统开销极低，能够实时运行在现有的智能体框架中。

### 7. 优点
*   **确定性**：将原本模糊的提示词工程（Prompt Engineering）转化为确定性的系统工程。
*   **多分辨率设计**：巧妙解决了“全有或全无”的上下文丢弃问题，允许状态优雅降级。
*   **可观测性**：引入故障代码，使得智能体在长程任务中的失败变得可诊断、可重现。
*   **框架无关性**：作为中间层实现，可以适配不同的 LLM 后端和检索插件。

### 8. 不足与局限
*   **语义正确性**：ClawVM 只能保证状态“在场”且“格式正确”，但无法阻止模型生成**语义错误**的内容（即模型一本正经地胡说八道）。
*   **工具模拟限制**：重放实验假设工具调用是可模拟的，对于具有高度非确定性或副作用的真实外部服务，重放的保真度可能受限。
*   **应用范围**：目前主要在单智能体、长对话场景下验证，在复杂的多智能体协作（Multi-agent Orchestration）中的表现尚待进一步研究。
