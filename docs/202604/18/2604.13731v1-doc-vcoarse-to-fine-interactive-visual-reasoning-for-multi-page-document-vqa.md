## 摘要
多页文档视觉问答（DocVQA）需要对长篇且视觉密集的文档中的语义、布局和视觉元素进行推理。现有的无 OCR 方法在容量和精度之间面临权衡：端到端模型在处理文档长度时的扩展性较差，而基于视觉检索的流水线则显得脆弱且被动。我们提出了 Doc-$V^*$，这是一个无 OCR 的智能体框架，它将多页 DocVQA 建模为序列化的证据聚合过程。Doc-$V^*$ 从缩略图概览开始，随后通过语义检索和针对性的页面获取进行主动导航，并在结构化工作记忆中聚合证据以进行有据推理。通过专家轨迹的模仿学习进行训练，并利用群体相对策略优化（GRPO）进一步优化，Doc-$V^*$ 在回答准确性与证据寻找效率之间取得了平衡。在五个基准测试中，Doc-$V^*$ 的表现优于开源基准模型，并接近专有模型，在域外性能上比 RAG 基准提升了高达 47.9%。其他结果表明，有效的证据聚合源于选择性注意力，而非增加输入页数。

## Abstract
Multi-page Document Visual Question Answering requires reasoning over semantics, layouts, and visual elements in long, visually dense documents. Existing OCR-free methods face a trade-off between capacity and precision: end-to-end models scale poorly with document length, while visual retrieval-based pipelines are brittle and passive. We propose Doc-$V^*$, an \textbf{OCR-free agentic} framework that casts multi-page DocVQA as sequential evidence aggregation. Doc-$V^*$ begins with a thumbnail overview, then actively navigates via semantic retrieval and targeted page fetching, and aggregates evidence in a structured working memory for grounded reasoning. Trained by imitation learning from expert trajectories and further optimized with Group Relative Policy Optimization, Doc-$V^*$ balances answer accuracy with evidence-seeking efficiency. Across five benchmarks, Doc-$V^*$ outperforms open-source baselines and approaches proprietary models, improving out-of-domain performance by up to \textbf{47.9\%} over RAG baseline. Other results reveal effective evidence aggregation with selective attention, not increased input pages.

#### 1. 核心问题与整体含义
多页文档视觉问答（Multi-page DocVQA）要求模型在长篇且视觉复杂的文档（如论文、财务报告）中同时处理语义、布局和视觉元素。现有的无 OCR 方法面临**容量与精度的权衡**：
*   **端到端模型**：受限于计算成本和上下文长度，处理长文档时效率低下且容易产生“中间丢失”效应。
*   **检索增强生成（RAG）**：属于被动处理，容易受到检索错误的影响，且难以处理需要多步跳转的复杂推理。
**研究动机**：受人类“先扫视全局、再按需检索、最后精读”的认知过程启发，论文提出 Doc-V* 框架，将 DocVQA 视为一个**主动的、序列化的证据聚合过程**。

#### 2. 方法论
Doc-V* 是一个基于智能体（Agent）的无 OCR 框架，其核心流程如下：
*   **全局缩略图概览（Global Thumbnail Overview）**：将文档所有页面缩放为低分辨率缩略图并排列成网格，作为初始观察。这为智能体提供了文档类型、布局和图表分布的全局结构先验，且显存消耗极低。
*   **动作空间（Action Space）**：
    *   `retrieval_page`：基于语义查询检索最相关的页面。
    *   `fetch_page`：根据缩略图观察到的页码直接获取高分辨率页面。
    *   `answer`：在证据充足时输出最终答案。
*   **结构化视觉推理（ReAct 风格）**：智能体在每一步输出 `<think>`（包含分析、计划、总结）和 `<action>`。
*   **工作记忆（Working Memory）**：通过连接每一轮的摘要，维护一个结构化的记忆库，防止长序列推理中的遗忘和重复操作。
*   **两阶段训练策略**：
    1.  **监督微调（SFT）**：使用 GPT-4o 蒸馏出的 9,019 条高质量交互轨迹进行模仿学习。
    2.  **强化学习（GRPO）**：采用群体相对策略优化（Group Relative Policy Optimization），通过奖励函数（准确性、证据召回率、格式规范）联合优化回答精度与搜索效率。

#### 6. 主要结论与发现
*   **性能卓越**：Doc-V* 在开源模型中达到领先水平，甚至在某些指标上超越了 GPT-4o 和 Claude-3.7。

#### 7. 优点
*   **认知启发式设计**：模拟人类阅读行为，通过“由粗到精”的策略有效解决了长文档的计算冗余问题。
*   **无 OCR 鲁棒性**：直接处理图像，避免了 OCR 错误导致的连锁反应，保留了丰富的布局信息。
*   **强化学习优化**：通过 GRPO 显著提升了智能体在有限步数内寻找证据的效率。

#### 8. 不足与局限
*   **底座依赖**：实验主要基于 Qwen-2.5-VL，尚未在其他视觉语言模型底座上验证通用性。
*   **场景限制**：目前仅针对单文档问答，尚未扩展到需要跨多个异构文档聚合信息的复杂场景。
*   **幻觉风险**：作为生成式模型，在证据极度模糊或文档质量极差时，仍存在产生幻觉的可能。
