## Abstract 

Agent memory failures are silent: an LLM-based agent can produce a fluent response even when it fails to extract, retain, or retrieve the information needed across sessions. The write-manage-read loop describes the external pipeline of these systems but leaves open which internal computations implement each stage. Tracing internal feature circuits across the Qwen-3 family (0.6B–14B) and two memory frameworks (mem0 and A-MEM), we report three findings. First, control is detectable before content : routing circuitry is causally active at 0.6B, while content circuitry produces no detectable signal until 4B under our tracing setup, creating a deployment regime where small models route with apparent competence but silently fail at extraction and grounding. Second, within the content group, Write and Read share a late-layer hub that operates as a context-grounding substrate already present in the base model; only memory framing recruits a functional grounding direction on this substrate, and the hub transfers across both frameworks. Third, emergence does not imply steerability : although the content circuit becomes detectable at 4B, it becomes reliably steerable only at 8B, indicating that detection and intervention have distinct scale thresholds. As a practical implication, the feature-space separation between the two circuit groups enables per-operation failure localization at 76.2% accuracy without supervision, providing a stage-level diagnostic for otherwise silent agent-memory failures.

Agent memory systems enable LLM-based agents to per-sist, organize, and recall information across sessions, and have become a core component of production agent archi-tectures [Chase, 2022, Chhikara et al., 2025]. Zhang et al. [2025], Du [2026] formalize the shared structure underly-ing these systems as a write-manage-read loop: the agent extracts facts from conversations ( Write ), decides whether to add, update, delete, or keep stored memories ( Manage ), and grounds its answers in retrieved content ( Read ). This loop unifies mem0 [Chhikara et al., 2025], A-MEM [Xu et al., 2025], and RL-trained systems [Yan et al., 2025, Huo et al., 2026], and its explicit stage decomposition makes it amenable to per-operation analysis.

The stage decomposition suggests a natural diagnostic question: when an agent gives a wrong answer, which stage failed? In practice this question cannot be answered from external behavior alone. Each stage emits syntactically valid output, extracted facts look well-formed, routing decisions are legal tokens, final answers are fluent, so failures are silent by construction. The community’s evaluation vocabulary reflects this opacity: a single end-to-end accuracy number that conflates extraction, routing, and grounding errors into one score [Cemri et al., 2025, Hu et al., 2026]. As these systems scale to longer horizons and more autonomous decisions, the cost of these undiagnosed failures compounds [Zhao et al., 2026]. Opening the black box requires looking inside. Mechanistic interpretability has begun to address related problems in single-turn retrieval: ReDeEP and Retrieval Heads dissect hallucination and context retrieval within a single forward pass [Sun et al., 2025, Wu et al., 2025b], and ROME/MEMIT localize factual associations in MLPs [Meng et al., 2022, 2023]. But agent memory is not a single call. A Write failure corrupts what enters the store, a wrong Manage decision persists in the store, and these errors surface only at Read time; each stage is a separate forward pass with its own failure mode, and no existing work traces circuits through this multi-call pipeline [Hu et al., 2025]. A second dimension is equally unaddressed: agent memory systems are deployed across a wide range of model scales, and the same framework can behave differently depending on the backbone [Bandel et al., 2026, Zhang et al., 2026]. When a failure vanishes under scaling, it is consistent with a capacity-limited regime; when it persists, it is more suggestive of a structural bottleneck [Nanda et al., 2023, Bahri et al., 2024]. Only cross-scale internal analysis can make this distinction. This paper investigates whether the write–manage–read stages of agent memory system leave separa-ble internal signatures that are stable enough to localize silent failures to specific memory pipeline stages. To this end, we trace feature circuits [Hanna et al., 2025] through each memory operation across the Qwen-3 family (0.6B, 4B, 8B, 14B) and two memory frameworks, mem0 and A-MEM. To our knowledge, this is the first circuit-level analysis of memory-stage circuits that moves beyond single-turn retrieval to study how memory computations emerge, specialize, and support diagnosis across a multi-call agent pipeline. Our analysis yields three key contributions: 1. Scale-Dependent Memory Computation: We identify a “control-before-content” asymme-try where routing circuitry is already causal at 0.6B, whereas content circuitry for writing and reading becomes detectable only at 4B. This reveals a regime where small agents can actively route and update memories before developing robust internal machinery to extract and ground the relevant content. 2. Shared Grounding Hub for Write and Read operations: Within the late-emerging content group, Write and Read share a late-layer hub that acts as a grounding substrate. This hub carries transferable state across operations, but it is the memory framing that recruits a functional grounding direction on this shared substrate. 3. Unsupervised Stage-Level Failure Diagnostic: We propose an unsupervised diagnostic that exploits the feature-space separation of memory-stage circuits to localize silent failures to specific pipeline stages, achieving 76.2% localization accuracy. In developing this tool, we find that detectability does not imply steerability, direct amplification of content circuits is fragile in pipeline, which motivates redirecting their use from write-in intervention to read-out diagnosis. 

## 2 Background and Related Work 

Agent memory systems. Zhang et al. [2025], Du [2026] unify the design space of agent memory under a write-manage-read loop. The loop is universal across explicit LLM calls [Chhikara et al., 2025, Xu et al., 2025], hierarchical paging [Packer et al., 2023], observation-reflection-retrieval [Park et al., 2023] and RL-trained policies [Yan et al., 2025, Huo et al., 2026, Yu et al., 2026], and graph-based storage [Jiang et al., 2026]. Xiong et al. [2025] empirically show that addition and deletion operations shape long-term agent behavior; benchmarks expose persistent weaknesses including selective forgetting [Hu et al., 2026] and a gap between passive recall and active use [He et al., 2026]. 

Circuit analysis and scaling. Circuit tracing with transcoders [Dunefsky et al., 2024, Hanna et al., 2025] produces feature-level attribution graphs; automated circuit discovery methods [Conmy et al., 2023] provide complementary graph-search approaches. Sparse feature circuits [Marks et al., 2025, Kharlapenko et al., 2025] and cross-task reuse studies [Merullo et al., 2024, Mondorf et al., 2025, He 2et al., 2025] extend this to practical models. Most relevant, Hanna and Ameisen [2026] trace planning circuits across Qwen-3, finding scale-dependent emergence, and Tigges et al. [2024] show circuit stability across training. MI is also used as an engineering tool: SAE attribution for debugging [Betley et al., 2025] and representation-level interventions for safety [Zou et al., 2024]. 

MI for retrieval and factual memory. ReDeEP [Sun et al., 2025] and Retrieval Heads [Wu et al., 2025b] identify mechanisms behind RAG hallucination and context retrieval, while ContextFo-cus [Anand et al., 2026] and RAGLens [Xiong et al., 2026] improve faithfulness via activation steering. ROME [Meng et al., 2022] and MEMIT [Meng et al., 2023] localize and edit factual associations in MLPs; Geva et al. [2023] dissect the recall mechanism for factual associations, identifying attribute extraction in mid-layers and subject enrichment in early layers. These studies address single-turn retrieval or static knowledge; we extend to the multi-turn write-manage-read loop and trace how memory circuits develop with scale. ## 3 Method 

3.1 Agent Memory Pipeline 

The pipeline comprises three LLM operations: Write (extract facts from conversation), Manage 

(decide add/update/delete/none), and Read (answer from retrieved memory). Each is an independent forward pass with a stage-specific prompt. The pipeline processes sessions sequentially: each session triggers Write (facts added to store), then Manage (each new fact compared against top-5 retrieved neighbors; add/update/delete/none decided per pair). At question time, top-5 facts are retrieved and passed to Read. Retrieval is handled by embedding-based search in both mem0 and A-MEM and does not involve an LLM call. Our primary system is mem0 [Chhikara et al., 2025], 1 a widely adopted open-source agent memory framework that stores extracted facts as flat key-value entries and retrieves them via semantic search. We additionally analyze A-MEM [Xu et al., 2025], which organizes memories as a self-linking Zettelkasten network with three LLM-driven stages: Note Construction (Write, prompt Ps1), Link Generation + Memory Evolution (Manage, prompts Ps2/Ps3), and embedding-based Retrieve Relative Memory (Read). 3.2 Feature Circuits with Pre-Trained Transcoders 

We trace feature circuits [Hanna et al., 2025] through each memory operation. Pre-trained linear transcoders (PLTs) [Dunefsky et al., 2024] replace each MLP layer with a sparse encoder-decoder that produces monosemantic feature activations. The resulting feature circuit connects input token embeddings, transcoder features (identified by layer, position, and feature index), and output logit nodes through weighted causal edges, computed exactly under a locally linear replacement model where attention patterns and normalization terms are frozen. All circuits are traced on correct instances, filtered by per-stage correctness flags assigned by a Qwen-3 32B judge (Section C). This ensures the discovered circuits reflect successful computation rather than error pathways. For each operation at each scale, we apply a five-step pipeline (hyperparameters in Section D): 

Step 1 (Attribution). Per-sample feature circuits via backward pass through the replacement model, retaining the top 4,096 feature nodes and pruning to 80% node influence / 95% edge influence. Step 2 (Aggregation). Cross-sample recurrence counting, reducing each circuit to 50–200 recurrent features. 

Step 3 (Path tracing). Canonical paths extracted by greedily following highest-weighted edges from input to output. Step 4 (Causal verification). On a held-out 30% split [Hanna and Ameisen, 2026], we compare zero ablation and 5× amplification of top-10 circuit features against matched random baselines. 2 The target y∗ is the model’s own top prediction at the JSON prefix position (up to 5 logits covering ≥80% cumulative probability; Section B). The primary metric is ∆p, the mean change in probability of y∗ under intervention. The causal gap , |∆pcircuit | − | ∆prandom |, isolates discovered features from arbitrary same-layer features; we report causal gaps with bootstrap 95% CIs (Section Q). A near-zero gap means no contribution is detected, which could reflect true absence, distributed implementation, or replacement-model limitations. We also report Jaccard similarity 

J(A, B ) = |A ∩ B|/|A ∪ B| over top-30 features to quantify circuit sharing. 

3.3 Experimental Setup 

Each stage uses a short prompt ending with a JSON prefix that positions the model to generate the stage-specific content token next (Table 1; full prompt details in Section E). We study four models from the Qwen-3 family (0.6B, 4B, 8B, 14B) [Yang et al., 2025], each with corresponding PLTs from the mwhanna transcoder collection ( dfeature = 163 ,840 , ReLU, full-layer coverage). Both systems are evaluated on LongMemEval [Wu et al., 2025a] (500 questions, 5 memory types; dataset details in Section C). For cross-system analysis (Section 5), we replace mem0’s prompts with A-MEM’s equivalents. All experiments run on NVIDIA H200 GPUs; details are in Section D. 

## 4 Memory-Stage Circuits Across Scale 

We apply the five-step analysis pipeline (Section 3) to each memory operation at four model scales (0.6B, 4B, 8B, 14B), producing 200 feature circuits per operation per scale. The organizing finding is a control-before-content asymmetry : under our tracing setup, routing circuitry (Manage) produces detectable causal signal at 0.6B, while content circuitry (Write, Read) produces none until 4B (Figures 3 and 4). 4.2 Content Circuits Emerge Late and Distribute with Scale 

In contrast, the content operations (Write, Read) produce no detectable causal signal at 0.6B (bootstrap CIs include zero; Section Q). We illustrate this with the Write operation, which extracts personal facts from conversation sessions. 3

At 8B, 200 Write attribution graphs reveal a three-stage information flow: subject anchoring at L22, word-specific extraction at L28, and category aggregation at L34 where a cluster of hub features converges dozens of token types into a shared output channel (Table 2). Write circuits show a non-monotonic causal trajectory across scale (Figure 4, left): no detectable signal at 0.6B, a sharp peak at 4B as the hub cluster emerges in a concentrated phase, and a gradual decline at 8B–14B as computation distributes across more features. This non-monotonic pattern foreshadows a key distinction we develop in Section 6: identifiability and steerability come apart. This trajectory contrasts sharply with the routing circuitry, which is already detectable at 0.6B.

4.4 What Does the Shared Hub Compute? 

Having established the two groups and their divergence, we now zoom into the content group to ask: what does the shared hub cluster actually compute? Four lines of evidence progressively narrow the interpretation: the hub is a shared representational substrate that carries transferable state, supports a functional grounding direction under memory framing, and recruits a memory-specific axis that is not reducible to generic in-context grounding. Write and Read share this cluster at 8B despite entirely different output distributions (verbs vs. nouns; zero token overlap). The overlap documented in Section 4.3 rules out the possibility that the two operations merely occupy nearby but functionally independent regions; they share a common representational substrate. But sharing structure does not yet tell us what the shared structure carries. A shared cluster could still reflect colocation rather than a common computation. Transplanting hub-cluster activations between unrelated samples at 8B disrupts 55% of within-stage predictions, while matched-layer controls produce zero change (Section F). The recipient never shifts to the donor’s specific answer, indicating the hub carries transferable state but not specific content. What does this state modulate? A grounding-direction intervention further supports the memory-grounding interpretation. We define a memory-grounding direction from the mean hub activation difference between memory-present and memory-absent Read prompts, then intervene on conflict questions. At 8B, dampening this direction shifts the model away from the memory answer by 0.5 log-prob margin; at 14B the direction is cleanly bidirectional (Section G). This directional effect is consistent with the hub modulating how strongly retrieved content is incorporated into decoding. But is this modulation specific to memory, or would any in-context grounding task recruit the same direction? To test whether this direction reflects a memory-specific mechanism or general in-context grounding, we compare against a direct-context control: the same questions and facts presented as a short context block rather than through the memory framing (Section H). We apply the same direction-extraction 6methodology to both conditions and steer each condition’s conflict prompts along its own direction, measuring ∆M , the shift in log-probability margin between the in-context answer and the model’s parametric answer. The memory-derived direction produces a clear steering effect ( ∆M = −2.30 ), while the direction extracted under direct-context framing does not ( ∆M = +0 .32 ). Geometrically, the two directions share substantial subspace (cosine 0.74 , vs. a split-half upper bound of 0.94 ), confirming the hub is a shared substrate, but only memory framing recruits a functional grounding axis on it. Together, these four lines of evidence converge: the hub is a shared context-grounding substrate 

on which memory operations recruit a delicate, memory-specific functional direction . The highly specific nature of this mechanism naturally dictates our next steps. First, does such a delicate structure survive across different agent architectures? We confirm its robust generalization across two distinct memory interfaces in Section 5. Following this, we ask how to harness it practically: Section 6 reveals that its intricate directional nature makes blunt steering interventions fragile, but Section 7 demonstrates that its distinct topological signature ultimately provides the key to solving our initial diagnostic problem.

Rather than forcing blunt interventions on delicate circuits, we pivot back to our motivating problem of silent failures. ## 7 Localizing Silent Failures 

Steering addresses only half the problem: even where it works, it does not tell a practitioner which operation failed. And end-to-end evaluation fares no better—agent memory failures are silent (Figure 1): outputs are syntactically valid, so no error is flagged, and a single accuracy score cannot distinguish whether each stage failed. Because the signatures are derived from circuit structure rather than benchmark-specific labels, they should generalize to other evaluation suites. We verify this on LoCoMo [Maharana et al., 2024] and MemoryAgentBench [Hu et al., 2026], where accuracy remains above 65% without retraining (Table 5; benchmark details in Section I.5). Once localized, each failure mode maps to a corrective action (Table 6): Write failures (incomplete extraction) are addressed by re-prompting the Write stage with amplified circuit features, forcing the model to re-extract; Manage failures (spurious conflict) are addressed by rule-based deletion of the erroneous update, bypassing the unreliable routing decision; Read failures (grounding failure—the model retrieves the relevant entry but fails to integrate it into the answer) are addressed by amplifying the hub grounding direction, pushing the model to attend to retrieved content it is ignoring. Applying each correction only to instances flagged by the diagnostic yields +5 –8 pp fact recall at 8B, compared with +3 pp for unconditional amplification applied to all samples (full cross-scale comparison in Section R). The diagnostic requires one forward pass per operation with transcoder hooks, keeping the overhead compatible with offline debugging workflows. 

## 8 Discussion 

Control is detectable before content. Routing circuitry is causal at 0.6B under both systems, while content circuits produce no detectable signal until 4B. This creates a deployment regime current evaluation cannot detect: a small model routes memory decisions with apparent competence while silently failing at the extraction and grounding those decisions act upon. Backbone selection therefore needs extractive capability verified independently of routing competence. 

The hub is recruited, not created. The shared hub carries transferable state, is steerable along a grounding direction, and persists across frameworks. Section 4.4 shows the hub is not memory-exclusive: the same cluster activates under plain in-context framing, but only memory framing recruits a functional grounding axis on it. This recruit-versus-create distinction reframes where memory reliability comes from. If memory frameworks do not build grounding machinery but borrow a pre-existing substrate, then differences between frameworks are unlikely to be resolved by prompt engineering or storage-format changes alone; the upper bound is set by how effectively a given framing aligns with the direction the base model already supports. 

From steerability to diagnosis. Steering is feasible only in a narrow regime: under our tested multipliers, only 8B produces consistent gains across all intervention strengths (Section 6). Where steering is unreliable, per-operation diagnosis takes over. Agent memory forces a question: when a pipeline of LLM calls produces a wrong answer, which call failed? Our diagnostic (Section 7) answers this via circuit signatures, localizing the responsible operation and mapping each failure to a corrective action. This works because circuits for different operations occupy separable feature-space regions, and their cross-scale ordering is consistent enough that a diagnostic built at one scale transfers within the same model family. 

## 9 Conclusion 

This paper reports three mechanistic findings about LLM agent memory circuits: a control-before-content asymmetry across scales, a shared context-grounding hub activated by memory framing, and scientifically intriguing but practically unreliable steerability. Crucially, the feature-space separation between these circuits enables precise, per-operation failure localization. These results demonstrate that improving agent memory reliability requires leveraging the base model’s internal computations, paving the way for circuit-based monitoring and structurally-guided memory architectures. 9References

Our analysis covers one model family (Qwen-3), two memory frameworks, short factorized prompts (45–65 tokens), and MLP-traced circuits via per-layer transcoders. ## B Transcoders and Circuit Tracing Details 

B.1 Transcoders 

Standard MLP activations are dense and polysemantic, making it difficult to attribute model behavior to individual concepts. Per-layer transcoders [Dunefsky et al., 2024] address this by learning a sparse, monosemantic decomposition of each MLP’s computation. Given the input to an MLP, 

h ∈ Rd, a transcoder encodes it into a high-dimensional sparse code z = f (Wenc h + benc ) ∈ Rn

(where n ≫ d and most entries of z are zero) and decodes it back to approximate the MLP’s output: 

ˆh′ = Wdec z + bdec . Because each active dimension of z tends to correspond to a single interpretable concept, transcoders enable fine-grained causal interventions: to test whether a feature matters, we modify its activation in z to produce z′ and patch the difference Wdec (z′ − z) into the MLP’s output during the forward pass. We use the Qwen-3 ReLU transcoders released by Hanna and Ameisen [2026], which share a feature dimension of 163 ,840 across all scales and take post-LayerNorm MLP inputs. To trace how information flows from input tokens through transcoder features to output logits, we build a feature circuit for each prompt [Hanna et al., 2025]. The circuit is a weighted DAG whose nodes are input embeddings, transcoder features, and logit targets, and whose edge weights quantify each node’s direct causal contribution to its downstream neighbors. Construction proceeds in two stages. First, we build a local replacement model by swapping each MLP with its transcoder and an input-specific error term that accounts for the reconstruction gap. Attention patterns and LayerNorm denominators are detached from the computation graph so that all remaining dependencies are linear. Under this linearization, the direct effect of any node on any other node is exact and can be recovered in a single backward pass per target logit. Second, we select the most influential nodes and edges: we keep the top 4,096 feature nodes ranked by total downstream influence, then prune to the smallest subgraph that retains 80% of node influence and 95% of edge influence. 

B.4 Attribution Target Selection 

For each prompt, the attribution target is the model’s own top prediction at the final token position (immediately after the JSON prefix). We select logits covering ≥80% of the cumulative probability mass (up to 5 logits), so the discovered circuits reflect the computation the model actually performs. Because all circuits are traced on correct-only instances (Section C), the model’s top prediction coincides with the ground truth on these samples, and the discovered circuits reflect successful extraction, routing, or grounding.

## C Evaluation Dataset 

We evaluate on LongMemEval [Wu et al., 2025a], a multi-session agent memory benchmark com-prising 500 questions across 5 memory types: single-session factual recall, cross-session factual recall, temporal reasoning, knowledge update, and multi-hop reasoning. Each question is paired with a conversation history and a ground-truth answer; 1–4 evidence sessions (sessions containing answer-relevant information) are embedded within a larger haystack of filler sessions. The benchmark tests whether the agent can correctly write facts to memory during conversation, manage updates when information changes across sessions, and read from memory to answer questions. We use the full 500-question test set; the 70/30 train-test split for causal verification (Section 3) is applied within each operation’s 200 attribution graphs, not across questions. 

Sampling pipeline. Each question contains 1–4 evidence sessions, and each session generates one Write prompt and one Manage prompt per fact. Read prompts are generated once per question at answer time. We trace circuits exclusively on correct-only instances to ensure the discovered circuits reflect successful computation. Correctness flags are assigned by a Qwen-3 32B judge applied to each pipeline output; the per-stage criteria are: • Write : is_evidence=True (session contains answer-relevant information) AND 

extraction_correct=True (extracted facts judged correct by Qwen-3 32B). • Manage : decision_correct=True (the decision is correct). • Read : retrieval_correct=True AND generation_correct=True (correct memories re-trieved and correct answer generated, both judged by Qwen-3 32B). 14 After correctness filtering, prompts where the model’s top prediction at the JSON prefix position is a structural token (quotation mark, brace) rather than a content word are further excluded. E.1 mem0 Architecture and Prompts

## G Hub Memory-Engagement Experiment 

G.1 Motivation 

The transport experiment (Section F) establishes that the hub carries causally relevant content, but does not specify what the hub functionally controls. The engagement experiment tests whether the hub governs how strongly the model relies on retrieved memory versus parametric knowledge. 

G.2 Setup 

We construct 200 general-knowledge questions (e.g., “What is the capital of France?”) where the model has reliable parametric knowledge. For each question, we create three prompts: • Support : memory contains the correct answer (agrees with parametric knowledge). • Conflict : memory contains a wrong answer (disagrees with parametric knowledge). • Null : no memory provided. 17 G.3 Memory-Grounding Direction 

We define the memory-grounding direction as: 

d = 12 (zsupport + zconflict ) − znull 

where z is the hub activation vector at the last token position. This direction captures what changes in the hub when retrieved memory is present, regardless of whether it agrees or disagrees with parametric knowledge. 

G.4 Intervention and Metric 

On the conflict prompt, we add β · d to the hub activation for β ∈ {− 2, −1, −0.5, +0 .5, +1 , +2 }

and measure the log-prob margin: 

M = log P (memory answer ) − log P (parametric answer )

## H Hub Specificity: Memory vs. Direct-Context Grounding 

H.1 Motivation 

Section G establishes that the hub can be steered along a memory-grounding direction. A natural concern is that this direction may reflect a generic in-context grounding mechanism rather than anything specific to memory: any prompt that places retrievable facts in context—a retrieved passage, a few-shot demonstration, a reference document—might recruit the same hub activation pattern. If so, “memory-grounding” would overstate the specificity of what the hub computes. We test this by comparing memory-framed prompts against an direct-context control with matched question and fact content but different surface framing. 

H.2 Setup 

We reuse the LongMemEval Read instances from Section G and construct two conditions with identical questions and identical supporting/conflicting facts: • Memory condition : the mem0 Read prompt format, in which facts are presented as retrieved memory entries under a MEMORY: header. • Direct-context condition : the same facts presented directly in the prompt as a short context block preceding the question, without memory framing or the mem0 system prompt. Each condition has three variants: support (correct fact provided), conflict (wrong fact provided), and 

null (no context provided). The only variable that changes across conditions is surface framing; the underlying grounding task—answering a factual question given a retrievable fact—is identical. 18 H.3 Direction Extraction 

For each condition cond ∈ { memory , direct }, we compute a grounding direction using the hub cluster activations (top-10 hub features identified in Section 4.4): 

dcond = 12 (zcond support + zcond conflict ) − zcond null 

To calibrate what cosine values are achievable under within-condition noise, we split the memory condition into two random halves and extract dmem ,A , dmem ,B independently. This gives a self-consistency upper bound: cos( dmem ,A , d mem ,B ) = 0 .942 at the pooled hub level, 0.952 at L34 alone.

Together, these three null or wrong-sign results isolate dmemory as the only direction with consistent, sign-correct steering effect on memory-framed prompts. 

## I Diagnostic Localization 

I.1 Motivation 

This section validates whether these signatures can localize the responsible operation when a failure occurs, a capability unavailable from behavioral evaluation. 19 I.2 Operation-Specific Feature Construction 

For each operation o and scale s, we identify operation-specific features from the Step 1 attribution data: 1. Pool the top-20 features from each per-sample circuit within each layer. 2. Count cross-sample recurrence per (layer, feature) pair. 3. For each layer in the operation’s active range, keep the top-10 most recurrent features. 4. Group features into stage-specific banks (e.g., Write: early L20–24, mid L25–30, hub L31–35; Manage: trunk L10–18, routing L19–28). Each operation’s final feature set contains 30 operation-specific features (top-10 per characteristic layer group) drawn from its characteristic layers. 

I.3 Localization Protocol 

The diagnostic is evaluated on real pipeline failure instances (not synthetic probes). Failure instances are drawn from the 500 LongMemEval questions by running the full mem0 pipeline at each scale and retaining questions where the final answer’s token-level F1 against the ground-truth falls below 0.3

(yielding 180/500 at 4B, 193/500 at 8B, and 113/500 at 14B). The 0.3 threshold includes clear failures while excluding near-correct answers with trivial format differences. For each failure instance, we apply the following protocol: 1. Run the model on the failure instance to obtain the baseline top-1 prediction and its probability. 2. Ablate each operation’s feature set in turn (zero the 30 features) and record the change in top-1 probability ( ∆p). 3. The operation whose ablation causes the largest probability drop is flagged as most causally responsible. 4. Localization is correct if the flagged operation matches the ground-truth failure label (e.g., if the fact was never extracted, the ground-truth label is Write; if it was overwritten by a spurious update, the label is Manage; if it was in memory but the answer was wrong, the label is Read). Ground-truth failure labels are assigned from pipeline observables (Write/Read) and LLM-judge attribution validated against human annotations at 84% agreement (Manage). All three failure types (Write, Manage, and Read) are included in the evaluation. 

I.4 Localization Metric 

Localization accuracy = 1

> N

PNi=1 1[most-damaged operation i = ground-truth failure i], measured separately per failure type and per scale. 

I.5 Cross-Benchmark Generalization 

The localization protocol above is validated on LongMemEval. To test whether the circuit signatures transfer to other evaluation suites, we run the same 8B diagnostic on two additional benchmarks: 

LoCoMo [Maharana et al., 2024] provides 1,986 QA pairs across 10 multi-session conversations, covering single-hop, multi-hop, temporal, commonsense, and adversarial question types. MemoryA-gentBench [Hu et al., 2026] tests four competencies (accurate retrieval, test-time learning, long-range understanding, conflict resolution) across contexts from 6K to 1.2M tokens. For each benchmark, we run the mem0 pipeline on Qwen-3 8B, collect failure cases, and obtain ground-truth failure attribution via an independent LLM judge (Qwen-3 8B with a dedicated clas-sification prompt) that labels each failure as Write (fact not extracted), Manage (fact corrupted or overwritten), or Read (fact in memory but answer incorrect). We validated the LLM judge against human annotations on 100 samples per benchmark: LongMemEval 84%, LoCoMo 82%, MemoryA-gentBench 79% agreement. We then apply our circuit diagnostic and compare its predictions against the LLM labels. 20 I.6 Diagnostic Baselines

I.8 Corrective Action Procedure 

Each corrective action is applied conditionally on the diagnostic output. All three operate within the replacement model using the same intervention API as Section 6. 1. Re-prompt Write stage. Re-run the Write forward pass with 5× amplification of the Write circuit features (same feature set and multiplier as the pipeline intervention in Section 6). The amplified pass generates a new set of extracted facts, which replace the original extraction in the memory store. Retrieval and Read then proceed on the updated store. 2. Rule-based deletion. Revert the most recent Manage decision: if the diagnostic flags Manage, discard the UPDATE/DELETE output and keep the original memory entry unchanged. This bypasses the LLM routing entirely and requires no forward pass. 3. Hub amplification. Re-run the Read forward pass with 5× amplification of the Read circuit features (hub cluster). The grounding direction is amplified by adding β · d (β = 5 ) to the hub-cluster activations at the last token position, using the same direction d defined in Section G. The model then generates a new answer from the amplified state.

## S Amplification Strength Sweep 

The main text reports the amplification sweep ( 2×–10 ×). Here we additionally report zero ablation (0×) and matched random controls for completeness (Table 39). At 8B, multipliers 2× through 5× all produce similar fact recall gains ( ≈ 0.92 , +2 –3 pp), indicating a broad plateau. At 0.6B, fact recall degrades with multiplier strength (0.532 at 2×, 0.218 at 5×). At 4B, 5× is uniquely destructive ( 0.218 ) while 2× and 10 × match intact, suggesting a scale-specific resonance at 5×. At 14B, all multipliers produce fact recall ≥ 0.79 

and QA ≥ 0.53 , with no multiplier producing a meaningful improvement over intact. Random controls ( 0× and 5× of random features) produce < 1 pp change at all scales, bounding intermediate multipliers. At 14B, zero ablation slightly improves fact recall ( +5 pp). Random zero ablation at the same layers produces no improvement, confirming the effect is feature-specific.

(i) Dual-use steering. The same hub-grounding direction that can dampen hallucination (Section G) could be used to force an agent to over-rely on attacker-injected memories. Production deployments should gate steering behind authenticated channels. 

(ii) Privacy in diagnostic traces. Circuit-level logging captures which memories influenced a decision, which is more revealing than output-level logging. Organizations should treat circuit logs with the same sensitivity as the underlying memory contents. 

(iii) False assurance. A diagnostic that reports “no failure detected” does not certify correctness; it only indicates that the discovered signatures are not active. Practitioners should avoid interpreting clean diagnostics as safety guarantees, particularly for high-stakes decisions.
