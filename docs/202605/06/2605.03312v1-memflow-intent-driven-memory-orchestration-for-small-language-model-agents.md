## 摘要
现代语言智能体必须在长时程、多轮历史记录中运行，但在小语言模型（SLM）上部署此类智能体仍然存在根本性困难。全上下文提示会导致上下文溢出，扁平化检索会使模型暴露在噪声证据中，而开放式智能体循环在推理能力有限的情况下并不可靠。我们认为，SLM 内存失效的很大一部分原因在于内存操作不匹配：不同的查询类型需要截然不同的检索策略、证据转换和上下文预算，而 SLM 无法通过开放式推理可靠地自我编排这些操作。我们引入了 MemFlow，这是一种无需训练的内存编排框架，它将内存规划从 SLM 中外置。路由智能体（Router Agent）根据意图对每个查询进行分类，并将其分发给内存智能体（Memory Agent），后者执行三个专门层级（档案查找、定向检索或深度推理）之一，并在动态的、层级感知的 Token 预算下组装生成的证据。随后，回答智能体（Answer Agent）根据这一紧凑的上下文生成响应，如果响应未得到所提供证据的支持，验证智能体（Validator Agent）可以选择使用更高级别的内存层级进行重试。这种“路由后编译”的设计避免了工具选择幻觉和推理循环，同时保持了回答上下文的紧凑性。在 LongMemEval、LoCoMo 和 LongBench 等长时程内存基准测试中，基于冻结的 Qwen3-1.7B 骨干模型进行的评估显示，MemFlow 的准确率比全上下文 SLM 基准提高了近 2 倍。这些结果表明，结构化的意图路由和确定性的证据准备可以使能力有限的模型在资源受限的长时程智能体中发挥显著更高的效能。

## Abstract
Modern language agents must operate over long-horizon, multi-turn histories, yet deploying such agents with Small Language Models (SLMs) remains fundamentally difficult. Full-context prompting causes context overflow, flat retrieval exposes the model to noisy evidence, and open-ended agentic loops are unreliable under limited reasoning capacity. We argue that a substantial portion of SLM memory failure arises from mismatched memory operations: different query types demand categorically different retrieval strategies, evidence transformations, and context budgets that SLMs cannot reliably self-orchestrate through open-ended reasoning. We introduce MemFlow, a training-free memory orchestration framework that externalizes memory planning from the SLM. A Router Agent classifies each query by intent and dispatches it to the Memory Agent, which executes one of three specialized tiers (Profile Lookup, Targeted Retrieval, or Deep Reasoning) and assembles the resulting evidence under a dynamic, tier-aware token budget. An Answer Agent then generates a response from this compact context, and a Validator Agent optionally retries with a heavier memory tier when the response is not supported by the provided evidence. This route-then-compile design avoids tool-selection hallucination and reasoning loops while keeping the answer context compact. Evaluated on a frozen Qwen3-1.7B backbone across long-horizon memory benchmarks - LongMemEval, LoCoMo, and LongBench - MemFlow improves accuracy by nearly 2x over full-context SLM baselines. These results suggest that structured intent routing and deterministic evidence preparation can make limited-capacity models substantially more effective in resource-constrained long-horizon agents.

### 1. 核心问题与研究动机
*   **研究背景**：现代语言智能体需要处理长程（Long-horizon）、多轮对话的历史记录。
*   **核心问题**：在资源受限的环境下部署**小语言模型（SLM）**时，面临三大挑战：
    1.  **上下文溢出**：全量历史记录超出模型处理范围。
    2.  **检索噪声**：简单的 RAG（检索增强生成）会将无关信息引入上下文，干扰推理。
    3.  **推理不可靠**：SLM 在执行复杂的 ReAct 循环或工具调用时，极易产生幻觉或逻辑断裂。
*   **核心观点**：SLM 的内存失效本质上是“意图路由不匹配”。不同的查询需要不同的检索策略、证据转换和 Token 预算，而 SLM 无法通过开放式推理自我编排这些操作。

### 2. 方法论：MemFlow 框架
MemFlow 采用“**先路由后编译**”（Route-then-compile）的设计，将内存规划从 SLM 中外置，通过四个专门的代理协作：
*   **路由代理 (Router Agent)**：使用三层级联机制（规则层 -> SLM 分类层 -> 关键字启发层）将查询归类为 7 种意图标签（如个人资料注入、时间推理、冲突解决等）。
*   **内存代理 (Memory Agent)**：根据意图执行三个层级的操作：
    *   **Tier 1 (档案查找)**：直接加载预编译的用户画像，无需检索。
    *   **Tier 2 (定向检索)**：执行多路径实体感知检索（BM25 + 向量检索）。
    *   **Tier 3 (深度推理)**：在 SLM 介入前进行**确定性预处理**（如按时间排序、日期计算、规则过滤），解决 SLM 无法处理的逻辑运算。
*   **上下文打包器 (Context Packer)**：根据意图标签动态分配 Token 预算，确保最关键的证据被优先保留。
*   **回答代理 (Answer Agent)**：基于精简后的上下文，使用特定标签的提示词生成最终回答。
*   **验证代理 (Validator Agent)**：检查回答是否得到证据支持。若失败，则触发“升级”机制，使用更重的内存层级重新尝试。

### 3. 实验设计
*   **数据集/场景**：
    1.  **LongMemEval**：测试长程聊天记忆（500 个问题）。
    2.  **LoCoMo**：超长对话记忆（1,986 个问题，涉及 600 轮对话）。
    3.  **LongBench**：长文档理解与摘要（1,750 个问题）。
*   **对比方法**：
    *   **SLM 基线**：直接问答（短/全上下文）、标准 RAG、ReAct 模式。
    *   **专用内存系统**：Memobase、MemGPT。
    *   **前沿模型参考**：GPT-4o（在匹配预算和全预算下）。
*   **骨干模型**：主要使用 Qwen3-1.7B，并扩展至 SmolLM2-1.7B、LLaMA-3.2-1B 等多个 3B 以下模型。

### 4. 资源与算力
*   **硬件环境**：实验在 Google Colab 上运行，使用了 NVIDIA L4、A100 和 H100 GPU。
*   **训练时长**：**无需训练**。该框架是 Training-free 的，所有模型权重均处于冻结状态，仅涉及推理开销。
*   **推理成本**：论文详细报告了 Token 消耗。虽然 MemFlow 的总流水线 Token 消耗（约 10k）略高于 ReAct，但它显著减少了最终回答阶段的上下文负担（约 2.2k）。

### 5. 实验数量与充分性
*   **实验规模**：总计评估了 4,236 个测试项，涵盖了 5 种不同的 SLM 骨干模型。
*   **充分性**：
    *   **消融实验**：分别移除路由、检索策略、工具、验证器等组件，验证了各模块的贡献。
    *   **泛化性测试**：在不同架构的 SLM 上均观察到了显著提升。
    *   **客观性**：使用了 GPT-4o-mini 作为公正评判员（Judge），并通过人工标注验证了 Judge 的可靠性（一致性达 89.6%）。

### 6. 主要结论与发现
*   **性能飞跃**：在 Qwen3-1.7B 上，MemFlow 的准确率比全上下文基线提高了近 **2 倍**，比 RAG 和 ReAct 提升了 **6.2 个百分点**。
*   **小模型救星**：对于一些在直接问答中表现近乎崩溃的模型（如 SmolLM2），MemFlow 能将其准确率从不到 20% 提升至 40% 以上。
*   **结构化胜过开放式**：实验证明，对于 SLM 而言，确定性的结构化编排比开放式的 ReAct 推理循环更有效且更稳定。

### 7. 优点与亮点
*   **无需训练**：通过纯工程化的编排提升模型能力，易于部署和迁移。
*   **确定性处理**：将 SLM 不擅长的逻辑（如日期计算）交给确定性代码处理，极大降低了幻觉率。
*   **动态预算**：层级感知的 Token 管理解决了“迷失在中间”（Lost-in-the-middle）的问题。
*   **验证机制**：引入验证与升级循环，增强了系统在面对对抗性或复杂查询时的鲁棒性。

### 8. 不足与局限
*   **编排开销**：路由代理的提示词较长，导致初始推理的 Token 成本较高（尽管可以通过缓存技术缓解）。

*   **评估偏差**：依赖 LLM 作为 Judge 可能存在语义理解上的偏差，尽管论文通过人工审计进行了部分校正。
*   **语言限制**：实验仅限于英文数据集，跨语言表现尚未验证。
