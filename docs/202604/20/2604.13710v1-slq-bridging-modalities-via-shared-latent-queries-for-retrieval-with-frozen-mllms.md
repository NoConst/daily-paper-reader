## 摘要
多模态大语言模型（MLLMs）展现出强大的推理能力和世界知识，但将其适配于检索任务仍然具有挑战性。现有方法依赖于侵入性的参数更新，如全量微调和 LoRA，这可能会破坏预训练的语义空间，并损害对推理至关重要的结构化知识。在这项工作中，我们认为将 MLLM 适配于检索应侧重于诱导预训练表示，而非重写它们。为此，我们提出了 SLQ，这是一个高效且有效的框架，通过一小组共享潜查询（Shared Latent Queries）将冻结的 MLLM 适配为检索器。这些查询被附加在文本和图像标记序列的末尾，利用模型原生的因果注意力作为全局聚合接口，在保持主干网络不变的情况下，在统一空间中生成紧凑的嵌入。此外，为了更好地评估超越表层模式匹配的检索能力，我们构建了 KARR-Bench，这是一个专为知识感知推理检索设计的基准测试。广泛的实验表明，SLQ 在 COCO 和 Flickr30K 上的表现优于全量微调和 LoRA，同时在 MMEB 上取得了具有竞争力的性能，并在 KARR-Bench 上获得了显著提升。结果表明，SLQ 在保留预训练表示的同时，为将 MLLM 适配到检索任务提供了一个高效且有效的框架。

## Abstract
Multimodal Large Language Models (MLLMs) exhibit strong reasoning and world knowledge, yet adapting them for retrieval remains challenging. Existing approaches rely on invasive parameter updates, such as full fine-tuning and LoRA, which may disrupt the pre-trained semantic space and impair the structured knowledge essential for reasoning. In this work, we argue that adapting MLLMs for retrieval should focus on eliciting pre-trained representations rather than overwriting them. To this end, we propose SLQ, an effective and efficient framework that adapts a frozen MLLM into a retriever through a small set of Shared Latent Queries. Appended to the end of both text and image token sequences, these queries leverage the model's native causal attention to serve as global aggregation interfaces, producing compact embeddings in a unified space while keeping the backbone unchanged. Furthermore, to better evaluate retrieval beyond superficial pattern matching, we construct KARR-Bench, a benchmark designed for knowledge-aware reasoning retrieval. Extensive experiments show that SLQ outperforms full fine-tuning and LoRA on COCO and Flickr30K, while achieving competitive performance on MMEB and yielding substantial gains on KARR-Bench. The results demonstrate that SLQ, which preserves pre-trained representations, provides an effective and efficient framework for adapting MLLMs to retrieval.

### 2. 方法论：SLQ 框架
*   **核心思想**：保持 MLLM 主干网络完全冻结，通过一小组可学习的**共享潜查询（Shared Latent Queries）**作为接口，提取跨模态对齐的特征。
*   **技术细节**：
    *   **共享查询**：定义 $N$ 个可学习的向量 $Q$，在输入端将其附加在文本 Token 序列和图像 Token 序列的末尾。
    *   **因果注意力聚合**：利用 Decoder-only 模型原生的因果注意力机制，让这些位于末尾的查询 Token 能够“看到”前面所有的多模态上下文，从而聚合全局信息。
    *   **嵌入提取**：提取最后一层中对应查询 Token 的隐藏状态，进行均值池化（Mean Pooling）并归一化，得到最终的检索嵌入。
    *   **训练目标**：仅优化这组查询向量（参数量极小，仅数万个），使用对称的 **InfoNCE 损失函数**进行对比学习训练。

### 3. 实验设计
*   **数据集与场景**：

*   **主干网络**：使用了 InternVL3 (1B, 8B) 和 Qwen3-VL (2B, 4B) 多种规模的模型。

### 6. 主要结论与发现
*   **性能卓越**：SLQ 在 COCO 和 Flickr30K 上优于全量微调和 LoRA；在 MMEB 上达到了 SOTA 水平。
*   **能力保留**：在 MMMU、RealWorldQA 等推理评测中，SLQ 几乎没有性能损失，而全量微调和 LoRA 均出现了明显的推理能力下降。
*   **推理优势**：在 KARR-Bench 上，SLQ 显著领先，证明了冻结主干能更好地利用模型内置的世界知识进行复杂检索。
*   **模态对齐**：SLQ 生成的嵌入空间更紧凑，模态间隙更小。

### 7. 优点
*   **极度高效**：极小的参数量和极低的训练成本，适合资源受限的场景。
*   **非破坏性**：通过冻结主干，完美解决了检索适配中的“灾难性遗忘”问题。
*   **基准贡献**：提出的 KARR-Bench 填补了多模态检索领域缺乏“深层推理”评估的空白。
*   **通用性强**：适用于各种基于 Decoder 的 MLLM 架构。

### 8. 不足与局限
*   **域外适配限制**：由于主干冻结，对于极端的域外数据（如专业的医学影像或卫星图像），SLQ 可能无法学习到全新的视觉特征。

*   **长文本处理**：虽然查询 Token 放在末尾，但对于极长序列的聚合效率和注意力衰减问题仍有待进一步探讨。
