## 摘要
神经检索器通常在由查询、正向段落和一组难负样本组成的大规模三元组数据上进行训练。在实践中，难负样本挖掘可能会引入假负样本和其他模糊负样本，包括与查询相关或包含部分答案的段落。这种标签噪声会导致不一致的监督，并可能降低检索效果。我们提出了 ARHN（以答案为中心的难负样本重标注），这是一个两阶段框架，利用开源大语言模型（LLMs）通过以答案为中心的相关性信号来优化难负样本。在第一阶段，对于每个查询-段落对，ARHN 提示 LLM 生成基于段落的答案片段，或者指出该段落不支持回答。在第二阶段，ARHN 对候选集应用基于 LLM 的列表排序（listwise ranking），根据对查询的直接可回答性对段落进行排序。排名高于原始正样本的段落被重标注为额外的正样本。在排名低于正样本的段落中，ARHN 从负样本集中剔除任何包含答案片段的段落，以避免模糊监督。我们在 BEIR 基准测试上评估了 ARHN 的三种配置：仅重标注、仅过滤以及两者的结合。在不同数据集上，结合策略的效果始终优于任何单一步骤，这表明同时重标注假负样本和过滤模糊负样本能为训练神经检索模型提供更清晰的监督。通过严格依赖开源模型，ARHN 建立了一个具有成本效益且可扩展的优化流水线，适用于大规模训练。

## Abstract
Neural retrievers are often trained on large-scale triplet data comprising a query, a positive passage, and a set of hard negatives. In practice, hard-negative mining can introduce false negatives and other ambiguous negatives, including passages that are relevant or contain partial answers to the query. Such label noise yields inconsistent supervision and can degrade retrieval effectiveness.   We propose ARHN (Answer-centric Relabeling of Hard Negatives), a two-stage framework that leverages open-source LLMs to refine hard negative samples using answer-centric relevance signals. In the first stage, for each query-passage pair, ARHN prompts the LLM to generate a passage-grounded answer snippet or to indicate that the passage does not support an answer. In the second stage, ARHN applies an LLM-based listwise ranking over the candidate set to order passages by direct answerability to the query. Passages ranked above the original positive are relabeled to additional positives. Among passages ranked below the positive, ARHN excludes any that contain an answer snippet from the negative set to avoid ambiguous supervision.   We evaluated ARHN on the BEIR benchmark under three configurations: relabeling only, filtering only, and their combination. Across datasets, the combined strategy consistently improves over either step in isolation, indicating that jointly relabeling false negatives and filtering ambiguous negatives yields cleaner supervision for training neural retrieval models. By relying strictly on open-source models, ARHN establishes a cost-effective and scalable refinement pipeline suitable for large-scale training.

### 1. 核心问题与整体含义（研究动机和背景）
稠密检索（Dense Retrieval）模型通常采用对比学习训练，依赖“硬负样本”（Hard Negatives）来提升区分能力。然而，现有的挖掘算法常会引入**假负样本（False Negatives）**——即那些实际上与查询相关或包含答案、却被错误标注为负例的文档。
*   **研究动机**：这些标签噪声会产生错误的监督信号，扭曲嵌入空间，降低检索精度。
*   **核心目标**：利用开源大语言模型（LLM）的语义理解能力，通过识别文档中是否包含“答案片段”来重新校准负样本的标签，从而提供更纯净的训练数据。

### 2. 方法论：ARHN 框架
ARHN 采用两阶段流程，核心思想是**以答案为中心（Answer-Centric）**进行判断：

*   **第一阶段：答案片段生成（Answer Snippet Generation）**
    *   输入查询和一组文档（1个正样本 + N个硬负样本）。
    *   提示 LLM 从每个文档中**逐字提取**支持回答查询的片段。
    *   如果文档不含答案，LLM 必须输出 `NO_ANSWER`。这一步通过限制 LLM 仅做提取而非生成，有效减少了幻觉。
*   **第二阶段：以答案为中心的列表排序与重标注（Reranking & Relabeling）**
    *   LLM 对提取出的所有片段进行**列表式排序（Listwise Ranking）**，依据是片段回答查询的直接程度和完整性。
    *   **重标注逻辑**：
        1.  **正向重标注（Relabeling）**：若某个负样本的片段排名高于原始正样本，则将其提升为正样本。
        2.  **过滤（Filtering）**：若负样本片段排名低于正样本但仍含有答案片段，则将其视为“歧义样本”从负样本集中剔除。
        3.  **保留（Retention）**：仅保留那些被判定为 `NO_ANSWER` 的文档作为真正的硬负样本。

### 3. 实验设计
*   **数据集**：在 BGE 训练集（包含 MS MARCO, NQ, HotpotQA 等 7 个核心子集）上进行精炼，并在 **BEIR 基准测试**（16 个数据集）上进行零样本（Zero-shot）评估。
*   **基准模型（Backbones）**：
    *   E5-base（基于 BERT 的双编码器）。
    *   LG-ANNA-Embedding（基于 Mistral-7B 的大型嵌入模型）。
*   **对比方法**：
    *   **No Refinement**：原始未处理数据。
    *   **TopK-PercPos**：基于重排序器分数的启发式过滤。
    *   **RLHN**：使用闭源模型（GPT-4o）进行重标注的现有先进方法。

### 4. 资源与算力
*   **算力投入**：使用了 **16 台 NVIDIA A100 GPU** 进行 LLM 推理（Stage 1 和 Stage 2）。
*   **模型规模**：实验对比了 Qwen3-8B、14B 和 32B 三种规模的开源模型。
*   **训练细节**：E5-base 微调 5 个 epoch；Mistral-7B 模型使用 LoRA 技术微调 2 个 epoch。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了 16 个 BEIR 数据集，包括 7 个完全未见过的领域（OOD），验证了泛化性。
*   **消融实验**：分别测试了“仅过滤”、“仅重标注”以及“两者结合（R+F）”的效果。
*   **充分性评价**：实验设计较为全面，不仅对比了不同规模的 LLM，还通过人工标注（500 组样本）计算了 Cohen’s Kappa 系数，验证了 LLM 标注与人类判断的一致性，体现了实验的客观性。

### 6. 主要结论与发现
*   **性能提升**：ARHN(R+F) 在 E5-base 上比原始数据提升了 1.3 个 nDCG@10 点，在 Mistral-7B 上提升了 0.8 点。
*   **协同效应**：重标注和过滤结合的效果远好于单一操作，说明纠正假负例和剔除歧义样本同样重要。
*   **模型规模正相关**：随着 LLM 规模增加（从 8B 到 32B），重标注的准确性显著提升，检索性能也随之增强。
*   **超越闭源**：使用开源的 Qwen3-32B 效果优于使用 GPT-4o-mini 的 RLHN 方法，证明了开源模型在数据精炼任务中的潜力。

### 7. 优点
*   **粒度更细**：相比于传统的“段落级”比较，ARHN 聚焦于“答案片段”，减少了背景噪声对 LLM 判断的干扰。
*   **成本效益高**：完全依赖开源模型，避免了调用昂贵商业 API 的成本，且更易于在大规模工业级数据上扩展。
*   **泛化能力强**：在分布外（OOD）数据集上的提升尤为明显，说明清洗标签噪声能显著增强模型的鲁棒性。

### 8. 不足与局限
*   **对 LLM 规模的依赖**：实验显示 8B 规模的模型可能会引入更多噪声甚至降低性能，这意味着该方法需要一定的算力门槛（至少 32B 级别模型）。
*   **推理开销**：两阶段的 LLM 推理（提取 + 排序）在大规模数据集（如数百万条记录）上仍然存在较高的计算时间成本。
*   **最大长度限制**：由于检索模型有最大序列长度限制，若答案片段出现在文档末尾（被截断处），ARHN 的优化效果可能会受限。
