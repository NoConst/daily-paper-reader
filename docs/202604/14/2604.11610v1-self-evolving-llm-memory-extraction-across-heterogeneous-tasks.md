## 摘要
随着基于大语言模型（LLM）的助手变得持久化和个性化，它们必须从过去的对话中提取并保留有用信息作为记忆。然而，值得记住的信息类型在不同任务之间存在显著差异。我们形式化了“异构记忆提取”任务，并推出了 BEHEMOTH，这是一个重新利用了 18 个现有数据集的基准测试，涵盖了个性化、问题解决和智能体任务，并使用下游效用驱动的指标进行系统评估。我们的实证分析证实，没有单一的静态提取提示词能在所有任务类别中占据主导地位，且现有的最初为同质分布设计的自演化提示词优化框架在训练任务具有异构性时性能会下降。为了解决这一问题，我们提出了 CluE，这是一种基于聚类的自演化策略，它根据提取场景将训练样本分组为簇，独立分析每个簇，并综合跨簇的见解来更新提取提示词。在 BEHEMOTH 上的实验表明，CluE 在异构任务中具有良好的泛化能力（相对提升 9.04%），始终优于先前的自演化框架。

## Abstract
As LLM-based assistants become persistent and personalized, they must extract and retain useful information from past conversations as memory. However, the types of information worth remembering vary considerably across tasks. We formalize the \textit{heterogeneous memory extraction} task and introduce \textbf{BEHEMOTH}, a benchmark that repurposes 18 existing datasets spanning personalization, problem-solving, and agentic tasks, using a downstream utility-driven metric for systematic evaluation. Our empirical analysis confirms that no single static extraction prompt dominates across all task categories, and that existing self-evolving prompt optimization frameworks, originally designed for homogeneous distributions, degrade when training tasks are heterogeneous. To address this, we propose \textbf{CluE}, a cluster-based self-evolving strategy that groups training examples into clusters by extraction scenarios, analyzes each cluster independently, and synthesizes cross-cluster insights to update the extraction prompt. Experiments on BEHEMOTH show that CluE generalizes effectively across heterogeneous tasks ($+$9.04\% relative gain), consistently outperforming prior self-evolving frameworks.

*   **核心问题**：现有的记忆提取系统通常依赖预定义的静态规则，或仅针对单一任务分布进行优化。在面对真实世界中多样且异构（Heterogeneous）的任务时，单一的提取提示词（Prompt）难以兼顾所有场景，且现有的自演化框架在异构数据分布下容易出现性能波动或过拟合。

### 2. 方法论：CluE（基于聚类的演化策略）
论文提出了 **CluE (Cluster-based Evolution)** 框架，旨在通过模拟人类“分类总结”的逻辑来优化提取提示词。其核心流程包含四个阶段：
1.  **总结（Summarization）**：针对训练批次中的每个样本，由 LLM 生成一个“提取场景总结”，描述需要提取的信息类型（如因果链、用户偏好）及提取难点。
2.  **聚类（Clustering）**：聚类管理器根据场景总结将样本分组，而不是根据原始数据集标签。这使得具有相似提取逻辑的任务（如数学题和代码调试）能被归为一类。
3.  **基于簇的分析（Cluster-based Analysis）**：独立分析每个簇内的成功和失败案例，识别该场景下的特定模式，并给出针对性的改进建议。
4.  **跨簇提案（Cross-cluster Proposal）**：提案器综合所有簇的分析报告，识别通用原则并保留场景特定的指导方针，最终合成一个新的、结构化的全局提取提示词。

### 6. 主要结论与发现
*   **无“银弹”提示词**：没有任何一个静态提示词能在所有异构任务中表现最优。
*   **异构挑战**：现有的自演化框架在处理异构任务时，往往会为了提升某一类任务的性能而损害另一类（如 GEPA 提升了智能体任务但损害了个性化任务）。

### 7. 优点
*   **基准测试贡献**：BEHEMOTH 填补了异构记忆提取评估的空白，整合了 18 个数据集并统一了评估协议。
*   **策略创新**：引入“提取场景聚类”而非“任务类型聚类”，更本质地捕捉了记忆提取的共性。
*   **实用性强**：演化出的提示词具有良好的结构（包含分类定义和通用准则），且能直接迁移到其他模型。

### 8. 不足与局限
*   **计算开销**：虽然比 ACE 快，但相比静态提示词，自演化过程仍需数百次 LLM 调用，存在一定的成本。
