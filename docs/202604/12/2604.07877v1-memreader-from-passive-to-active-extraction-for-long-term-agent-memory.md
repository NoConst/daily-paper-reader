## 摘要
长期记忆是实现个性化和自主智能体的基础，但如何填充记忆仍是一个瓶颈。现有系统将记忆提取视为从上下文到结构化条目的一次性、被动转录，这在处理噪声对话、缺失引用和跨轮次依赖时面临挑战，导致记忆污染、低价值写入和不一致性。在本文中，我们推出了用于智能体系统主动长期记忆提取的 MemReader 系列：MemReader-0.6B 是一个紧凑且具有成本效益的被动提取器，通过蒸馏实现准确且符合模式（schema）一致性的结构化输出；MemReader-4B 是一个通过群体相对策略优化（GRPO）进行优化的主动提取器，用于做出记忆写入决策。在 ReAct 风格的范式下，MemReader-4B 在行动前会明确评估信息价值、引用歧义和完整性，并能选择性地写入记忆、推迟不完整的输入、检索历史上下文或丢弃无关的闲聊。在 LOCOMO、LongMemEval 和 HaluMem 上的实验表明，MemReader 始终优于现有的基于提取的基准模型。特别是，MemReader-4B 在涉及知识更新、时间推理和减少幻觉的任务上达到了最先进的性能。这些结果表明，有效的智能体记忆不仅需要提取更多信息，还需要进行推理驱动的选择性记忆提取，以构建低噪声且动态演进的长期记忆。此外，MemReader 已集成到 MemOS 中，并正在实际应用中部署。为了支持未来的研究和应用，我们发布了模型并提供公开的 API 访问。

## Abstract
Long-term memory is fundamental for personalized and autonomous agents, yet populating it remains a bottleneck. Existing systems treat memory extraction as a one-shot, passive transcription from context to structured entries, which struggles with noisy dialogue, missing references, and cross-turn dependencies, leading to memory pollution, low-value writes, and inconsistency. In this paper, we introduce the MemReader family for active long-term memory extraction in agent systems: MemReader-0.6B, a compact and cost-efficient passive extractor distilled for accurate and schema-consistent structured outputs, and MemReader-4B, an active extractor optimized with Group Relative Policy Optimization (GRPO) to make memory writing decisions. Under a ReAct-style paradigm, MemReader-4B explicitly evaluates information value, reference ambiguity, and completeness before acting, and can selectively write memories, defer incomplete inputs, retrieve historical context, or discard irrelevant chatter. Experiments on LOCOMO, LongMemEval, and HaluMem show that MemReader consistently outperforms existing extraction-based baselines. In particular, MemReader-4B achieves state-of-the-art performance on tasks involving knowledge updating, temporal reasoning, and hallucination reduction. These results suggest that effective agent memory requires not merely extracting more information, but performing reasoning-driven and selective memory extraction to build low-noise and dynamically evolving long-term memory. Furthermore, MemReader has been integrated into MemOS and is being deployed in real-world applications. To support future research and adoption, we release the models and provide public API access.

### 1. 核心问题与研究动机
*   **核心问题**：现有的长期记忆系统（如 Mem0, Zep）通常采用“一次性被动提取”模式，即直接将对话转录为结构化条目。这种方式在处理**噪声对话（闲聊）**、**指代不明（代词）**、**信息不完整**以及**跨轮次依赖**时表现较差，容易导致记忆污染、低价值写入和状态不一致。
*   **研究背景**：长期记忆不应仅是输入的静态记录，而应是一个动态维护用户状态的机制。系统需要判断：这段信息值得记吗？现在能记吗？需要查历史记录来消除歧义吗？

### 2. 方法论
MemReader 提出了两种互补的模型方案，核心思想是将记忆提取建模为 **ReAct（推理+行动）** 范式。

*   **MemReader-0.6B**：针对成本敏感场景，通过蒸馏技术将高质量的结构化提取能力迁移到小模型中，专注于准确的 JSON 输出。
*   **MemReader-4B（核心贡献）**：基于 Qwen3-4B，采用 **ReAct 框架**和 **GRPO（群体相对策略优化）** 强化学习。
    *   **推理过程**：在行动前回答三个问题：价值判断（是否值得记）、歧义检测（是否需要检索历史）、完整性检查（是否需要缓冲等待）。
    *   **工具空间（Action Space）**：
        1.  `add_memory`：信息完整且有价值时写入。
        2.  `buffer_memory`：信息有用但不完整时暂存。
        3.  `search_memory`：检索历史记忆以消除指代歧义。
        4.  `ignore_memory`：丢弃低价值闲聊。
*   **训练流程**：
    1.  **SFT 预热**：使用 Gemini-3-Flash 生成的 7k 条轨迹数据进行监督微调。
    2.  **GRPO 强化学习**：设计了多维度奖励函数，包括**格式奖励**、**动作对齐奖励**（解决信用分配问题）、**LLM-Judge 语义奖励**（评估正确性、完整性、幻觉）和**效率奖励**（鼓励简洁）。

### 3. 实验设计
*   **数据集/基准测试**：
    1.  **LOCOMO**：评估长对话记忆（单跳、多跳、时间推理等）。

    3.  **HaluMem-Medium**：专门评估记忆系统中的幻觉、更新和 QA 能力。

*   **实验规模**：在三个主流的长期记忆基准测试上进行了全面评估，涵盖了从提取准确率到端到端 QA 的多个维度。
*   **消融与对比**：对比了 SFT 版本与 GRPO 版本的性能差异，并展示了 0.6B 模型与 4B 模型的不同侧重点。
*   **客观性**：使用了统一的 GPT-4o-mini 作为评估器，实验设计较为严谨，涵盖了知识更新和时间推理等高难度场景。

### 6. 主要结论与发现
*   **主动管理优于被动提取**：MemReader-4B 在知识更新和时间推理方面表现卓越，证明了“思考后再行动”能有效减少记忆冲突。
*   **强化学习的必要性**：GRPO 显著提升了模型的决策质量，尤其是在减少幻觉和提高记忆条目价值方面。
*   **小模型的潜力**：MemReader-0.6B 在纯结构化提取任务上甚至超过了基于 GPT-4o-mini 的基线，证明了任务特定蒸馏的有效性。
*   **记忆质量的传递性**：更高质量的记忆写入（低噪声、高准确）能显著提升下游问答任务的稳定性。

### 7. 优点（亮点）
*   **范式创新**：将记忆提取从简单的“翻译”任务提升为“状态管理”任务。
*   **工具化思维**：引入 `buffer` 和 `search` 工具，解决了对话中常见的指代不明和信息碎片化问题。
*   **高效的 RL 方案**：采用 GRPO 避免了训练额外的 Critic 模型，并通过多维度奖励设计解决了长轨迹下的信用分配难题。

### 8. 不足与局限
*   **训练数据规模**：用于强化学习的轨迹数据（3k）相对较小，可能限制了模型处理极复杂边缘案例的能力。
*   **推理开销**：ReAct 范式涉及多次推理和工具调用，虽然提升了质量，但相比一次性提取增加了推理延迟和 Token 消耗。
*   **工具集扩展性**：目前仅支持四种基础操作，尚未涉及记忆合并（Merging）、冲突自动消解（Conflict Resolution）或分层抽象等更高级的记忆组织功能。
