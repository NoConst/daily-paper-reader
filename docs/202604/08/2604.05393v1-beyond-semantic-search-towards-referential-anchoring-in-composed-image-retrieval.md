## 摘要
组合图像检索（CIR）通过结合参考图像和修改文本，实现了灵活的多模态查询，展现出巨大的潜力。然而，CIR 本质上优先考虑语义匹配，难以在不同语境下可靠地检索出用户指定的实例。在实践中，强调具体实例的保真度往往比宽泛的语义更具重要意义。在这项工作中，我们提出了对象锚定组合图像检索（OACIR），这是一种要求严格实例级一致性的新型细粒度检索任务。为了推进该任务的研究，我们构建了 OACIRR（真实图像上的 OACIR），这是首个大规模、多领域的基准测试，包含超过 16 万个四元组以及四个包含难负样本实例干扰项的挑战性候选库。每个四元组都通过边界框增强了组合查询，该边界框在视觉上锚定了参考图像中的对象，为确保实例保留提供了一种精确且灵活的方法。为了解决 OACIR 任务，我们提出了 AdaFocal 框架，该框架具有上下文感知注意力调制器，能够自适应地增强指定实例区域内的注意力，动态平衡锚定实例与更广泛组合上下文之间的关注点。大量实验表明，AdaFocal 显著优于现有的组合检索模型，特别是在保持实例级保真度方面，从而为这一具有挑战性的任务建立了一个强大的基准，同时也为更灵活、具有实例感知能力的检索系统开辟了新方向。

## Abstract
Composed Image Retrieval (CIR) has demonstrated significant potential by enabling flexible multimodal queries that combine a reference image and modification text. However, CIR inherently prioritizes semantic matching, struggling to reliably retrieve a user-specified instance across contexts. In practice, emphasizing concrete instance fidelity over broad semantics is often more consequential. In this work, we propose Object-Anchored Composed Image Retrieval (OACIR), a novel fine-grained retrieval task that mandates strict instance-level consistency. To advance research on this task, we construct OACIRR (OACIR on Real-world images), the first large-scale, multi-domain benchmark comprising over 160K quadruples and four challenging candidate galleries enriched with hard-negative instance distractors. Each quadruple augments the compositional query with a bounding box that visually anchors the object in the reference image, providing a precise and flexible way to ensure instance preservation. To address the OACIR task, we propose AdaFocal, a framework featuring a Context-Aware Attention Modulator that adaptively intensifies attention within the specified instance region, dynamically balancing focus between the anchored instance and the broader compositional context. Extensive experiments demonstrate that AdaFocal substantially outperforms existing compositional retrieval models, particularly in maintaining instance-level fidelity, thereby establishing a robust baseline for this challenging task while opening new directions for more flexible, instance-aware retrieval systems.

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题**：传统的 CIR 任务（通过“参考图 + 修改文本”检索目标图）主要关注**语义匹配**（如：将红裙子换成蓝裙子），但在实际应用中，用户往往希望检索的是**同一个特定实例**（如：检索同一件特定款式的衣服在不同场景下的样子）。现有模型在面对视觉相似的干扰项时，往往无法保证实例级的保真度（Instance Fidelity）。
*   **研究动机**：在数字记忆检索、长期身份追踪等场景下，保持具体实例的一致性比宽泛的语义对齐更重要。因此，作者提出将“指代锚定（Referential Anchoring）”引入 CIR，要求模型在执行文本修改的同时，严格锁定并保留参考图中的特定对象。

### 2. 方法论：AdaFocal 框架
*   **核心思想**：通过一个轻量级的模块，根据查询的上下文动态调整模型对锚定区域（Bounding Box）的注意力权重，在“保留实例特征”与“理解组合语境”之间取得平衡。
*   **关键技术细节**：
    *   **上下文感知注意力调制器（CAAM）**：利用可学习的“探测 Token（Probe Tokens）”与多模态输入交互，通过 Transformer 结构推理出当前查询所需的**调制标量 $\beta$**。
    *   **注意力激活机制**：在多模态编码器的交叉注意力计算中，将 $\beta$ 注入到与边界框（Bounding Box）对应的空间掩码中。公式表达为：在注意力分数矩阵上加上 $\beta \cdot M_{Br}$（$M_{Br}$ 为边界框掩码），从而动态增强对目标实例区域的聚焦。
    *   **对比对齐损失**：采用端到端的批次对比学习损失函数，将查询表示与目标图像表示投影到共享嵌入空间进行对齐。

### 3. 实验设计
*   **数据集（OACIRR）**：作者构建了首个大规模、多领域的 OACIR 基准数据集，包含 **16.1 万个四元组**（参考图、边界框、修改文本、目标图）。
*   **覆盖领域**：涵盖服装（Fashion）、汽车（Car）、商品（Product）和地标（Landmark）四个领域。
*   **Benchmark 设置**：
    *   **候选库构建**：为了增加难度，候选库中专门加入了大量“难负样本”（即同类别但不同实例的干扰图像）。
    *   **评价指标**：除了标准的 Recall@K，还引入了 **Instance Recall (RID@K)**，即只有检索到完全相同的实例才算正确。
*   **对比方法**：对比了通用多模态检索（UMR，如 UniIR, GME）、零样本 CIR（如 Pic2Word）以及有监督 CIR（如 SPRC）。

*   **硬件环境**：使用了 **4 台 Tesla V100 GPU (32GB)**。
*   **训练细节**：训练 20 个 epoch，batch size 为 128。
*   **优化器**：AdamW，采用了差异化学习率策略（CAAM 模块 1e-4，多模态编码器 1e-5）。

### 5. 实验数量与充分性
*   **实验规模**：论文在四个不同领域的数据集上进行了详尽的定量评估。
*   **消融实验**：非常充分。作者测试了 CAAM 的不同架构（Pooling vs. MLP vs. Transformer）、调制标量 $\beta$ 的取值影响、探测 Token 的数量、以及对边界框噪声（缩放和偏移）的鲁棒性。
*   **公平性**：通过将现有模型适配到 OACIR 任务（如将边界框信息转化为文本提示或直接裁剪 ROI），确保了在相同输入信息下的公平对比。

### 6. 主要结论与发现
*   **现有模型的局限**：即使是强大的多模态大模型（如 GME-7B），在没有针对性训练的情况下，其实例级检索能力依然有限，容易被视觉相似的干扰项误导。
*   **数据集的价值**：在 OACIRR 上进行微调能显著提升所有模型的实例保真度，证明了该数据集在填补“实例一致性”训练数据空白方面的作用。
*   **AdaFocal 的优越性**：AdaFocal 通过动态调制注意力，在 RID@1 指标上显著超过了固定注意力的基准模型，证明了“上下文感知”地分配注意力比盲目增强局部特征更有效。

### 7. 优点（亮点）
*   **任务创新**：首次正式定义了 OACIR 任务，将 CIR 从语义级推向了更实用的实例级。
*   **数据质量高**：OACIRR 数据集基于真实图像而非合成图像，且通过 MLLM 和人工校验确保了修改文本的复杂性和准确性。
*   **设计简洁高效**：AdaFocal 模块轻量且即插即用，不需要对主干网络进行大规模改动即可实现显著的性能提升。

### 8. 不足与局限
*   **对边界框的依赖**：虽然论文测试了鲁棒性，但在实际应用中，如果前端检测器完全失效或用户提供的边界框极度不准，模型性能仍会受限。
*   **计算开销**：虽然 CAAM 很轻量，但在推理阶段仍需多一轮前向计算来生成调制标量。
*   **领域覆盖**：虽然涵盖了四个领域，但对于更复杂的动态场景（如视频中的实例追踪检索）尚未涉及。
