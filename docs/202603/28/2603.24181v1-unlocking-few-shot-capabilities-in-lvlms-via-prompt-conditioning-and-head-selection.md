## 摘要
当前的大型视觉语言模型 (LVLM) 在图像描述、视觉问答和 OCR 等许多零样本任务中表现出色。然而，这些模型在图像分类任务中的表现较差，逊于基于 CLIP 的方法。值得注意的是，这一差距令人惊讶，因为许多 LVLM 使用了经过 CLIP 预训练的视觉编码器。然而，LVLM 本质上并不受限于 CLIP 具有独立视觉和文本编码器的架构。在 CLIP 中，这种分离使分类偏向于类名匹配，而非联合视觉-文本推理。在本文中，我们展示了尽管 LVLM 的原始性能较差，但它们可以在推理过程中利用提示词调节来提高视觉特征的类别可分性，且 LVLM 的内部表示（尤其是注意力头）在零样本和少样本分类中的表现可以优于模型本身。我们引入了注意力头集成分类器 (HEC)，以弥合基于 CLIP 和基于 LVLM 的分类方法之间的性能差距。受高斯判别分析的启发，HEC 对最具辨别力的视觉和文本头进行排序，并将它们组合成一个无需训练的分类器。我们证明了 HEC 在 12 个数据集的少样本和零样本分类中均达到了最先进的性能。

## Abstract
Current Large Vision Language Models (LVLMs) excel at many zero-shot tasks like image captioning, visual question answering and OCR. However, these same models suffer from poor performance at image classification tasks, underperforming against CLIP-based methods. Notably, this gap is surprising because many LVLMs use CLIP-pretrained vision encoders. Yet LVLMs are not inherently limited by CLIP's architecture with independent vision and text encoders. In CLIP, this separation biases classification toward class-name matching rather than joint visual-text reasoning. In this paper we show that, despite their poor raw performance, LVLMs can improve visual feature class separability at inference using prompt conditioning, and LVLMs' internal representations, especially attention heads, can outperform the model itself at zero-shot and few-shot classification. We introduce Head Ensemble Classifiers (HEC) to bridge the performance gap between CLIP-based and LVLM-based classification methods. Inspired by Gaussian Discriminant Analysis, HEC ranks the most discriminative vision and text heads and combines them into a training-free classifier. We show that HEC achieves state-of-the-art performance in few-shot and zero-shot classification across 12 datasets.

### 1. 核心问题与研究动机
*   **核心矛盾**：尽管许多大型视觉语言模型（LVLM）内置了强大的 CLIP 视觉编码器，但在图像分类任务（尤其是少样本和零样本场景）中的表现却远逊于原始的 CLIP 模型。
*   **研究动机**：作者发现 LVLM 的最终输出（Summary Token）未能充分释放其内部表示的潜力。CLIP 的视觉和文本编码器是独立的，缺乏推理时的交互；而 LVLM 虽然具备联合推理能力，但其分类性能却被“浪费”了。
*   **目标**：探索如何通过无需训练（Training-free）的方法，利用 LVLM 的内部机制来弥合与 CLIP 之间的分类性能差距。

### 2. 方法论
核心思想是结合**提示词调节（Prompt Conditioning）**和**注意力头选择（Head Selection）**，提出 **HEC（Head Ensemble Classifiers）**。

*   **提示词调节**：通过不同层级的文本引导特征提取：
    *   *任务调节（Task）*：如“这张图片里是什么？”
    *   *领域调节（Domain）*：如“这只狗是什么品种？”（针对细粒度任务）。
    *   *类别调节（Class）*：在提示词中加入候选类别列表。
*   **注意力头筛选与集成**：
    *   **视觉头（HEC-V）**：利用**高斯判别分析（GDA）**。在少样本支持集上，假设每个注意力头的特征符合高斯分布，计算每个头的类别可分性得分，筛选出 Top-k 个最具判别力的“视觉专家头”。
    *   **文本头（HEC-T）**：在支持集上评估各头对文本类名的零样本匹配准确率，筛选出“文本专家头”。
    *   **集成分类器（HEC-VT）**：将筛选出的视觉头和文本头的预测概率进行加权平均，构建最终分类器。
*   **算法流程**：输入查询图像和提示词 -> 提取 LLM 解码器各层的注意力向量 -> 根据支持集标签进行头排序 -> 集成 Top-k 头的输出概率 -> 得到分类结果。

### 3. 实验设计
*   **数据集**：涵盖 12 个多样化数据集，包括通用分类（ImageNet, Caltech101）、细粒度分类（OxfordPets, Flowers102, Stanford-Cars, FGVC-Aircraft, CUB-200）、场景（SUN397）、纹理（DTD）、动作（UCF101）及交通标志（Traffic-Signs）。
*   **Benchmark**：N-way K-shot（主要是 4-shot）少样本分类和零样本分类。
*   **对比方法**：
    *   *基于 CLIP 的 SOTA*：Tip-Adapter, GDA, ProKeR, Linear Probing。
    *   *基于 LVLM 的方法*：SAVs, 原始 Baseline（Next-token prediction）, Summary Token (ST)。
    *   *视觉模型*：DINOv1/v2/v3。

### 4. 资源与算力
*   **硬件**：实验在一张 **NVIDIA V100 GPU** 上完成。
*   **模型**：主要基于 **Qwen2-VL-7B** 和 **LLaVA-OV-7B**，同时也验证了 Idefics2 和 Finedefics。
*   **算力特点**：由于是无需训练的方法，主要开销在于特征提取（推理阶段）。作者指出，HEC 的分类器拟合速度比传统的线性探测（Linear Probing）快约 28 倍（以 Qwen2-VL 为例）。

### 5. 实验数量与充分性
*   **实验规模**：在 12 个异构数据集上进行了广泛测试，每项实验均使用 5 个随机种子取平均值。
*   **消融实验**：非常充分。涵盖了提示词层级（None vs Task vs Domain vs Class）、注意力头数量（k 的取值）、温度系数（$\tau$）、以及不同集成策略（投票法 vs 概率平均等）的对比。
*   **公平性**：对比 CLIP 时，使用了 LVLM 对应的同源视觉骨干（如 DFN, SigLIP），确保性能提升源于方法而非骨干网络差异。

### 6. 主要结论与发现
*   **提示词的力量**：提示词调节能显著增强 LVLM 内部特征的类别可分性，领域特定的提示词比通用提示词效果更好。
*   **内部专家头**：LVLM 内部存在少量注意力头，其分类能力远超模型最终的输出 Token。
*   **性能跨越**：HEC 在 12 个数据集上均达到了 SOTA，成功弥补了 LVLM 与 CLIP 之间的差距，甚至在非物体中心的数据集（如纹理、场景）上超越了 CLIP。
*   **通用性**：筛选出的“文本头”具有跨领域通用性，在 ImageNet 上选出的头可以直接应用于其他数据集。

### 7. 优点
*   **无需训练（Training-free）**：不需要任何参数微调，极大地降低了适配成本。
*   **即插即用**：可以轻松集成到现有的各种 LVLM 架构中。
*   **可解释性强**：通过可视化注意力图，证明了提示词确实引导模型关注到了图像中更具判别力的区域。
*   **效率高**：分类器构建过程极快，适合实时或资源受限的少样本场景。

### 8. 不足与局限
*   **推理成本**：尽管分类器构建快，但 LVLM 本身的推理（Forward Pass）比轻量级的 CLIP 慢得多，且显存占用更高。
*   **白盒依赖**：该方法需要访问模型的内部注意力头表示，因此无法应用于仅提供 API 调用的闭源模型（如 GPT-4, Claude）。
*   **上下文限制**：类别调节（Class Conditioning）受限于 LLM 的上下文窗口，当类别数量极多（如上千类）时，提示词会变得过长导致性能下降。
